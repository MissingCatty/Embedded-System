# 1.进程概念

进程是正在运行的【程序的实例】。

> 一个程序可以运行多个实例

是一个独立功能的程序关于某个数据集合的一次运行活动。

> 一个进程 = 程序代码 + 运行时数据

是操作系统动态执行的基本单元，在传统操作系统中，进程是资源分配的基本单位，也是基本的执行单元。

# 2.单道和多道程序

- 单道程序：计算机内存中只能有一个程序运行

  ```
  内存
  ┌─────────────┐
  │ 操作系统     │
  ├─────────────┤
  │ 进程A       │
  └─────────────┘
  ```

- 多道程序：计算机内存中可以加载多个程序

  使用程序A的过程中，当CPU空闲的时候（比如A等待磁盘IO），CPU可以跳到程序B上执行。

  ```
  内存
  ┌─────────────┐
  │ 操作系统     │
  ├─────────────┤
  │ 进程A       │
  ├─────────────┤
  │ 进程B       │
  ├─────────────┤
  │ 进程C       │
  └─────────────┘
  ```

# 3.时间片

一个进程在CPU上连续执行的时间。

给每个进程分配的时间片时间，涉及到分配算法。

# 4.并行和并发

- 并行：同时执行
- 并发：宏观上同步执行，微观上异步

# 5.进程管理

系统为每个进程设置一个PCB进程控制块，块中存放了各个进程的信息。

Linux内核的进程控制块是 `task_strut`结构体。

在 `/usr/src/linux-headers-xxx/include/linux/sched.h` 文件中可以查看 `struct task_struct` 结构体定义。其内部成员有很多，我们只需要掌握以下部分即可：

- **进程id**：系统中每个进程有唯一的 id，用 `pid_t` 类型表示，其实就是一个非负整数
- **进程的状态**：有就绪、运行、挂起、停止等状态
- **进程切换时需要保存和恢复的一些 CPU 寄存器**
- **描述虚拟地址空间的信息（进程内存布局和地址映射）**
- **描述控制终端的信息**

- **当前工作目录**
- **umask 掩码**
- **【重要】文件描述符表，包含很多指向 file 结构体的指针**
- **和信号相关的信息**
- **用户 id 和组 id**
- **会话（Session）和进程组**
- **进程可以使用的资源上限**
