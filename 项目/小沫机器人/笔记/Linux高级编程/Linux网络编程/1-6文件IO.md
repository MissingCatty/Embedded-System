# 1.概念

文件IO就是读写文件，而读写文件就是数据在内存和文件（硬盘）的流转。

一般来说，IO是**站在内存的角度**来讲的。

- 文件Input：把文件读取到内存
- 文件Output：把内存的数据存到文件里

# 2.标准C库的IO函数

**标准 c 库 IO 函数** vs **Linux 系统 IO 函数**是调用与被调用关系，标准 c 库 IO 函数更高级，Linux 系统 IO 函数更偏底层。

- 效率

  - **标准 c 库 IO 函数效率高**

    > 标准 C 库 IO 带有**用户态缓冲区**，它的“高效”在于减少了程序从用户态切换到内核态的**系统调用次数**（即减少了上下文切换的开销）。

  - Linux 系统 IO 虽然没有用户态缓冲区，但操作系统拥有**内核态缓冲区（Page Cache / Buffer Cache）**。你调用一次 `write`，数据只是写到了内核缓存中，由操作系统择机（异步）刷入磁盘，并不会立刻引发物理磁盘 IO

- 使用场景

  网络通信（Socket 编程）通常使用系统 IO 函数（如 `send`, `recv`, `read`, `write`），**主要原因是为了精准控制和兼容性**，而不是单纯的“效率”。

# 3.文件指针结构体

一般对文件进行读写需要

```c
FILE *fp = fopen("data.bin", "wb")
```

这句话中的文件指针`FILE *`实际上指向的是一个描述文件的结构体，该结构体存放了**对文件的描述**以及**文件读入到内存后的所在缓冲区的地址**。

> 文件读取实际上一次性读取（比如8KB）的数据进入缓冲区，缓冲区取完了之后，再从文件中读进来。类似拿一个固定大小的盆在蓄水池中舀水。

文件结构体中存放了以下几个东西：

- 文件描述符：一个整型值，文件的id
- 指向文件的指针：用于读取和写入文件数据，维护了两个指针，可对文件数据进行操作。
- IO缓冲区地址
  - 工作原理：使用标准 c 库 IO 函数写数据时，数据先写入缓冲区，缓冲区满后调用一次 Linux 系统 IO 函数将数据写入磁盘。
  - 写入时机：缓冲区数据满、强制调用 fflush 函数、正常关闭文件（如调用 fclose 或函数结束退出）时，缓冲区数据会写入磁盘。
  - 大小设置：缓冲区默认大小约 8K（8192 字节），这是理想大小，不建议调节。

# 4.虚拟地址空间

每一个进程运行起来后，都有自己的虚拟地址空间，所有虚拟内存空间都会被MMU映射到物理内存。

进程 A 里面的地址 `0x08048000` 和进程 B 里面的地址 `0x08048000`，虽然数字一模一样，但它们指向的是各自虚拟空间里的不同地方，最终会被映射到**完全不同的真实物理内存**上。

在现代操作系统（开启了 MMU）下，**任何一个运行中的普通程序，它能看到、能操作的所有内存地址，100% 都是虚拟地址**。

需要注意的是：**虚拟地址空间根本不占用真实的物理内存**。给一个进程分配 4GB 的虚拟地址空间，并不等于吃掉了电脑里 4GB 的内存条。

> **画大饼**：进程刚启动，操作系统大手一挥：“这 4GB 的地址（0x00000000 到 0xFFFFFFFF）全归你用了！”
>
> **实际占用极小**：但实际上，一个普通的程序（比如一个简单的 `Hello World` 或者小工具）它的代码、数据、变量加起来可能也就几百 KB。操作系统**只会把这几百 KB** 映射到真实的物理内存条上。
>
> **大片空白**：在那 4GB 的虚拟空间里，剩下的 3.999 GB 都是“未映射”的真空地带。它们只是一堆合法的地址数字，背后并没有真实的内存芯片做支撑。
>
> 假设你在代码里用 `malloc` 申请了一大块内存，操作系统依然只是在虚拟地址空间里给你划了一块“地皮”，还是没给真物理内存。
>
> 只有当你的代码**第一次真正往这块地皮上写数据时**（比如 `array[0] = 1;`）：
>
> 1. CPU 会突然发现：“哎呀，这个虚拟地址背后没有真内存啊！”（这在底层叫 **缺页中断 Page Fault**）。
> 2. 操作系统立刻被唤醒，赶紧跑去物理内存条上找一块空闲的小地方（通常是 4KB 大小的一页）。
> 3. 操作系统把这真实的 4KB 挂（映射）到那个虚拟地址后面。
> 4. 你的程序继续运行，完全不知道操作系统刚刚在底层手忙脚乱地做了一次“现货补给”。
>
> 这就叫**“按需分页（Demand Paging）”**。用多少，才给多少。

# 5.文件描述符

![image-20260310121104927](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2F9e5eff606a24c421a3db643c7ca391ca.png)

每一个进程都由一个进程控制块来维护，控制块中存放了一个文件描述符表（数组），用于管理打开的文件。

表的长度就是一个进程最大能够打开文件的数量（默认为1024）。

文件描述符表中初始时前三个是固定的，分别是

# 6.Linux系统IO函数

## 6.1 open函数

C语言中的fopen函数底层会调用Linux系统的open函数。

```c
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>

// 打开一个已经存在的文件
int open(const char *pathname, int flags);

// 创建一个新的文件
int open(const char *pathname, int flags, mode_t mode);
```

- flags：对文件的操作权限，以及其他设置

  - `O_RDONLY`：只读
  - `O_WRONLY`：只写
  - `O_RDWR`：读写

  三个权限互斥

- 返回值：一个新的文件描述符

  - 如果失败，返回的永远是`-1`，且操作系统在底层会偷偷修改一个叫`errno`的变量。使用`perror`会打印`errno`对应的错误信息。

>案例：尝试打开一个不存在的文件
>
>```c
>#include <sys/types.h>
>#include <sys/stat.h>
>#include <fcntl.h>
>#include <stdio.h>
>
>int main(void)
>{
>    int fd = open('a.txt', O_RDONLY);
>    if (fd == -1)
>    {
>        perror("error occur: ");
>    }
>    close(fd);
>    return 0;
>}
>```
>
>输出：
>
>```bash
>zyc@ubuntu:~/Linux/Lesson07$ ./open 
>error occur: : No such file or directory
>```

---

对于带三个参数的那个函数，通常用来创建新的文件。

```c
int open(const char *pathname, int flags, mode_t mode);
```

- flags：除了`O_RDONLY`，`O_WRONLY`，`O_RDWR`是**必选的**之外，还有可选的：

  - `O_APPEND`：如果不加这个，新写入的内容会把原来的覆盖掉，这个是在最后追加
  - `O_CREAT`：如果文件不存在，就会新创建一个

- mode：八进制数（**三位二进制，每一位表示可读/可写/可执行**），表示用户对新创建的文件的操作权限

  - 前提：标志位必须指定为`O_CREATE`或`O_TEMFILE`时才生效

  在linux中使用`ll`命令可以查看文件权限

  ![image-20260311101215395](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2F01d25ee69f5a28c1c4d6f56e37ff6d28.png)

  比如，`mode=0775`（0开头表示八进制数），`0777`表示最高权限，因为`111 111 111`（第一个表示该文件对当前用户的权限，第二个表示对用户所在用户组的权限，第三个表示对其他用户组的权限）

  然而，实际设置的权限结果为`mode & ~umask`，`umask`需要自己设置，默认为`0002`。

  ```
  ~umask = ~0002 = ~000 000 010 = 111 111 101
  ```

  就是把第2位权限关闭，即默认关闭其他用户组的修改文件的权限。

> 案例：创建新文件，并指定权限为最高
>
> ```C
> #include <sys/types.h>
> #include <sys/stat.h>
> #include <fcntl.h>
> #include <stdio.h>
> 
> int main(void)
> {
>     int fd = open("a.txt", O_RDONLY);
>     if (fd == -1)
>     {
>         perror("error occur: ");
>     }
>     close(fd);
> }
> ```

## 6.2 read和write函数

```c
#include <unistd.h>

ssize_t read(int fd, void *buf, size_t count);
```

- buf：读取数据需要存放的地方
- count：buf的大小

如果调用失败，返回`-1`，使用`perror`打印错误描述。

如果调用成功，**返回实际的读取到的字节数**。

```c
#include <unistd.h>

ssize_t write(int fd, const void *buf, size_t count);
```

- buf：数据来源
- count：要写的数据实际大小

返回值：

- 如果成功，返回实际写入字节数
- 如果失败，返回`-1`并设置`errno`

> 案例：功能包含：
>
> - 通过open打开english.txt文件
> - 创建一个新的文件（拷贝文件）
>
> ```c
> #include <sys/types.h>
> #include <sys/stat.h>
> #include <fcntl.h>
> #include <unistd.h>
> #include <stdio.h>
> 
> #define BUF_SIZE 64
> u_int8_t buf[BUF_SIZE];
> 
> int main(void)
> {
>     // 打开两个文件
> 	int fd_in = open("english.txt", O_RDONLY);
> 	if (fd_in == -1)
> 	{
> 		perror("open input file error");
> 		return -1;
> 	}
> 
> 	int fd_out = open("english_copy.txt", O_WRONLY|O_CREAT, 0777);
> 	if (fd_out == -1)
> 	{
> 		perror("open output file error");
> 		return -1;
> 	}
> 
>     // 数据搬运
> 	int bytes;
> 	while ((bytes = read(fd_in, buf, BUF_SIZE)) > 0)
> 	{
> 		printf("%d bytes read\n", bytes);
> 		write(fd_out, buf, bytes);
> 	}
> 	if (bytes == -1) {
> 		perror("read bytes error");
> 		return -1;
> 	}
> 	if (bytes == 0) {
> 		printf("file read complete\n");
> 	}
> 
>     // 关闭文件
> 	close(fd_out);
> 	close(fd_in);
> 	return 0;
> }
> ```

## 6.3 lseek函数

标准C库中的函数：

```c
#include <stdio.h>
int fseek(FILE *stream, long offset, int whence);
```

Linux的系统函数：

```c
#include <sys/types.h>
#include <unistd.h>

off_t lseek(int fd, off_t offset, int whence);
```

- 功能：改变文件读写位置（文件偏移量）

- offset：偏移量

- whence：偏移的参考位置

  | 宏         | 含义               |
  | ---------- | ------------------ |
  | `SEEK_SET` | 从文件开头开始     |
  | `SEEK_CUR` | 从当前读写位置开始 |
  | `SEEK_END` | 从文件末尾开始     |

- 返回值：

  | 返回值 | 含义                             |
  | ------ | -------------------------------- |
  | 成功   | 返回文件读写指针距离开头的偏移量 |
  | 失败   | 返回 `-1`                        |

本质：手动移动文件的读写指针

> 案例：获取文件大小
>
> ```c
> int size = lseek(fd, 0, SEEK_END);
> ```
>
> 案例：拓展文件长度
>
> ```c
> int size = lseek(fd, 100, SEEK_END);
> ```
>
> - 注意
>
>   1. 这里拓展的是文件的逻辑地址，和物理存储无关。文件在逻辑上是连续的，但在物理上是一块一块存储的。
>
>   2. 光执行这个移动文件读写指针还不够，只有往这个位置实际写入数据的时候，文件的大小才会发生变化。
>
>      ```c
>      write(fd, "A", 1);
>      ```
>
>      文件才会真正变成：
>
>      ```
>      0123456789[很多空洞]A
>      ```

## 6.4 stat和lstat函数

```c
#include <sys/types.h>
#include <sys/stat.h>
#include <unistd.h>

// 获取文件相关的信息
int stat(const char *pathname, struct stat *statbuf); // 跟随符号链接，返回链接指向的文件信息
int lstat(const char *pathname, struct stat *statbuf); // 不跟随符号链接，返回符号链接本身的信息
```

- statbuf：结构体指针，-O，结构体中保存了文件信息

  ```c
  struct stat {
      dev_t     st_dev;     // 文件的设备编号
      ino_t     st_ino;     // 节点
      mode_t    st_mode;    // 文件的类型和存取的权限
      nlink_t   st_nlink;   // 连到该文件的硬连接数目
      uid_t     st_uid;     // 用户ID
      gid_t     st_gid;     // 组ID
      dev_t     st_rdev;    // 设备文件的设备编号
      off_t     st_size;    // 文件字节数（文件大小）
      blksize_t st_blksize; // 块大小
      blkcnt_t  st_blocks;  // 块数
      time_t    st_atime;   // 最后一次访问时间
      time_t    st_mtime;   // 最后一次修改时间
      time_t    st_ctime;   // 最后一次改变时间（指属性）
  };
  ```

- 返回值：

  - 如果成功，返回0
  - 如果失败，返回-1，并设置errno

## 6.5 文件操作函数

### 6.5.1 access：查看文件权限

access主要负责查看文件的权限

```c
#include <unistd.h>

int access(const char *pathname, int mode);
```

- mode：
  - `F_OK`：是否有该文件
  - `R_OK`：当前**进程**对该文件是否有读权限
  - `W_OK`：写权限
  - `X_OK`：执行权限

- 返回值：
  - 成功，表示当前进程具有指定权限，返回0
  - 失败，没有该权限或文件不存在，返回-1，可通过errno查看错误信息

### 6.5.2 chmod：修改文件权限

```c
#include <sys/stat.h>

int chmod(const char *pathname, mode_t mode);
```

- mode：需要修改的权限值，一个8进制数
- 返回值
  - 成功返回0
  - 失败返回-1，errno

### 6.5.3 chown：修改文件所有者

```c
#include <unistd.h>

int chown(const char *pathname, uid_t owner, gid_t group);
```

- owner：所有者id

  ```bash
  # 查看所有者id
  vim /etc/passwd
  ```

  ```c
  # 用户名:
  zyc:x:1000:1000:robot,,,:/home/zyc:/bin/bash
  ```

  | 字段位置 | 内容        | 含义                                       |
  | -------- | ----------- | ------------------------------------------ |
  | 1        | `zyc`       | 用户名（login name）                       |
  | 2        | `x`         | 密码占位符（真实密码在 `/etc/shadow` 中）  |
  | 3        | `1000`      | 用户ID（UID）                              |
  | 4        | `1000`      | 组ID（GID）                                |
  | 5        | `robot,, ,` | 用户描述信息（GECOS 字段，如姓名、电话等） |
  | 6        | `/home/zyc` | 用户的home目录                             |
  | 7        | `/bin/bash` | 登录后使用的 shell                         |

- group：所在组id

  ```bash
  # 查看组信息
  vim /etc/group
  ```

  ```c
  lpadmin:x:120:zyc
  ```

  | 字段位置 | 字段       | 含义                                        |
  | -------- | ---------- | ------------------------------------------- |
  | 1        | group_name | 组名                                        |
  | 2        | password   | 组密码（通常为 `x`，实际在 `/etc/gshadow`） |
  | 3        | GID        | 组 ID                                       |
  | 4        | user_list  | 属于该组的用户列表                          |

### 6.5.4 truncate：扩展/缩减文件大小

```c
#include <unistd.h>
#include <sys/types.h>

int truncate(const char *path, off_t length);
```

- 作用：缩减/扩展文件的大小至`length`
- length：修改后文件的大小，单位字节
- 返回值
  - 成功，返回0
  - 失败，返回-1

## 6.6 目录操作函数

### 6.6.1 mkdir：创建目录

```c
#include <sys/stat.h>
#include <sys/types.h>

int mkdir(const char *pathname, mode_t mode);
```

- mode：目录权限，八进制值，实际权限还会受 `umask` 影响
- 返回值：成功返回 `0`；失败返回 `-1`，并设置 `errno`

### 6.6.2 rmdir：删除目录

```c
#include <unistd.h>

int rmdir(const char *pathname);
```

- pathname：目录路径，**必须是空目录**
- 返回值：成功返回 `0`；失败返回 `-1`，并设置 `errno`

### 6.6.3 rename：重命名或移动文件/目录

```c
#include <stdio.h>

int rename(const char *oldpath, const char *newpath);
```

- 返回值：成功返回 `0`；失败返回 `-1`，并设置 `errno`

### 6.6.4 chdir：改变当前工作目录

```c
#include <unistd.h>

int chdir(const char *path);
```

- `path`：要切换到的目录路径
- 返回值：成功返回 `0`；失败返回 `-1`，并设置 `errno`

### 6.6.5 getcwd：获取当前工作目录

```c
#include <unistd.h>

char *getcwd(char *buf, size_t size);
```

- buf：用于存储当前工作目录路径的缓冲区
- `size`：缓冲区大小
- 返回值：成功返回 `buf` 指针；失败返回 `NULL`，并设置 `errno`

### 6.6.6 opendir：打开目录

```c
#include <sys/types.h>
#include <dirent.h>

DIR *opendir(const char *name);
```

- 返回值：
  - 成功返回指向 **DIR 目录流** 的指针
  - 失败返回 `NULL`

### 6.6.7 readdir：读取目录项

```c
#include <dirent.h>

struct dirent *readdir(DIR *dirp);
```

- 返回值
  - 成功返回指向 **struct dirent** 的指针（表示一个目录项）
  - 读到目录末尾返回 `NULL`

> DIR是目录结构体，struct dirent是目录项结构体

### 6.6.8 closedir：关闭目录流

```c
#include <dirent.h>

int closedir(DIR *dirp);
```

- 成功返回 `0`；失败返回 `-1`

> `struct dirent` 常见成员
>
> ```
> struct dirent {
>     ino_t          d_ino;       // inode号
>     off_t          d_off;       // 偏移量
>     unsigned short d_reclen;    // 记录长度
>     unsigned char  d_type;      // 文件类型
>     char           d_name[256]; // 文件名
> };
> ```
>
> 最常用的是：
>
> ```
> d_name   // 文件名
> d_type   // 文件类型
> ```

> 案例：遍历目录
>
> ```c
> #include <stdio.h>
> #include <dirent.h>
> 
> int main()
> {
>     DIR *dir = opendir(".");
> 
>     struct dirent *entry;
> 
>     while ((entry = readdir(dir)) != NULL)
>     {
>         printf("%s\n", entry->d_name);
>     }
> 
>     closedir(dir);
>     return 0;
> }
> ```

## 6.7 dup和dup2函数

### 6.7.1 dup：复制文件描述符

```c
#include <unistd.h>

int dup(int oldfd);
```

**本质：给`oldfd`指向的文件分配一个新的文件描述符。**

在 Linux 中，进程里有一张 **文件描述符表**：

```
进程
│
├── fd 0 → stdin
├── fd 1 → stdout
├── fd 2 → stderr
├── fd 3 → file object
└── fd 4 → file object
```

假设执行：

```c
int fd = open("a.txt", O_RDWR);
```

内核大致结构：

```
fd table
   │
   └── fd=3
        │
        ▼
open file object
        │
        ├─ file offset = 0
        ├─ flags
        └─ inode
```

```c
int fd2 = dup(fd);
```

结构变成：

```
fd table
   │
   ├── fd=3 ─┐
   │         │
   └── fd=4 ─┘
              ▼
        open file object
              │
              ├─ file offset
              ├─ flags
              └─ inode
```

注意：

- **两个 fd 指向同一个 open file object**

- **共享文件偏移量**

  ```c
  int fd1 = open("a.txt", O_RDONLY);
  int fd2 = dup(fd1);
  
  read(fd1, buf, 10);
  read(fd2, buf, 10);
  ```

  第二次 read 会从 **第11字节开始读**。

  因为：

  ```
  fd1 ─┐
       ├── same file offset
  fd2 ─┘
  ```

### 6.7.2 dup2：把文件描述符复制到指定的 fd

```c
#include <unistd.h>

int dup2(int oldfd, int newfd);
```

- 让`newfd`也指向`oldfd`指向的文件

> 案例：重定向stdout
>
> ```c
> int tmp = dup(1);				// 把原本的stdout用一个新的fd存起来
> 
> dup(fd, 1);						// 把1重定向到文件
> printf("write to file\n");		// 向文件输出内容
> 
> dup(tmp, 1);					// 把1重定向到stdout
> printf("write to terminal\n");	// 向终端输出内容
> ```

## 6.8 fcntl函数

```c
#include <fcntl.h>

int fcntl(int fd, int cmd, ...);
```

| 参数  | 含义                   |
| ----- | ---------------------- |
| `fd`  | 文件描述符             |
| `cmd` | 控制命令               |
| `...` | 可选参数，取决于 `cmd` |

| 返回值 | 含义                      |
| ------ | ------------------------- |
| 成功   | 返回值取决于 `cmd`        |
| 失败   | 返回 `-1`，并设置 `errno` |

**常见cmd命令**

| cmd        | 作用               |
| ---------- | ------------------ |
| `F_DUPFD`  | 复制文件描述符     |
| `F_GETFD`  | 获取文件描述符标志 |
| `F_SETFD`  | 设置文件描述符标志 |
| `F_GETFL`  | 获取文件状态标志   |
| `F_SETFL`  | 设置文件状态标志   |
| `F_SETLK`  | 设置文件锁         |
| `F_SETLKW` | 设置文件锁（阻塞） |
| `F_GETLK`  | 获取文件锁状态     |

> **案例1：复制文件描述符**
>
> ```c
> int newfd = fcntl(oldfd, F_DUPFD, minfd);
> ```
>
> -  minfd：新文件描述符的 **最小值**，找到 **≥ minfd 的最小可用 fd**
>
> - 返回值
>
>   | 返回值  | 含义         |
>   | ------- | ------------ |
>   | ≥ minfd | 新文件描述符 |
>   | -1      | 失败         |
>
> ```c
> int newfd = fcntl(fd, F_DUPFD, 0);
> ```
>
> （与dup类似）
>
> **案例2：获取文件状态**
>
> ```c
> int flags = fcntl(fd, F_GETFL);
> ```
>
> 返回文件状态标志，例如：
>
> ```
> O_RDONLY
> O_WRONLY
> O_RDWR
> O_APPEND
> O_NONBLOCK # 非阻塞
> ```
>
> **案例3：设置文件状态**
>
> ```c
> fcntl(fd, F_SETFL, flags);
> ```
>
> 注意：
>
> - 只能修改 **部分状态标志**
> - 常用于设置：
>   - O_APPEND
>   - O_NONBLOCK
