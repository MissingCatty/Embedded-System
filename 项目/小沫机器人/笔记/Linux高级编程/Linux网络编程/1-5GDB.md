# 1.什么是GDB

GNU提出的一个调试是工具，同GCC组成的一套完整的开发环境，是Linux系统的标准开发环境。

包含下面四个功能：

- 启动程序
- 让程序在指定断点处停住
- 当程序被停住时，可以检查内存

# 2.准备工作

为调试而编译的时候，我们会关掉编译器的优化选项`-O`，并打开调试选项`-g`。

> 因为 **优化会改变代码结构**，导致调试很困难。
>
> ### 原代码
>
> ```
> int a = 10;
> int b = 20;
> int c = a + b;
> printf("%d\n", c);
> ```
>
> 如果开优化，编译器可能直接变成：
>
> ```
> printf("30");
> ```
>
> 甚至：
>
> - `a`、`b` 变量根本不存在
> - `c` 也可能不存在
>
> 当你用 **gdb 调试**：
>
> ```
> print a
> ```
>
> 可能会得到：
>
> ```
> optimized out
> ```
>
> 意思是：
>
> > 变量被优化掉了。

此外，尽可能的打开`-Wall`，提示所有warning信息。

```bash
gcc -g -Wall program.c -o program
```

`-g`：在可执行文件中加入源代码的信息，比如：机器码对应的是源代码中的哪一行

# 3.GDB使用

## 3.1 启动和退出

- 启动

  ```bash
  gdb 可执行程序
  ```

- 退出

  ```bash
  quit
  ```

## 3.2 示例

```cpp
#include <stdio.h>

int add(int a, int b) {
    int result = a + b;
    return result;
}

int factorial(int n) {
    int result = 1;

    for (int i = 1; i <= n; i++) {
        result = result * i;
    }

    return result;
}

int main() {
    int x = 3;
    int y = 4;

    int sum = add(x, y);
    printf("sum = %d\n", sum);

    int n = 5;
    int fact = factorial(n);

    printf("factorial(%d) = %d\n", n, fact);

    return 0;
}
```

执行

```bash
g++ -g -Wall main.cpp -o main
```

调试

```bash
gdb main
```

| 步骤 | 输入命令                 | 含义                       | 典型输出 / 结果                                              |
| ---- | ------------------------ | -------------------------- | ------------------------------------------------------------ |
| 1    | `gdb ./main`             | 启动 gdb 调试程序          | 进入 `(gdb)` 提示符                                          |
| 2    | `break main` 或 `b main` | 在 `main` 函数设置断点     | `Breakpoint 1 at ... file debug_demo.c, line 18`             |
| 3    | `run`                    | 运行程序                   | 程序运行到 `main` 停止：`Breakpoint 1, main() at debug_demo.c:18` |
| 4    | `list`                   | 查看源码                   | 显示当前附近代码                                             |
| 5    | `next` 或 `n`            | 执行一行代码（不进入函数） | 程序执行 `x = 3`，停在 `int y = 4`                           |
| 6    | `print x` 或 `p x`       | 查看变量 `x` 的值          | `$1 = 3`                                                     |
| 7    | `next`                   | 执行下一行                 | 停在 `int sum = add(x, y)`                                   |
| 8    | `print y`                | 查看变量 `y`               | `$2 = 4`                                                     |
| 9    | `step` 或 `s`            | 进入函数调用               | 进入 `add (a=3, b=4)`                                        |
| 10   | `print a`                | 查看函数参数 `a`           | `$3 = 3`                                                     |
| 11   | `print b`                | 查看函数参数 `b`           | `$4 = 4`                                                     |
| 12   | `next`                   | 执行函数中的一行           | 停在 `return result`                                         |
| 13   | `print result`           | 查看计算结果               | `$5 = 7`                                                     |
| 14   | `next`                   | 返回 `main` 函数           | 停在 `printf("sum = %d\n", sum)`                             |
| 15   | `print sum`              | 查看 `sum` 值              | `$6 = 7`                                                     |
| 16   | `next`                   | 执行 `printf`              | 输出 `sum = 7`                                               |
| 17   | `backtrace` 或 `bt`      | 查看函数调用栈             | `#0 main() at debug_demo.c`                                  |
| 18   | `continue` 或 `c`        | 继续运行程序               | 输出 `factorial(5) = 120` 程序结束                           |

- 补充

  - 步出是`finish`命令

  - 删除断点

    > 先查看断点：
    >
    > ```
    > (gdb) info breakpoints
    > ```
    >
    > 输出类似：
    >
    > ```
    > Num     Type           Disp Enb Address            What
    > 1       breakpoint     keep y   0x0000000000401136 in main at debug_demo.c:18
    > ```
    >
    > 删除断点 **1**：
    >
    > ```
    > (gdb) delete 1
    > ```

    > 删除所有断点
    >
    > ```
    > (gdb) delete
    > ```
    >
    > gdb 会询问：
    >
    > ```
    > Delete all breakpoints? (y or n)
    > ```
    >
    > 输入：
    >
    > ```
    > y
    > ```

    每次进入gdb都需要重新设置

总结得到各命令的汇总如下：

| GDB命令                                   | 功能             | 说明                               |
| ----------------------------------------- | ---------------- | ---------------------------------- |
| `gdb 可执行程序`                          | 启动 GDB         | 启动 gdb 并加载指定的可执行程序    |
| `quit`                                    | 退出 GDB         | 退出 gdb 调试环境                  |
| `set args 10 20`                          | 设置程序参数     | 设置程序运行时的命令行参数         |
| `show args`                               | 查看程序参数     | 查看当前设置的程序运行参数         |
| `help`                                    | 查看帮助         | 查看 gdb 的帮助信息                |
| `list` 或 `l`                             | 查看当前文件代码 | 从默认位置开始显示当前源代码       |
| `list 行号` 或 `l 行号`                   | 查看指定行代码   | 从指定行号开始显示源代码           |
| `list 函数名` 或 `l 函数名`               | 查看函数代码     | 从指定函数位置开始显示源代码       |
| `list 文件名:行号` 或 `l 文件名:行号`     | 查看其他文件代码 | 从指定文件的某一行开始显示代码     |
| `list 文件名:函数名` 或 `l 文件名:函数名` | 查看其他文件函数 | 显示指定文件中某个函数的代码       |
| `show listsize`                           | 查看显示行数     | 查看 `list` 命令默认显示的代码行数 |
| `set listsize 行数`                       | 设置显示行数     | 设置 `list` 命令一次显示的代码行数 |

| GDB命令                                           | 功能     | 说明                                             |
| ------------------------------------------------- | -------- | ------------------------------------------------ |
| `b 行号` / `break 行号`                           | 设置断点 | 在指定行号处设置断点                             |
| `b 函数名` / `break 函数名`                       | 设置断点 | 在指定函数入口处设置断点                         |
| `b 文件名:行号` / `break 文件名:行号`             | 设置断点 | 在指定文件的某一行设置断点                       |
| `b 文件名:函数` / `break 文件名:函数`             | 设置断点 | 在指定文件中的函数入口处设置断点                 |
| `i b` / `info break`                              | 查看断点 | 查看当前程序中所有断点信息                       |
| `d 断点编号` / `del 断点编号` / `delete 断点编号` | 删除断点 | 删除指定编号的断点                               |
| `dis 断点编号` / `disable 断点编号`               | 禁用断点 | 让指定断点暂时失效但不删除                       |
| `ena 断点编号` / `enable 断点编号`                | 启用断点 | 重新启用已经被禁用的断点                         |
| `b 10 if i==5` / `break 10 if i==5`               | 条件断点 | 当程序运行到第10行且满足条件 `i==5` 时才触发断点 |

| GDB命令                      | 功能              | 说明                                           |
| ---------------------------- | ----------------- | ---------------------------------------------- |
| `start`/`quit`               | 启动/退出程序调试 | 启动程序并停在程序第一行                       |
| `run`                        | 运行程序          | 运行程序，遇到断点才会停止                     |
| `c` / `continue`             | 继续运行          | 从当前位置继续执行程序，直到遇到下一个断点     |
| `n` / `next`                 | 单步执行          | 向下执行一行代码，但不会进入函数体             |
| `s` / `step`                 | 单步进入          | 向下单步执行，若遇到函数会进入函数内部         |
| `finish`                     | 跳出函数          | 执行完当前函数剩余代码并返回到调用处           |
| `p 变量名` / `print 变量名`  | 打印变量值        | 输出指定变量当前的值                           |
| `ptype 变量名`               | 查看变量类型      | 输出指定变量的数据类型                         |
| `display 变量名`             | 自动显示变量      | 每次程序停止时自动打印该变量的值               |
| `i display` / `info display` | 查看自动显示变量  | 查看当前所有自动显示的变量                     |
| `undisplay 编号`             | 取消自动显示      | 取消指定编号的自动显示变量                     |
| `set var 变量名=变量值`      | 修改变量值        | 在调试过程中手动修改变量的值                   |
| `until`                      | 跳出循环          | 执行程序直到跳出当前循环或到达更后面的代码位置 |
