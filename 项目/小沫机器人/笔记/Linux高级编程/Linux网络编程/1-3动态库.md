# 1.动态库的格式

- Linux：libxxx.so
- Windows：libxxx.dll

# 2.动态库的制作

- gcc得到.o文件

  ```bash
  gcc -c -fpic a.c b.c
  ```

  - fpic：指定编译出来的文件不使用绝对地址，使用相对寻址。使得该段代码放在任何地方都能运行。

- gcc得到动态库

  ```bash
  gcc -shared a.o b.o -o libmath.so
  ```

# 3.动态库的使用

## 3.1 动态库使用时出现的问题

现在，将目录整理成比较规范的样子：

```bash
.
├── inc
│   └── math.h
├── lib
│   └── libmath.so
└── main.cpp
```

编译main.cpp的时候需要联合刚刚制作好的动态库和头文件：

```bash
g++ main.cpp -I inc/ -L lib/ -lmath
```

现在执行输出的文件`a.out`，发现报错：

```bash
zyc@ubuntu:~/Linux/Lesson04$ ./a.out 
./a.out: error while loading shared libraries: libmath.so: cannot open shared object file: No such file or directory
```

说明该目标文件的运行是依赖于动态库的，其库文件的代码没有拼到编译后的目标文件里，是在运行的时候组装的。

可以使用ldd命令查看可执行程序依赖的动态库：

```bash
zyc@ubuntu:~/Linux/Lesson04$ ldd ./a.out 
        linux-vdso.so.1 (0x00007ffed01eb000)
        libmath.so => not found
        libstdc++.so.6 => /lib/x86_64-linux-gnu/libstdc++.so.6 (0x00007fd80442b000)
        libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6 (0x00007fd804239000)
        libm.so.6 => /lib/x86_64-linux-gnu/libm.so.6 (0x00007fd8040ea000)
        /lib64/ld-linux-x86-64.so.2 (0x00007fd804623000)
        libgcc_s.so.1 => /lib/x86_64-linux-gnu/libgcc_s.so.1 (0x00007fd8040cf000)
```

发现其依赖的`libmath.so`找不到，这个动态库的地址是存在哪里的呢？

- 首先到可执行文件的`DT_RPATH`段找依赖库（一般不可修改）
- 如果没找到，则到环境变量`LD_LIBRARY_PATH`指定的路径中查找

- 如果还没找到，则到`/etc/ld.so.cache`文件中查找动态库的路径
- 如果还没找到，则到`/usr/lib/`目录中找到

根据上面这些方法，发现可行且方便的只有第二种，只要修改环境变量即可，方法二和三都要把库文件放到指定的系统目录，不是很方便。

## 3.2 问题解决

在环境变量`LD_LIBRARY_PATH`中加入指定的动态库路径即可：

```bash
# 先获取原先环境变量的值再拼接
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/home/zyc/Linux/Lesson04/lib/
```

- 注意：命令不要随便带空格

现在再执行

```bash
./a.out
```

就能成功执行了。
