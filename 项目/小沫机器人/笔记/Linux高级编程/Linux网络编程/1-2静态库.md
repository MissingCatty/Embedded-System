# 1.静态库和动态库

- 静态库：程序链接阶段被复制到程序中
- 动态库：在链接时没有被复制到程序中，而是在程序运行时被加载到内存中供程序调

---

**库有什么好处？**

- 程序加密
- 方便移植

# 2.静态库的格式

在Linux中，静态库标准的命名格式为`libxxx.a`

- `lib`：前缀
- `xxx`：库的名字，自己起

在Windows中，为`libxxx.lib`

# 3.静态库的制作

静态库本质上是**提前编译好的目标文件的集合**，在链接阶段可以被直接复制到可执行程序里。

所以步骤很清晰：

- 先gcc编译生成目标文件

- 再多个目标文件打包

  ```bash
  ar rcs libxxx.a xxx.o xxx.o
  ```

  - r：将文件charity备存文件中
  - c：建立备存文件
  - s：索引

---

假设我们现在有加减乘除四个函数

```
mathlib/
│
├─ add.cpp
├─ sub.cpp
├─ mul.cpp
├─ div.cpp
│
├─ math.h
│
└─ main.cpp
```

```cpp
#include "math.h"

int add(int a, int b) {
    return a + b;
}

#include "math.h"

int sub(int a, int b) {
    return a - b;
}

#include "math.h"

int mul(int a, int b) {
    return a * b;
}

#include "math.h"

int divide(int a, int b) {
    return a / b;
}
```

```cpp
#ifndef MATH_H
#define MATH_H

int add(int a, int b);
int sub(int a, int b);
int mul(int a, int b);
int divide(int a, int b);

#endif
```

首先编译成目标文件

```bash
g++ -c add.cpp sub.cpp div.cpp mul.cpp
```

使用`ar`工具打包成静态库

```bash
ar rcs libmath.a add.o sub.o mul.o div.o
```

发现文件夹下出现了一个`libmath.a`文件。

# 4.静态库的使用

上一步我们已经准备好了一个静态库，现在这个库怎么使用呢？

```cpp
#include <iostream>
#include "math.h"

int main() {

    std::cout << add(10,5) << std::endl;
    std::cout << sub(10,5) << std::endl;
    std::cout << mul(10,5) << std::endl;
    std::cout << divide(10,5) << std::endl;

    return 0;
}
```

在编译的时候将静态库加到命令里

```bash
g++ main.cpp -L. -lmath
```

- `-L.` ：在当前目录找库
- `-lmath`：找 libmath.a（因为`-lxxx  == libxxx.a`）

## **注意！！！（重中之重）**

现在删掉库：

```
rm libmath.a
```

再运行：

```
./a.out
```

程序 **仍然可以运行**。

因为：

> 静态库的代码已经被复制进 a.out 里了。
