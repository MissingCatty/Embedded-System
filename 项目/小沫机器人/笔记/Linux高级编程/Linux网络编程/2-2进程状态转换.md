# 1.进程的状态

**三态模型**

- 就绪态：资源够，时间片不够
- 运行态：资源和时间片都够
- 阻塞态：资源不够、时间片结束

---

**五态模型**

- 就绪态
- 运行态
- 阻塞态
- 新建态、终止态
  - 新建态：刚被创建时的状态，尚未进入就绪队列
  - 终止态：进程完成到达正常结束点，或出现错误而异常中止，或被操作系统终止。一旦进入终止态，进程不再执行。主要用于被父进程获取信息。

# 2.进程相关命令

## 2.1 查看进程

```bash
ps aux / ajx
```

- a：显示终端上的所有进程，包括其他用户的进程
- u：显示进程的详细信息
- x：显示没有控制终端的进程
- j：列出与作业控制相关的信息

> aux
>
> ```bash
> zyc@ubuntu:~/Linux/Lesson10$ ps -aux
> USER         PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
> root           1  0.0  0.0 169572 12808 ?        Ss   Mar05   0:12 /sbin/init auto noprompt
> root           2  0.0  0.0      0     0 ?        S    Mar05   0:00 [kthreadd]
> root           3  0.0  0.0      0     0 ?        I<   Mar05   0:00 [rcu_gp]
> root           4  0.0  0.0      0     0 ?        I<   Mar05   0:00 [rcu_par_gp]
> ```
>
> ajx
>
> ```bash
> zyc@ubuntu:~/Linux/Lesson10$ ps -ajx
>    PPID     PID    PGID     SID TTY        TPGID STAT   UID   TIME COMMAND
>       0       1       1       1 ?             -1 Ss       0   0:12 /sbin/init auto noprompt
>       0       2       0       0 ?             -1 S        0   0:00 [kthreadd]
>       2       3       0       0 ?             -1 I<       0   0:00 [rcu_gp]
>       2       4       0       0 ?             -1 I<       0   0:00 [rcu_par_gp]
>       2       5       0       0 ?             -1 I<       0   0:00 [slub_flushwq]
>       2       6       0       0 ?             -1 I<       0   0:00 [netns]
> ```
>
> - COMMAND：运行该进程对应的指令
> - TTY：进程所属终端
> - SID：会话id

## 2.2 动态显示进程信息

```bash
top
```

在 `top` 命令执行后，可以按以下按键对显示的结果进行排序：

- **M**：根据内存使用排序
- **P**：根据 CPU 占有率排序
- **T**：根据进程运行时间长短排序
- **U**：根据用户名来筛选进程
- **K**：输入指定的 PID 杀死进程

## 2.3 杀死进程

| 命令                   | 功能               | 说明                                          |
| ---------------------- | ------------------ | --------------------------------------------- |
| `kill [-signal] pid`   | 向指定进程发送信号 | `pid` 为进程ID，`signal` 为信号类型           |
| `kill -l`              | 查看所有信号       | 列出系统支持的所有信号编号和名称              |
| `kill -SIGKILL 进程ID` | 强制终止进程       | 向进程发送 `SIGKILL` 信号，进程无法捕获或忽略 |
| `kill -9 进程ID`       | 强制终止进程       | `9` 是 `SIGKILL` 的信号编号，效果同上         |
| `killall name`         | 按进程名终止进程   | 根据进程名结束所有匹配的进程                  |
