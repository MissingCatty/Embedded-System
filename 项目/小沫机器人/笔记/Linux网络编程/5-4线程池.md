# 1.概念

由服务器预先创建一组子线程，线程池中的线程数量应该和cpu数量差不多。

> IO密集型的一般线程数要大于cpu核心数，因为IO密集型大部分事件花在IO上，此时线程是阻塞的，为了让CPU利用率高起来，应该多给一些线程

线程池中所有子线程都运行着相同的代码。当有新任务到来的时候，主线程通过某种方式，选择线程池中某一个子线程来进行服务。

相比于动态的创建线程，开销要小得多。

# 2.线程选择方式

具体如何选择线程池中的哪一个线程呢？

- 随机挑选
- （Round Robin算法）轮流选取
- 其他更优秀和智能的算法，均匀分配工作线程到任务上，减轻服务器整体压力

# 4.处理完任务后的线程应该怎么办

处理完客户端任务后，把相关资源放回池子里，无需执行系统调用释放资源。

# 5.实际案例

需要开辟两个空间，

- 线程池

- 任务队列

  谁会来操作任务队列？

  - 主线程：往队列里添加任务
  - 工作线程：从队列里取任务

  因此，需要互斥访问队列。

  同时，队列涉及生产者和消费者模型

  - 生产者：主线程，往队列里放入任务后，需要增加一个信号量
  - 消费者：工作线程，从队列里取出任务后，需要减少一个信号量

  > 如果队列有最大任务量需求，则还需要设置一个初始值为队列最大长度的信号量（**一共两个信号量**），
  >
  > - 主线程：放入一个任务，则减少一个信号量
  > - 工作线程：取出一个任务，则增加一个信号量
  >
  > **为什么要设置两个？**
  >
  > 因为信号量值能保证当信号量小于等于0的时候，阻塞；
  >
  > 不能保证当值大于某个值的时候阻塞。
  >
  > 正确的信号量释放和持有的顺序如下：
  >
  > - 务必先确认可以放入任务
  >
  > ```c
  > // 先确认任务可以放
  > _task_remain_semaphore.wait();
  > 
  > // 持有互斥锁并放入一个任务
  > _mutex.lock();
  > _work_queue.push(task);
  > _mutex.unlock();
  > 
  > // 任务信号量+1
  > _task_count_semaphore.post();
  > ```

# 6.基础模板

**线程池类声明**

```c
#ifndef THREADPOOL_H
#define THREADPOOL_H

#include <pthread.h>
#include <queue>
#include "mutex.h"
#include "sem.h"

template <typename T>
class ThreadPool
{
private:
    int _thread_total;           // 线程总数
    pthread_t *_threads;         // 线程数组
    int _task_total;             // 任务队列最大长度
    std::queue<T *> _work_queue; // 任务队列
    Mutex _mutex;                // 互斥锁（任务队列所有线程共享）
    Sem _task_count_semaphore;   // 队列中任务数信号量（任务队列空）
    Sem _task_remain_semaphore;  // 队列剩余可放任务数信号量（任务队列满）
    bool _thread_stop;           // 线程是否终止

    static void *work(void *arg); // 线程工作函数

public:
    ThreadPool(int max_thread_count = 4, int max_task_count = 100);
    ~ThreadPool();
    bool append(T *task); // 任务队列添加函数
    void run();           // 线程池运行函数
};

#endif
```

**类定义**

```c
#include "threadpool.h"

template <typename T>
void *ThreadPool<T>::work(void *arg)
{
    // 每个工作线程都运行这一个函数

    // 获取线程池对象
    ThreadPool<T> *pool = (ThreadPool<T> *)arg;
    // 统一调用执行函数
    pool->thread_run();

    return nullptr;
}

template <typename T>
ThreadPool<T>::ThreadPool(int thread_total, int task_total)
    : _thread_total(thread_total),
      _task_total(task_total),
      _threads(NULL),
      _work_queue(),                      // 初始化任务队列
      _task_count_semaphore(0),           // 初始化信号量（表示队列中任务数）
      _task_remain_semaphore(task_total), // 初始化信号量（表示队列里可放置的任务数）
      _mutex(),                           // 初始化互斥锁
      _thread_stop(false)                 // 初始时线程不停止
{
    if (thread_total <= 0 || task_total <= 0)
    {
        throw std::invalid_argument("thread_total or task_total must > 0");
    }

    // 初始化线程数组
    _threads = new pthread_t[thread_total];
}

template <typename T>
ThreadPool<T>::~ThreadPool()
{
    _thread_stop = true;
    for (int i = 0; i < _thread_total; i++)
    {
        pthread_join(_threads[i], NULL);
    }
    delete[] _threads;
}

// 任务队列添加任务
template <typename T>
bool ThreadPool<T>::append(T *task)
{
    // 先确认任务可以放
    _task_remain_semaphore.wait();

    // 持有互斥锁并放入一个任务
    _mutex.lock();
    _work_queue.push(task);
    _mutex.unlock();

    // 任务信号量+1
    _task_count_semaphore.post();

    return true;
}

template <typename T>
void ThreadPool<T>::thread_run()
{
    // 功能：工作线程不停的从任务队列里取任务运行

    while (!_thread_stop)
    {
        // 先看队列是否为空
        _task_count_semaphore.wait();

        _mutex.lock();
        T *task = _work_queue.front();
        _work_queue.pop();
        _mutex.unlock();

        // 队列可放信号量+1
        _task_remain_semaphore.post();

        // 任务运行接口
        // task->process();
    }
}

template <typename T>
void ThreadPool<T>::run()
{
    // 创建线程
    for (int i = 0; i < _thread_total; i++)
    {
        if (pthread_create(_threads + i, NULL, work, this) != 0) // 传递this指针给work函数！！！
        {
            delete[] _threads;
            throw std::runtime_error("thread create failed");
        }
    }
}

```

- 注意点
  1. 放入任务队列的两个信号量+一个互斥量及其顺序
     - 需要检查的那个信号量放开头
     - 互斥访问临界资源
     - 资源访问完毕再释放信号量
  2. 每个工作线程执行同一套代码，都是不断的去队列里取任务再执行
     - 线程的选择无法人为控制，由操作系统的任务调度算法选择
  3. 为什么工作函数要写成static？
     - 因为pthread_create要求绑定的函数格式必须严格符合`void *func(void *arg)`，如果是普通成员函数，其中隐含一个this指针
  4. 线程创建必须另外用一个函数，不能写在构造函数里
     - 因为构造函数执行时对象没有构造完毕，如果此时立刻创建线程调用`work`并传入this指针，容易发生数据不一致的问题。（可以在初始化列表中构造，保证成员全部构造完毕再进入构造函数函数体，但不规范）
