# 1.概念

两个或多个进程共享内存里的一块区域（通常称为段），这块**内存是由内核管理**的，用户进程把自己的一部分和共享内存长度相等的一段地址**建立映射**后，**用户进程每次往自己的这段地址上写数据时，实际写入的是共享内存中的地址**（不涉及数据的复制搬运）。

与管道相比，由于管道是创建在内核区的，操作管道需要涉及到用户态到内核态的切换，共享内存方式开销比较小。并且数据需要在内核空间与用户空间之间拷贝（例如写入时从用户缓冲区拷贝到内核管道缓冲区，读取时再从内核拷贝回用户缓冲区）。

# 2.使用方法

## 2.1 `shmget()`：拿到共享内存的“门牌号”

```c
#include <sys/ipc.h>
#include <sys/shm.h>

int shmget(key_t key, size_t size, int shmflg);
/*
	- key：表示共享内存的键值。不同进程如果想访问同一块共享内存，通常要使用同一个 key。
	- size：共享内存大小，单位是字节。创建新共享内存时这个值很重要；如果只是获取已有共享内存，这个值通常仍要写，但不会真正重新分配。
	- shmflg：控制标志
		- IPC_CREAT：如果不存在就创建
		- IPC_EXCL：和 IPC_CREAT 一起用，表示若已存在则报错
		- 权限位，比如 0666、0644（会受umask影响）
	- 返回值：
		- 成功，返回一个共享内存标识符 shmid，后面 shmat 和 shmctl 都要用它。
		- 失败，返回 -1。
*/
```

把共享内存想成一个**公共仓库**，那 `shmget()` 干的事就是：

- 要么新建一个仓库
- 要么找到已经存在的仓库

然后把这个仓库的编号给你。这个返回值不是内存地址，**只是一个标识符**，类似“仓库编号”。

常见的用法为：

```c
shmget(key, 1024, IPC_CREAT | 0666);
```

---

**key和返回的shmid是干嘛的？**

key是多个进程在开始时用来获取同一块共享内存的id的，获取到id后，后面操作就都用id而不用key了。

---

**key怎么来的？**

使用ftok函数，**用来生成 IPC 键值 `key_t` 的函数**。

根据一个已存在的文件路径 + 一个项目号，生成一个 key，方便多个进程约定使用同一个 IPC 资源。

这个 IPC 资源一般包括：

- 消息队列
- 共享内存
- 信号量

```c
#include <sys/types.h>
#include <sys/ipc.h>

key_t ftok(const char *pathname, int proj_id);
/*
	- pathname：一个已经存在的文件路径。（一个占位文件，起标识符作用，没有实际意义，通常为空）
	- proj_id：项目号，通常写一个整数或者字符。常见写法：A或1。它的作用是让同一个文件路径还能区分出不同的 key。
	- 返回值：
		- 成功，返回一个 key_t 类型的键值
		- 失败：返回 -1
*/
```

## 2.2 `shmat()`：把共享内存“映射”到当前进程

拿到仓库编号后，你还不能直接用。

因为这时这块共享内存还只是“存在于内核中”，**还没接到你这个进程的地址空间里**。

```c
#include <sys/ipc.h>
#include <sys/shm.h>

void *shmat(int shmid, const void *shmaddr, int shmflg);

/*
	- shmid：共享内存标识符，来自 shmget 的返回值。
	- shmaddr：指定附加到进程地址空间中的哪个地址（一般写 NULL，表示让系统自动选择合适地址）
	- shmflg：附加方式标志
		- 0：默认，可读可写
		- SHM_RDONLY：只读附加
	- 返回值：
		- 成功，返回共享内存映射后的起始地址，也就是一个指针。后续你就通过这个地址访问共享内存。
		- 失败，返回 (void *) -1
*/
```

## 2.3 `shmdt()`：把共享内存从当前进程身上“摘下来”

当你不用这块共享内存时，可以调用 `shmdt()`。

它的作用不是删除共享内存，而是：让“当前进程”不再和这块共享内存关联。

调用之后：

- 这个进程不能再通过原来的地址访问那块共享内存
- 但这块共享内存本身可能还在
- 其他进程如果还连着，照样能继续用

```c
#include <sys/ipc.h>
#include <sys/shm.h>

int shmdt(const void *shmaddr);
/*
	- shmaddr：要分离的共享内存起始地址。这个地址一般就是 shmat 成功后返回的地址。
	- 返回值：
		- 成功，返回0。
		- 失败，返回-1。
*/
```

## 2.4 `shmctl()`：对共享内存做控制，最常见是“删除”

`shmctl()`：对共享内存做控制，最常见是“删除”，最常见的一种：**删除共享内存段**。

**它不是立刻一定消失**，如果还有进程附加在这块共享内存上，系统不会马上把它清掉。通常是等到所有进程都 `shmdt()` 了，或者都结束了，这块内存才真正销毁。

```c
#include <sys/ipc.h>
#include <sys/shm.h>

int shmctl(int shmid, int cmd, struct shmid_ds *buf);
/*
	- shmid：共享内存标识符
	- cmd：控制命令
		- IPC_RMID：删除共享内存段
		- IPC_STAT：获取共享内存属性，放到 buf 中
		- IPC_SET：设置共享内存属性
	- buf：用来存放或提供共享内存属性信息
		- 如果 cmd 是 IPC_RMID，通常这个参数可以写 NULL。
	- 返回值
		- 成功，返回0。
		- 失败，返回-1。
*/
```
