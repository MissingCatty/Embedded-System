出现死锁的几种情况：

1. 同一个线程重复上锁
2. 忘记解锁
3. 多线程多锁，抢占资源

对于3，下图所示：

A已经对资源1持有锁，B已经对资源2持有锁，A在等B释放资源2，B在等A释放资源1

![image-20260323204808137](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2F40ad4a5eaf1a97266460eeb9f8090a53.png)

```c
// 两个互斥锁（共享资源）
pthread_mutex_t mutex1;
pthread_mutex_t mutex2;
```

线程A

```c
// 线程A：先锁资源1，再请求资源2
void *threadA(void *arg)
{
    /******* 对资源1上锁 *******/
    pthread_mutex_lock(&mutex1);
    do_something_to_resource1();

    /******* 等待资源2 *******/
    pthread_mutex_lock(&mutex2);  // ❗阻塞在这里

    pthread_mutex_unlock(&mutex2);
    pthread_mutex_unlock(&mutex1);

    return NULL;
}
```

线程B

```c
// 线程B：先锁资源2，再请求资源1
void *threadB(void *arg)
{
    /******* 对资源2上锁 *******/
    pthread_mutex_lock(&mutex2);
    
    /******* 等待资源1 *******/
    pthread_mutex_lock(&mutex1);  // ❗阻塞在这里

    pthread_mutex_unlock(&mutex1);
    pthread_mutex_unlock(&mutex2);

    return NULL;
}
```

这种情况如何解决呢？统一加锁顺序

规定：所有线程必须按相同顺序获取锁

```c

// A
lock(mutex1);
lock(mutex2);

// B
lock(mutex2);
lock(mutex1);

/****** 改成 ******/
// 所有线程都这样写
lock(mutex1);
lock(mutex2);

// 使用资源
unlock(&mutex2);
unlock(&mutex1);
```
