# 1.概念

- 进程：操作系统并发执行，是分配资源的基本单位
- 线程：一个进程并发执行多任务，是系统调度的基本单位

同一进程中的线程通常共享这些东西：

- **共享的：**

  - 进程的**地址空间**

  - **代码段**（程序代码）

  - **全局变量**、**静态变量**

  - **堆**上的内存

  - **打开的文件描述符**

  - **当前工作目录**

  - **信号处理方式**

  - 进程拥有的大部分内核资源

- **每个线程自己独有的：**

  - **程序计数器**（执行到哪条指令）

  - **寄存器现场**

  - **栈**
  - 信号掩码（屏蔽哪些信号）

  - 线程自己的调度状态、线程 ID

**线程的本质就是**：从进程代码中的某个入口点开始执行，但它的本质不是“一段代码”，而是“一条执行流”。

---

**线程别名**：LWP（Light Weight Process），轻量级的进程，在Linux环境下线程的本质仍是进程。

**如何查看指定线程的LWP号？**

```bash
ps -Lf pid
```

# 2.进程和线程的区别

- 进程
  1. 如果只用进程，由于进程之间是独立的，数据共享非常麻烦
  2. 调用fork创建子进程，开销太大，需要复制虚拟地址空间
- 线程
  1. 同一进程下的线程之间共享数据更方便更快速
  2. 创建线程的速度比创建进程快10倍，线程间共享虚拟地址空间，不需要复制内存
