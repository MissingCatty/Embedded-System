# 1.概念

不同进程不能直接访问另一个进程的资源。

但是，进程之间不是完全孤立的，进程之间的数据传输就涉及到进程间的通信。

进程之间通信的目的包含：

- 数据传输
- 事件通知
- 资源共享（需要内核提供互斥和同步）

# 2.匿名管道

## 2.1 引例

统计一个目录文件中文件数量的命令如下：

```bash
ls | wc -l
```

- ls：列出目录中所有文件
- wc -l：统计输入内容的行数

这两个命令中间的符号`|`是管道符，ls出来的结果（数据）通过管道送给后一个命令作为输入。

![image-20260317095932002](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2F262247037c9cd938424029f2eb228966.png)

**shell创建了两个进程来执行`ls`和`wc`**。

![image-20260317100618644](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2Fa071879f8e3790548a6e0dfd2f8a73c6.png)

ls命令默认输出的目标是1（stdout），使用管道符后，会输出到管道的输入端。

wc默认输入来源是0（stdin），使用管道符后，来源切换为管道的输出端。

## 2.2 管道的原理

管道的实质：**一个在内核内存维护的缓冲区**，可读可写。

匿名管道和有名管道的区别就是：

- 匿名管道无文件实体
- 有名管道有文件实体，但不存储数据

可以把管道视为一个FIFO的队列，数据入队的顺序和出队的顺序一致。

一端输入，一端输出，半双工（可单向同时读写）

---

**管道如果满了怎么办？**

- 默认情况：写进程阻塞，直到读进程从管道中取出数据。
- 非阻塞模式：写进程不阻塞，直接返回`-1`，并设置`errno = EAGAIN`

---

管道持有两个文件描述符。文件描述符可以用于指定读/写的位置。

- 普通文件：只要一个fd即可，因为读和写公用一个偏移量就行
- 管道：需要明确区分读的位置和写的位置，所以要两个

## 2.3 匿名管道

**匿名管道**只能在有血缘关系进程的两个进程中使用

> 血缘关系：
>
> - 父进程与子进程
> - 两个兄弟进程

解释一下为什么：管道的读写本质就是找到输入和输出的位置，而这个位置进程是用文件描述符fd来指示的。有血缘关系的两个进程，他们的文件描述符表有一部分是相同的（因为都继承自父进程，父进程fork的时候会拷贝自己的fd表给子进程）。有了fd，且fd指向的文件位置是一样的，这样就能找到文件输入输出的位置。

但无血缘关系的两个进程的fd表不同，即使相同的fd，指向的文件位置也不相同，所以无法产生关联。

那有没有什么办法使两个无关进程同时指向一个管道的输入输出端呢？这就是有名管道的功能。

### 2.3.1 父子进程通过匿名管道通信

**管道创建**

```c
#include <unistd.h>

int pipe(int pipefd[2]);	// pipefd: -O, pipe[0]管道读端, pipe[1]管道写端
```

- 返回值：
  - 成功，0
  - 失败，-1

---

**写数据：`write()`**

将数据存入管道的缓冲区。

```c
ssize_t write(int fd, const void *buf, size_t count);
```

**参数解释**

- **`fd`**: 文件描述符。在管道中应传入 **`pipefd[1]`**。
- **`buf`**: 指向要发送数据的缓冲区。
- **`count`**: 打算写入的字节数。

**返回值**

- **正整数**: 实际写入管道的字节数。
- **`-1`**: 写入失败。
  - 如果读端 `pipefd[0]` 已经全部关闭，`write` 会失败，进程收到 **SIGPIPE** 信号。
  - 如果 `errno` 为 `EAGAIN`，说明管道已满（非阻塞模式下）。

**特性**：

- 如果管道缓冲区已满，`write` 会**阻塞**，直到读端取走数据。
- 如果所有读端文件描述符都被关闭，写入会触发 **SIGPIPE** 信号，导致进程异常终止（通常需要捕获或忽略该信号）。

---

**读数据：`read()`**

从管道缓冲区读取数据。

```c
ssize_t read(int fd, void *buf, size_t count);
```

**参数解释**

- **`fd`**: 文件描述符。在管道中应传入 **`pipefd[0]`**。
- **`buf`**: 存储读取数据的本地缓冲区。
- **`count`**: 期望读取的最大字节数。

**返回值**

- **正整数**: 实际读到的字节数。
- **`0`**: **非常重要**。表示管道的写端 **`pipefd[1]`** 已经全部关闭，且管道内已无残留数据，相当于读到了文件末尾（EOF）。
- **`-1`**: 读取失败。

**特性**：

- 如果管道为空且写端未关闭，`read` 会**阻塞**等待数据。
- 如果写端全部关闭，`read` 将返回 **0**（代表读到了文件末尾 EOF）。

---

**案例：子进程给父进程发数据，父进程读取数据并输出到终端。**
