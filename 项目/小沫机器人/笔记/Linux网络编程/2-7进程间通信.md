# 1.概念

不同进程不能直接访问另一个进程的资源。

但是，进程之间不是完全孤立的，进程之间的数据传输就涉及到进程间的通信。

进程之间通信的目的包含：

- 数据传输
- 事件通知
- 资源共享（需要内核提供互斥和同步）

# 2.匿名管道

## 2.1 引例

统计一个目录文件中文件数量的命令如下：

```bash
ls | wc -l
```

- ls：列出目录中所有文件
- wc -l：统计输入内容的行数

这两个命令中间的符号`|`是管道符，ls出来的结果（数据）通过管道送给后一个命令作为输入。

![image-20260317095932002](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2F262247037c9cd938424029f2eb228966.png)

**shell创建了两个进程来执行`ls`和`wc`**。

![image-20260317100618644](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2Fa071879f8e3790548a6e0dfd2f8a73c6.png)

ls命令默认输出的目标是1（stdout），使用管道符后，会输出到管道的输入端。

wc默认输入来源是0（stdin），使用管道符后，来源切换为管道的输出端。

## 2.2 管道的原理

管道的实质：**一个在内核内存维护的缓冲区**，可读可写。

匿名管道和有名管道的区别就是：

- 匿名管道无文件实体
- 有名管道有文件实体，但不存储数据

可以把管道视为一个FIFO的队列，数据入队的顺序和出队的顺序一致。

一端输入，一端输出，半双工（可单向同时读写）

---

**管道如果满了怎么办？**

- 默认情况：写进程阻塞，直到读进程从管道中取出数据。
- 非阻塞模式：写进程不阻塞，直接返回`-1`，并设置`errno = EAGAIN`

---

管道持有两个文件描述符。文件描述符可以用于指定读/写的位置。

- 普通文件：只要一个fd即可，因为读和写公用一个偏移量就行
- 管道：需要明确区分读的位置和写的位置，所以要两个

## 2.3 匿名管道pipe

**匿名管道**只能在有血缘关系进程的两个进程中使用

> 血缘关系：
>
> - 父进程与子进程
> - 两个兄弟进程

解释一下为什么：管道的读写本质就是找到输入和输出的位置，而这个位置进程是用文件描述符fd来指示的。有血缘关系的两个进程，他们的文件描述符表有一部分是相同的（因为都继承自父进程，父进程fork的时候会拷贝自己的fd表给子进程）。有了fd，且fd指向的文件位置是一样的，这样就能找到文件输入输出的位置。

但无血缘关系的两个进程的fd表不同，即使相同的fd，指向的文件位置也不相同，所以无法产生关联。

那有没有什么办法使两个无关进程同时指向一个管道的输入输出端呢？这就是有名管道的功能。

### 2.3.1 父子进程通过匿名管道通信

**管道创建**

```c
#include <unistd.h>

int pipe(int pipefd[2]);	// pipefd: -O, pipe[0]管道读端, pipe[1]管道写端
```

- 返回值：
  - 成功，0
  - 失败，-1

---

**写数据：`write()`**

将数据存入管道的缓冲区。

```c
ssize_t write(int fd, const void *buf, size_t count);
```

**参数解释**

- **`fd`**: 文件描述符。在管道中应传入 **`pipefd[1]`**。
- **`buf`**: 指向要发送数据的缓冲区。
- **`count`**: 打算写入的字节数。

**返回值**

- **正整数**: 实际写入管道的字节数。
- **`-1`**: 写入失败。
  - 如果读端 `pipefd[0]` 已经全部关闭，`write` 会失败，进程收到 **SIGPIPE** 信号。
  - 如果 `errno` 为 `EAGAIN`，说明管道已满（非阻塞模式下）。

**特性**：

- 如果管道缓冲区已满，`write` 会**阻塞**，直到读端取走数据。
- 如果所有读端文件描述符都被关闭，写入会触发 **SIGPIPE** 信号，导致进程异常终止（通常需要捕获或忽略该信号）。

---

**读数据：`read()`**

从管道缓冲区读取数据。

```c
ssize_t read(int fd, void *buf, size_t count);
```

**参数解释**

- **`fd`**: 文件描述符。在管道中应传入 **`pipefd[0]`**。
- **`buf`**: 存储读取数据的本地缓冲区。
- **`count`**: 期望读取的最大字节数。

**返回值**

- **正整数**: 实际读到的字节数。
- **`0`**: **非常重要**。表示管道的写端 **`pipefd[1]`** 已经全部关闭，且管道内已无残留数据，相当于读到了文件末尾（EOF）。
- **`-1`**: 读取失败。

**特性**：

- 如果管道为空且写端未关闭，`read` 会**阻塞**等待数据。
- 如果写端全部关闭，`read` 将返回 **0**（代表读到了文件末尾 EOF）。

---

**案例：子进程给父进程发数据，父进程读取数据并输出到终端。**

- 子进程每隔1s往管道里写
- 父进程循环读取

```c
#include <unistd.h>
#include <stdio.h>
#include <sys/types.h>
#include <stdlib.h>
#include <sys/wait.h>
#include <stdbool.h>

int main(void)
{
    // 在fork之前创建管道
    int pipefd[2];
    if (pipe(pipefd) == 0)
    {
        printf("pipe created successfully, pipefd = %d, %d\n", pipefd[0], pipefd[1]);
    }
    else
    {
        perror("pipe created error");
        exit(1);
    }

    pid_t pid = fork();
    if (pid < 0)
    {
        perror("child process created error");
        exit(1);
    }
    if (pid > 0)
    {
        // 关闭自己不需要的那一端
        close(pipefd[1]);
        printf("child process created, pid %d.\n", pid);
        // 父进程循环读取管道
        char command = '\0';
        char buf[256];
        while (1)
        {
            int count = read(pipefd[0], buf, sizeof(buf));
            if (count < 0)
            {
                perror("parent process read error:");
                exit(1);
            }
            if (count == 0)
            {
                printf("pipe write end closed, and no data in pipe.\n");
                break;
            }
            if (count > 0)
            {
                // 往终端输出
                write(1, buf, count);
            }
        }
        close(pipefd[0]);
        wait(NULL);
    }
    if (pid == 0)
    {
        // 关闭自己不需要的那一端
        close(pipefd[0]);
        // 子进程每隔一秒发送字符串到管道里
        char string[] = "this is a test string.\n";
        int i = 0;
        while ((i++) < 5)
        {
            write(pipefd[1], string, sizeof(string));
            sleep(1);
        }
        close(pipefd[1]);
    }
}
```

需要注意的是：

1. 父子进程在fork之后需要关闭自己不用的一端，否则父进程调用read函数无法返回0一直阻塞。

   > read函数返回0的条件：
   >
   > 1. 管道为空
   > 2. 管道写端关闭
   >
   > 如果写端不关闭，read函数会一直阻塞，因为会以为还有可能管道里会写。

2. 同样的子进程如果不关闭读端，等到父进程关闭管道读端后，子进程对管道的读端还开着，此时如果子进程再往管道里写还是允许的，则不会触发SIGPIPE信号，不符合安全规定。

   > 子进程一直写，但没有对应的进程来取，直到管道满了。满了之后子进程write会一直被阻塞。

### 2.3.2 将文件描述符设置为非阻塞

```c
int flags = fcntl(fd, F_GETFL);	// 获取文件状态
flags |= O_NONBLOCK;			// 添加非阻塞
fcntl(fd, F_SETFL, flags);		// 重新设置
```

设置为非阻塞后：

- 写端：当管道满时，写进程尝试写入数据时，不会阻塞，而是直接返回-1，然后设置errno为`EAGAIN`。告诉进程你先干别，之后再来试一试。
- 读端：读进程尝试读取一个空的管道时，不会阻塞，立即返回-1，并设置errno为`EAGAIN`。告诉读进程，这里没东西，你先干别的。

## 2.4 有名管道FIFO

### 2.4.1 概念

有名管道的本质也是内核中的内存，只不过我们给他按照文件名的方式起了个名字，之后方便各个进程通过文件名来访问他。

> - 普通文件名：指向的是硬盘上的地址
> - 有名管道名：指向的内存中的地址
>
> （因为linux中一切皆文件，都可以按照文件名去访问数据，在linux中文件名既可以指向是硬盘，也可以是内存，也可以设备寄存器地址）

匿名管道为什么叫匿名，是因为他是一段内存缓冲区，不是一个具有名字的文件，进程无法通过open去获取其文件描述符（因为open有个参数是文件路径），只能通过fork的方式继承文件描述符，所以只对有血缘关系的两个进程适用。

### 2.4.2 使用

- 通过命令创建

  ```
  mkfifo 管道名
  ```

- 通过函数创建

  ```c
  #include <sys/types.h>
  #include <sys/stat.h>
  
  int mkfifo(const char *pathname, mode_t mode);
  ```

  - mode：文件的权限，和open的mode一样
  - 返回值
    - 成功，返回0
    - 失败，返回-1，并设置errno

使用 mkfifo 创建了一个 FIFO，就可以使用 open 打开它，常见的文件 I/O 函数都可用于 fifo。如：close、read、write、unlink 等。

FIFO 严格遵循先进先出（First in First out），

- **读**总是**从开始处**返回数据
- **写**则把数据添加**到末尾**

**不支持诸如 lseek() 等文件定位操作**。

---

**如何区分fifo的读端和写端呢？**

- 打开fifo时

  - 指定标志位为`O_RDONLY`，表示以只读的方式打开，写端
  - 指定标志位为`O_WRONLY`，表示以只写的方式打开，读端

  > 使用`O_RDWR`的后果是未定义的，虽然不会有什么影响，但如果一次在只读模式下对fifo写了，不会报错，也就漏掉了检查。

### 2.4.3 注意事项

1. 一个为只读的进程**调用open函数**打开管道会阻塞，直到另一个为只写的进程打开管道，反之亦然

   > 管道会保证通信的意义，单方打开管道没用

2. 读端如果发现管道里没有数据，调用read函数后

   1. 写端未关闭：阻塞
   2. 写端关闭：返回0

3. 写端如果发现没有读端open这个fifo，此时调用write函数，会异常终止，并收到一个`SIGPIPE`信号

### 2.4.4 案例

**案例1：一个进程写，一个进程读到终端**

- 创建fifo

  ```c
  #include <sys/types.h>
  #include <sys/stat.h>
  #include <stdio.h>
  
  int main(void) {
      int res = mkfifo("./fifo-pipe", 0664); // 0664: 110 110 100
      if(res == -1) {
          perror("mkfifo error");
          return 1;
      }
  }
  ```

- 写进程

  ```c
  #include <sys/types.h>
  #include <sys/stat.h>
  #include <stdio.h>
  #include <stdlib.h>
  #include <fcntl.h>
  #include <unistd.h>
  
  int main(void)
  {
      int fd = open("./fifo-pipe", O_WRONLY);
      if(fd == -1) {
          perror("open fifo error");
          exit(1);
      }
  
      char content[] = "hello, world.\n";
      int i = 0;
      while ((i++) < 5)
      {
          write(fd, content, sizeof(content));
          sleep(1);
      }
      close(fd);
  }
  ```

- 读进程

  ```c
  #include <sys/types.h>
  #include <sys/stat.h>
  #include <stdio.h>
  #include <stdlib.h>
  #include <fcntl.h>
  #include <unistd.h>
  
  int main(void)
  {
      int fd = open("./fifo-pipe", O_RDONLY);
      if (fd == -1)
      {
          perror("open fifo error");
          exit(1);
      }
  
      char buf[256];
      while (1)
      {
          int count = read(fd, buf, sizeof(buf));
          if (count > 0)
              write(1, buf, count);
          else
              break;
      }
      close(fd);
  }
  ```
