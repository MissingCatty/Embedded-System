线程属性操作函数是用来操作线程属性`pthread_attr_t`的，通常配合 `pthread_create` 一起用。

# 1.`pthread_attr_init`：初始化一个线程属性变量

```c
int pthread_attr_init(pthread_attr_t *attr);
/*
	- 返回值：
		- 0：成功
		- 非0：失败
*/
```

你定义了一个 `pthread_attr_t` 变量后，不能直接乱用，通常先初始化：

```c
pthread_attr_t attr;
pthread_attr_init(&attr);
```

# 2.`pthread_attr_destroy`：销毁线程属性对象

```c
int pthread_attr_destroy(pthread_attr_t *attr);
/*
	- 返回值：
		- 0：成功
		- 非0：失败	
*/
```

当你这个属性对象用完了，就调用它释放相关资源：

```c
pthread_attr_destroy(&attr);
```

# 3.`pthread_attr_getdetachstate`：**获取线程属性中的分离状态**

```c
int pthread_attr_getdetachstate(const pthread_attr_t *attr, int *detachstate);
```

线程的分离状态有两种：

- `PTHREAD_CREATE_JOINABLE`
   可连接，默认值。线程结束后可以 `pthread_join`
- `PTHREAD_CREATE_DETACHED`
   分离状态。线程结束后系统自动回收，不能 `pthread_join`

```c
int state;
pthread_attr_getdetachstate(&attr, &state);
```

如果 `state` 是：

- `PTHREAD_CREATE_JOINABLE`：表示可连接
- `PTHREAD_CREATE_DETACHED`：表示分离

返回值：

- `0`：成功
- 非 `0`：失败

# 4.`pthread_attr_setdetachstate`：设置线程属性中的分离状态

```c
int pthread_attr_setdetachstate(pthread_attr_t *attr, int detachstate);
```

把线程设置成分离线程：

```c
pthread_attr_setdetachstate(&attr, PTHREAD_CREATE_DETACHED);
```

这样以后用这个属性创建出来的线程，就是分离线程。

返回值：

- `0`：成功
- 非 `0`：失败

---

注意：`pthread_attr_setdetachstate`设置线程分离状态是在线程创建的时候，而`pthread_detach`是之后在调用该函数的时候设置分离状态。
