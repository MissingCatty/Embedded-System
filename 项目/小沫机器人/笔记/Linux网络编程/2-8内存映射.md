# 1.概念

内存映射也是一种共享数据的方式。

他将**磁盘文件映射到内存**，使用户通过修改内存，修改会自动同步到文件里。

![image-20260318161235194](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2F607dccb68ed2a6768294e370a9900dd3.png)

由于内存映射是双向映射，进程A中内存修改，变化会反映到硬盘上，进程B中与硬盘关联的内存也会随之修改，等于说进程A中的内存和进程B中的内存通过硬盘这个媒介实现了同步，可以用来共享数据。

> 注意：内存映射存在并发问题，内存的修改不会立刻写入磁盘，而是**缓存和延迟写回**。
>
> 这就导致某一时间段，进程A修改内存后，硬盘上的数据没来得及同步，或者A→硬盘完成了，但硬盘→B的数据没有同步完成，从而进程A和进程B自己认为的数据和实际数据不一样（A认为B已经得到修改后的值了）。

# 2.用法

```c
#include <sys/mman.h>

void *mmap(void *addr, size_t length, int prot, int flags,
           int fd, off_t offset);
/*
	- 功能：映射一个文件到内存中
	- 参数：
		- addr：内存中的首地址，传递为NULL由内核指定
		- length：要映射的数据字节数，不能为0，建议使用文件的长度。如果没有达到一个页的大小，会默认取页的整数倍。
		- prot：对内存映射区的操作权限
			- PROT_EXEC：可执行
			- PROT_READ：可读
			- PROT_WRITE：可写
			- PROT_NONE：无
			一般常用的有：PROT_READ、PROT_READ|PROT_WRITE
		- flags：
			- MAP_SHARED：映射区数据会自动和磁盘文件进行同步（进程间通信必选）
			- MAP_PRIVATE：不同步，内存修改后，原文件不会修改，但会创建一个新的文件
		- fd：需要操作的文件描述符
			- 文件的大小不能为0
			- prot权限必须被open包含（prot映射的要求根据文件打开的要求来）
				open: 只读/读写			prot: PROT_READ
				open: 只写/读写			prot: PROT_WRITE
		- offset：文件映射部分的偏移量（图中所示），规定必须是4K的整数倍，0表示不偏移。
	- 返回值：指向映射内存的指针
		- 失败，返回MAP_FAILED
*/

int munmap(void *addr, size_t length);
/*
	- 功能：释放内存映射
	- 参数：
		- addr：内存首地址
		- length：字节数
*/
```

# 3.案例

案例1：进程A通过内存映射写入数据，进程B通过内存映射读取数据。

**读进程**

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <fcntl.h>
#include <sys/mman.h>
#include <unistd.h>
#include <sys/stat.h>
#include <sys/types.h>

#define FILE_NAME "./share.dat"
#define FILE_SIZE 4096

int main(void)
{
    // 打开文件
    int fd = open(FILE_NAME, O_RDWR, 0666);
    if (fd == -1)
    {
        perror("open shared file error");
        exit(1);
    }
    // 创建内存映射
    void *ptr = mmap(NULL, FILE_SIZE, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (ptr == MAP_FAILED)
    {
        perror("mmap called error");
        exit(1);
    }
    // 从内存输出
    write(1, ptr, FILE_SIZE);
    // 善后处理
    munmap(ptr, FILE_SIZE);
    close(fd);
}
```

**写进程**

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <fcntl.h>
#include <sys/mman.h>
#include <unistd.h>
#include <sys/stat.h>
#include <sys/types.h>

#define FILE_NAME "./share.dat"
#define FILE_SIZE 4096

int main(void)
{
    // 创建文件
    int fd = open(FILE_NAME, O_RDWR | O_CREAT, 0666);
    if (fd == -1)
    {
        perror("open shared file error");
        exit(1);
    }
    // 设置文件大小为4096
    int ret = ftruncate(fd, FILE_SIZE);
    if (ret == -1)
    {
        perror("truncate shared file error");
        close(fd);
        exit(1);
    }
    // 创建内存映射
    void *ptr = mmap(NULL, FILE_SIZE, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0);
    if (ptr == MAP_FAILED)
    {
        perror("mmap called error");
        exit(1);
    }
    // 获取输入内容
    printf("写进程准备写：");
    fgets(ptr, FILE_SIZE, stdin);
    // 善后处理
    munmap(ptr, FILE_SIZE);
    close(fd);
}
```

# 4.注意事项

1. 对mmap返回的内存指针使用（ptr++），之后munmap的时候能不能释放成功？

   不能，必须是映射内存的首地址

2. 如果文件偏移量为 1000 会怎样？

   失败，因为规定offset是4K的整数倍

3. `mmap` 什么情况下会调用失败？

   - prot和open模式冲突
   - offset不是4K的整数倍

4. 可以 `open` 时用 `O_CREAT` 创建新文件来建立映射区吗？

   可以，但光open还不够，还没有扩充文件大小，需要先使用`ftruncate(fd, 4096);`
