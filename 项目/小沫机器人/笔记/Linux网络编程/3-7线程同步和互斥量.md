# 1.案例

使用多线程，实现：有三个窗口，卖100张票。

```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

/* 代码被三个线程共享，但栈空间各自有一份 */
static int tickets = 100;
void *sellticket(void *arg)
{
    while (tickets > 0)
    {
        printf("thread %ld sell ticket %d\n", pthread_self(), tickets--);
        sleep(1);
    }
}

int main(void)
{
    /* 创建三个子线程 */
    pthread_t tid1, tid2, tid3;
    pthread_create(&tid1, NULL, sellticket, NULL);
    pthread_create(&tid2, NULL, sellticket, NULL);
    pthread_create(&tid3, NULL, sellticket, NULL);

    /* 回收子线程 */
    pthread_join(tid1, NULL);
    pthread_join(tid2, NULL);
    pthread_join(tid3, NULL);

    return 0;
}
```

这样写程序有几个问题：

1. `while(tickets>0)`和`print("xxx",...tickets)`不是原子性的，如果线程A在此刻执行`while(tickets>0)`判断发现`tickets=1`，进入循环取票，还没来得及执行`printf(...)`，就被线程B率先取走了，此时`tickets=0`，A照常取票，打印：`sell ticket 0`，肯定是错的。
2. `print("xxx",...tickets)`和`tickets--`也不是原子性的，这就导致如果线程A`print`之后，还没来得及将票数减去，线程B就紧跟着`print(tickets)`，结果就是两个线程

> 解决方法：保证tickets的原子性访问

# 2.互斥量

互斥量（mutex）是“所有权锁”，**只能谁加锁谁解锁**

## 2.1 定义互斥量

定义一个互斥锁变量

```c
pthread_mutex_t
```

## 2.2 互斥量初始化

### 2.2.1 静态初始化

```c
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
/*
    - 功能：使用默认属性初始化互斥量
    - 特点：不需要调用 pthread_mutex_init
*/
```

### 2.2.2 动态初始化

```c
int pthread_mutex_init(
    pthread_mutex_t *restrict mutex,
    const pthread_mutexattr_t *restrict attr
);
/*
	- 功能：初始化一个互斥锁
	- mutex：要初始化的互斥量
	- attr：属性（一般传 NULL，表示默认）
	- 返回值：
		- 成功：0
		- 失败：错误码（不是 -1）
*/
```

## 2.3 上锁

### 2.3.1 阻塞式

```c
int pthread_mutex_lock(pthread_mutex_t *mutex);
/*
    - 功能：对互斥量加锁（阻塞）
    - mutex：要加锁的互斥量
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 如果锁未被占用 → 立即加锁
- 如果锁已被占用 → 当前**线程阻塞等待**

### 2.3.2 非阻塞式

```c
int pthread_mutex_trylock(pthread_mutex_t *mutex);
/*
    - 功能：尝试加锁（不阻塞）
    - mutex：要加锁的互斥量
    - 返回值：
        - 成功：0
        - 失败：EBUSY（锁已被占用）
*/
```

- 如果锁空闲 → 加锁成功
- 如果锁被占用 → 立即返回，不等待

## 2.4 解锁

```c
int pthread_mutex_unlock(pthread_mutex_t *mutex);
/*
    - 功能：解锁互斥量
    - mutex：要解锁的互斥量
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 必须由加锁的线程来解锁，否则属于未定义行为

## 2.5 销毁互斥量

```c
int pthread_mutex_destroy(pthread_mutex_t *mutex);
/*
    - 功能：销毁互斥量
    - mutex：要销毁的互斥量
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 必须在互斥量未加锁时销毁，不能有线程正在使用

## 2.6 注意事项

1. lock → unlock必须成对出现，如果忘记unlock会导致死锁（其他线程拿不到锁，一直等）

2. 不要重复加锁同一 mutex

   ```c
   pthread_mutex_lock(&mutex); // 第一次，加锁成功
   
   pthread_mutex_lock(&mutex); // 由于第一次上锁未释放，第二次一直等待
   ```
