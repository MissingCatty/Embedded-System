# 2.两种高效的事件处理模式

服务器通常需要处理三类事件：

- IO事件
- 信号事件
- 定时事件

有两种高效的处理模式：

- Reactor模式
- Proactor模式

其中，**同步IO模型**通常实现于**Reactor模式**；异步IO模型通常实现与Proactor模式

## 2.1 Reactor模式

分为主线程和子线程，

- 主线程：负责监听fd上是否有事件发生，如果发生就通知工作线程（子线程），除此之外，主线程不处理任何其他的事
  - 如何通知？：将事件放入一个队列，挨个交给子线程处理
- 子线程
  - 接收连接请求
  - 收发数据

**Reactor的工作流程为**：

1. 主线程把某个`cfd`附上`EPOLLIN`（读就绪事件）并放入epoll中
2. 主线程调用`epoll_wait`查看哪些`cfd`有数据可读
3. 主线程将这些`cfd`挨个挂到一个队列上
4. 线程池释放多个工作线程，挨个去队列上处理这些`cfd`

总结而言：**主线程负责事件监听与分发，线程池负责具体处理**。
