# 2.两种高效的事件处理模式

服务器通常需要处理三类事件：

- IO事件
- 信号事件
- 定时事件

有两种高效的处理模式：

- Reactor模式
- Proactor模式

其中，**同步IO模型**通常实现于**Reactor模式**；异步IO模型通常实现与Proactor模式

## 2.1 Reactor模式

**Reactor：“来了事件，应用自己去处理 I/O”。**

1. 事件来了
2. 通知线程
3. **线程自己做 I/O 读写**
4. 再处理数据

**Reactor的工作流程为**：

1. 主线程把某个`cfd`附上`EPOLLIN`（读就绪事件）并放入epoll中
2. 主线程调用`epoll_wait`查看哪些`cfd`有数据可读
3. 主线程将这些`cfd`挨个挂到一个队列上
4. 线程池释放多个工作线程，挨个去队列上处理这些`cfd`

总结而言：**主线程负责事件监听与分发，线程池负责具体处理**。

## 2.2 Proactor模式

**Proactor：“来了事件时，I/O 已经做完了，应用只处理结果”。**

1. 先提交异步 I/O
2. I/O 完成了
3. 通知线程
4. **线程直接处理已经读写完成的数据**

## 2.3 两种模式的区别

- Reactor模式：主线程往队列上挂cfd（或者包含cfd的数据结构）
- Proactor模式：主线程往队列上挂IO接收的数据
