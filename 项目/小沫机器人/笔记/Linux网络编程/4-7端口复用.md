# 1.概念

当**服务器主动关闭**连接（服务器调了 `close` 或 `shutdown`）并发送了最后一个 `FIN` 后，连接并不会立刻消失，而是进入 `TIME_WAIT` 状态，持续时间通常是 **2MSL**（在 Linux 上通常是 **60** 秒）。

- **现状**：此时，该端口被内核锁定，处于“半死不活”状态。

- **后果**：如果你此时想重启服务器，再次调用 `bind` 绑定同一个端口，内核会无情地报错：

  > `bind error: Address already in use`

![image-20211123124611335](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F03%2Feb399a4538ea7e3597b5af269848923b.png)

**端口复用**的核心本质是：**告诉内核，即便这个端口正处于 `TIME_WAIT` 状态，也允许我的新进程强制“抢占”并绑定它。**

> 本质就是允许被锁定端口可以再次被其他的socket绑定

---

**为什么要端口复用？**

如果你不设置端口复用，内核其实是在保护你。想象一下：

1. 旧连接还没死透，网络上还有一个“迟到”的数据包发给旧连接。
2. 如果你此时刚好起了一个新进程，用了同一个端口。
3. **冲突**：这个迟到的包会莫名其妙地跑进你的新程序里，导致协议解析出错。

端口复用的场景如下：

1. 一个端口发送完最后一个ACK后，进入长达60s的`TIME_WAIT`状态。对于开发人员来说，如果你刚修改了服务器代码，关掉进程想立刻重启，内核会直接糊你一脸报错：`bind error: Address already in use`。
2. 一个高并发服务器因为异常崩溃了，成千上万个与客户端的连接瞬间被切断，服务器作为“主动断开方”，会导致该端口瞬间产生大量的 `TIME_WAIT` 状态连接。**如果不开启端口复用**：新进程会启动失败，服务将面临长达 1-2 分钟的绝对瘫痪（真空期），这对于商业应用是不可接受的。

# 2.用法

```c
#include <sys/types.h>
#include <sys/socket.h>

int setsockopt(int sockfd, int level, int optname, const void *optval, socklen_t optlen);
/*
    - 功能：设置一个套接字的选项属性
    - sockfd：要操作的文件描述符（套接字）
    - level：选项定义的层次
        - SOL_SOCKET：通用套接字选项
        - IPPROTO_IP：IP 层选项
        - IPPROTO_TCP：TCP 层选项
    - optname：需要设置的选项名称（如 SO_REUSEADDR, SO_RCVBUF 等）
    - optval：指向存放选项待设置值的缓冲区
    - optlen：optval 缓冲区的长度
    - 返回值：
        - 成功：0
        - 失败：-1，并设置 errno
*/
```

| **参数名**  | **填什么**      | **含义说明**                                                 |
| ----------- | --------------- | ------------------------------------------------------------ |
| **sockfd**  | `listen_fd`     | 你刚刚通过 `socket()` 创建的那个监听文件描述符。             |
| **level**   | `SOL_SOCKET`    | 选项的级别。`SOL` 代表 Socket Level，意思是我们要设置的是通用的套接字层属性，而不是特定的 TCP 或 IP 层属性。 |
| **optname** | `SO_REUSEADDR`  | 你要设置的具体选项名（Socket Option Reuse Address）。        |
| **optval**  | `&opt` (值为 1) | 一个指向整型变量的指针。在这里，`1` 代表“开启/True”，`0` 代表“关闭/False”。 |
| **optlen**  | `sizeof(opt)`   | 上一个参数 `optval` 变量占用内存的大小。                     |

```c
// 在 bind 之前执行以下代码
int opt = 1;
int ret = setsockopt(lfd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
/*
    - 功能：允许套接字强制绑定到一个正在被占用的端口上
    - 参数说明：
        - lfd：监听的套接字文件描述符
        - SOL_SOCKET：表示配置在通用套接字层
        - SO_REUSEADDR：具体的选项名称，表示端口复用
        - &opt：传入 1 表示启用该选项
        - sizeof(opt)：参数缓冲区的大小
    - 使用时机：
        - 必须在 bind() 调用之前设置
    - 返回值：
        - 成功：0
        - 失败：-1，并设置 errno
*/
```

注意：

1. 需要**在绑定前执行端口复用**
2. 同样的，如果上一次没有设置端口复用，服务器重启后，这次把代码加上去之后再次运行不能立即生效
