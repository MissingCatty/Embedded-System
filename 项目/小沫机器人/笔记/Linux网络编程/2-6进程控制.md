# 1.进程退出

```c
#include <stdlib.h>

void exit(int status);
```

- status：进程退出时的状态信息，父进程回收子进程资源的时候可以获取

```c
#include <unistd.h>

void _exit(int status);
```

二者有何区别？

- `exit`是标准C库的进程退出函数，底层除了调用`_exit`的退出函数外，还会做一些其他处理
  - 刷新IO缓冲区

    ```c
    #include <stdlib.h>
    #include <unistd.h>
    #include <stdio.h>
    
    int main(void)
    {
    
        printf("hello\n");
        printf("world.");
    
        exit(0);
        // _exit(0);
    }
    ```

    看一下这段程序，由于printf会将字符串写到IO缓冲区，遇到`\n`，会自动执行IO缓冲区刷新，将缓冲区的内容输出到终端上。所以无论执行`exit`还是`_exit`，终端上都会输出`hello\n`。

    而对于`_exit`，这个函数不是C标准库，在执行时不会刷新IO缓冲区，这就导致`world.`一直留在缓冲区里。

    ```bash

    root@lavm-pk9nq55hju:~/Linux/Lesson12# gcc exit.c -o exit
    root@lavm-pk9nq55hju:~/Linux/Lesson12# ./exit 
    hello
    world.root@lavm-pk9nq55hju:~/Linux/Lesson12#
    ```

    ```bash
    # _exit
    root@lavm-pk9nq55hju:~/Linux/Lesson12# gcc exit.c -o exit
    root@lavm-pk9nq55hju:~/Linux/Lesson12# ./exit 
    hello
    root@lavm-pk9nq55hju:~/Linux/Lesson12#
    ```

# 2.孤儿进程

- 概念：当父进程结束，子进程还在执行，子进程就被称为孤儿进程

每当出现一个孤儿进程的时候，内核就把孤儿进程的父进程设置为 init。而 init 进程会循环地 wait() 它的已经退出的子进程。这样，当一个孤儿进程凄凉地结束了其生命周期的时候，init 进程就出面处理它的一切善后工作。

# 3.僵尸进程

## 3.1 概念

子进程已经结束运行，但它的进程控制块（PCB）仍然保存在系统中，没有被父进程回收。

**僵尸进程 = 已经退出，但父进程没有调用 wait() 回收的子进程。**

> 子进程已经结束，但父进程没有调用 wait() 或 waitpid()。

僵尸进程产生的过程如下：

- 子进程调用exit终止进程

- 此时子进程释放了虚拟地址空间中的

  - 堆栈
  - 代码段
  - 打开的文件（文件描述符）

  但是自己无法删除pcb，需要由父进程删除。

- **父进程没有调用`wait/waitpid`函数**

  > 如果父进程执行：
  >
  > ```
  > wait(NULL);
  > ```
  >
  > 系统就会：
  >
  > 1. 把 B 的退出信息交给 A
  > 2. 删除 B 的 PCB
  > 3. B 完全消失

## 3.2 如何删除僵尸进程

僵尸进程无法使用`kill -9`删除，因为`kill`只能对正在运行的进程生效。

那具体如何删除僵尸进程呢？

1. 【方法一】父进程调用wait
2. 【方法二】终止父进程，因为父进程结束，子进程会被init接管，init会自动调用wait回收他

## 3.3 僵尸进程的危害

僵尸进程的危害来源于pcb未释放，由于pcb中包含了pid，所以主要问题会造成pid的占用。

# 4.wait函数

父进程可以调用wait函数**得到退出状态**，**同时彻底清除这个进程**。

```c
#include <sys/types.h>
#include <sys/wait.h>

pid_t wait(int *wstatus);
```

- 特点

  - **阻塞父进程**

  - 等待 **任意一个子进程结束**

  - 回收子进程资源

- 返回值：

  - 成功，返回被回收的子进程的pid

  - 失败，返回-1

- wstatus：获取子进程退出信息。获取到的退出信息可以用宏解析：

  | 宏                    | 作用               |
  | --------------------- | ------------------ |
  | `WIFEXITED(status)`   | 子进程是否正常退出 |
  | `WEXITSTATUS(status)` | 获取返回值         |
  | `WIFSIGNALED(status)` | 是否被信号杀死     |

  > 需要注意status，status不是子进程返回的值，而是返回值进行加密后的一个整型值。
  >
  > `status` 是一个 **编码后的整数**，里面包含了多种信息，比如：
  >
  > - 子进程是否正常退出
  > - 子进程返回值
  > - 是否被信号终止
  > - 终止信号是什么
  >
  > ```c
  > // 子进程
  > return 5;
  > ```
  >
  > ```c
  > // 父进程
  > wait(status);
  > ```
  >
  > 此时`status != 5`

# 5.waitpid函数

```c
#include <sys/types.h>
#include <sys/wait.h>

pid_t waitpid(pid_t pid, int *wstatus, int options);
```

`waitpid()` 是 `wait()` 的 **增强版**。

- 功能：

  - 指定等待哪个子进程

  - 可以不阻塞

- pid

  | pid值    | 含义           |
  | -------- | -------------- |
  | pid > 0  | 等待指定 PID   |
  | pid = -1 | 等待任意子进程 |
  | pid = 0  | 等待同组进程   |
  | pid < -1 | 等待指定进程组 |

- wstatus：获取子进程退出信息。

- options：控制是否阻塞

  | 选项    | 作用     |
  | ------- | -------- |
  | 0       | 阻塞等待 |
  | WNOHANG | 不阻塞   |

# 6.wait函数和waitpid函数的常见用法

## 6.1 场景一：父进程等待子进程结束并获取子进程返回信息

```c
#include <sys/types.h>
#include <sys/wait.h>
#include <unistd.h>
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    pid_t pid = fork();

    if(pid == -1) {
        perror("child process created error");
        exit(1);
    }
    // 子进程
    if(pid==0) {
        printf("child process created, pid: %d\n", getpid());
        sleep(10);
        exit(5);
    }
    // 父进程
    if(pid > 0) {
        int status;
        wait(&status);
        printf("子进程是否正常退出：%d\n", WIFEXITED(status));
        printf("子进程返回值：%d\n", WEXITSTATUS(status));
        printf("子进程是否被信号杀死：%d\n", WIFSIGNALED(status));
    }
}
```
