# 1.进程号和相关函数

每个进程都由进程号来标识，其类型为 `pid_t`（整型），进程号的范围：0～32767。

任何进程（除 init 进程）都是由另一个进程（至少是init进程）创建，该进程称为被创建进程的父进程，对应的进程号称为父进程号（PPID）。

> 比如一个进程是从终端输入指令运行的，那么他的父进程就是当前终端。

进程组是一个或多个进程的集合。他们之间相互关联，**进程组可以接收同一终端的各种信号**，关联的进程有一个进程组号（PGID）。默认情况下，当前的进程号会当做当前的进程组号。

进程号和进程组相关函数：

```c
pid_t getpid(void);			// 获取进程号
pid_t getppid(void);		// 获取父进程号
pid_t getpgid(pid_t pid);	// 获取进程组号
```

# 2.创建进程

系统允许一个进程创建新进程，子进程还可以创建新的子进程，形成一个树形结构。

使用`fork`函数创建子进程：

```c
#include <sys/types.h>
#include <unistd.h>

pid_t fork(void);
```

- 返回值：
  - 成功：父进程返回 **子进程 PID**，子进程返回 **0**；（调用一次，返回两次）
  - 失败：返回 **-1**（在父进程中返回）

> 子进程和线程的区别：
>
> - 子进程间的地址空间独
>
> - 立，相互之间不能访问
>
>   ```
>   进程A
>   ┌───────────┐
>   │ code      │
>   │ data      │
>   │ heap      │
>   │ stack     │
>   └───────────┘
>
>   进程B
>   ┌───────────┐
>   │ code      │
>   │ data      │
>   │ heap      │
>   │ stack     │
>   └───────────┘
>   ```
>
> - 线程之间共享同一地址空间，且能够访问同一变量（除了栈中的，栈中数据是独立的）
>
>   ```
>   进程
>   ┌──────────────────┐
>   │ code             │
>   │ data             │
>   │ heap             │
>   │                  │
>   │ stack (thread1)  │
>   │ stack (thread2)  │
>   │ stack (thread3)  │
>   └──────────────────┘
>   ```

```c
#include <sys/types.h>
#include <unistd.h>
#include <stdio.h>

int main(void)
{
  int pid = fork();
  if (pid == 0)
  {
    printf("this is child\n");
  }
  else
  {
    printf("this is parent\n");
  }
}
```

上面这段代码实际上会在终端中打印两次，因为fork被调用后，父进程和子进程后续代码都被执行了

```
zyc@ubuntu:~/Linux/Lesson11$ ./ls
this is parent
this is child
```

# 3.父子进程虚拟空间

`fork()` 调用时，操作系统会把父进程的 **地址空间、数据、打开的文件等资源复制一份给子进程**，然后父子进程都从 `fork()` 返回处继续执行。

- 用户区：子进程和父进程一样
- 内核区：pid不同，其他基本一致

在父进程调用fork之后，本质上类似于git里创建一个分支，就是父进程把自己的整块内存复制一份（实际上是用的时候复制）：

```
父进程A
┌───────────────┐
│ text          │
│ data          │
│ heap          │
│ stack         │
└───────────────┘


子进程B
┌───────────────┐
│ text          │
│ data          │
│ heap          │
│ stack         │
└───────────────┘
```

包含：

- 进程代码
- 数据
- 堆
- 栈

之后两个进程继续从fork之后开始执行，所以fork会返回两次，一次给父进程，一次给子进程。

> 终端中运行一个程序本质上是先创建一个子进程，再把代码和数据全部换掉：
>
> 本质分两步：`fork(创建子进程)` + `exec(替换数据)`

## 3.1 写时拷贝（也称读时共享）

fork 时不马上复制内存，只有当某个进程“写入”时才真正复制。

### **如果没有写时拷贝会怎样**

假设父进程有 100MB 内存：

```
fork()
```

如果直接复制：

```
父进程 100MB → 子进程再复制 100MB
```

问题：

- 很慢
- 占用大量内存
- 但很多情况下子进程马上 `exec()`，复制完全浪费

### **写时拷贝的做法**

`fork()` 时 **父子进程先共享同一份物理内存**。

```
fork 之后

父进程 ──┐
         ├── 同一块物理内存
子进程 ──┘
```

此时：

- 两个进程的虚拟地址空间相同
- 指向同一块物理页

但页被标记为：

```
只读
```

### 当发生写操作时

假设子进程执行：

```
x = 20;
```

CPU发现：

```
这块页是只读
```

于是触发：

```
page fault（缺页异常）
```

操作系统会：

1. 复制这块内存页
2. 子进程指向新的页（谁先触发缺页异常，谁先复制并创建新页，不一定是子进程指向新页）
3. 父进程继续使用旧页

结果：

```
父进程 ──→ page A
子进程 ──→ page B (复制出来的)
```

这就是：Copy On Write

# 4.子进程和线程的区别



- 子进程间的地址空间独立，相互之间不能访问

  ```
  进程A
  ┌───────────┐
  │ code      │
  │ data      │
  │ heap      │
  │ stack     │
  └───────────┘
  
  进程B
  ┌───────────┐
  │ code      │
  │ data      │
  │ heap      │
  │ stack     │
  └───────────┘
  ```

- 线程之间共享同一地址空间，且能够访问同一变量（除了栈中的，栈中数据是独立的）

  ```
  进程
  ┌──────────────────┐
  │ code             │
  │ data             │
  │ heap             │
  │                  │
  │ stack (thread1)  │
  │ stack (thread2)  │
  │ stack (thread3)  │
  └──────────────────┘
  ```
