# 1.预备知识

1. TCP和UDP通信时，数据到达后第一时间是存在内核的socket缓冲区里的，调用`send`和`recv`函数本质上是把数据从用户进程的缓冲区拷贝到内核的那块socket缓冲区。

2. 阻塞和非阻塞：socket缓冲区内没有数据到达时，进程是阻塞还是继续执行

   > 非阻塞情况下：当`recv`返回-1的时候，表示有错误发生，但`EINTR`,`EAGAIN`,`EWOULDBOLCK`不致命，需要额外写信号处理函数处理。

3. 同步和异步

   - 同步：应用程序自己调用`recv`去取内核缓冲区里的数据（程序自己要）

   - 异步：应用程序告诉内核自己的`buf`地址，有数据到了之后内核自动把数据放到buf里，然后通知应用程序（`SIGIO`信号），应用程序自己干自己的事（内核主动给）

     ```c
     #include <aio.h>
     
     int aio_read(struct aiocb *aiocbp);
     /*
         - 功能：请求内核异步读取数据
         - aiocbp：指向 aiocb 结构体（AIO 控制块）的指针
         - 工作机制：
             - 调用后立即返回，不阻塞。
             - 内核在后台完成数据读取。
         - 返回值：
             - 成功：0（仅表示请求已排队，不代表读完）
             - 失败：-1，并设置 errno
     */
     
     int aio_write(struct aiocb *aiocbp);
     /*
         - 功能：请求内核异步写入数据
         - aiocbp：指向 aiocb 结构体（AIO 控制块）的指针
         - 工作机制：
             - 调用后立即返回，数据由内核异步从缓冲区写入文件/套接字。
         - 返回值：
             - 成功：0
             - 失败：-1，并设置 errno
     */
     
     struct aiocb {
         int             aio_fildes;     /* 文件描述符 */
         off_t           aio_offset;     /* 文件偏移量（读写的起始位置） */
         volatile void  *aio_buf;        /* 缓冲区地址 */
         size_t          aio_nbytes;     /* 读写的字节数 */
         int             aio_reqprio;    /* 请求优先级 */
         struct sigevent aio_sigevent;   /* 通知方式（信号、回调或无） */
         int             aio_lio_opcode; /* 仅在 lio_listio 中使用 */
     };
     ```

4. 五种IO模型

   - 阻塞型

     阻塞是针对缓冲区来说的，内核缓冲区有数据的时候才产生事件，告知系统调用可以进行拷贝

     ![image-20211209201202971](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F04%2F0839085de5544aaa21bf8ccef6d0b192.png)

   - 非阻塞型

     内核缓冲区中没有数据，就不会产生事件，`read`系统调用直接返回-1，同时设置`errno`为`EAGAIN`

     ![image-20211209201425774](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/2026%2F04%2Fba130d7c697e2a4e905bec852263490c.png)

   - IO复用型

     IO复用解决的问题是在一个线程里管理多个连接，而不是一个线程管理一个连接，这就大大提升了连接的数量。在生产的场景下，应对于高并发服务器的高并发需求，需要开多个线程，每个线程都进行IO复用。

   - 信号驱动型

     每次数据准备好后，向用户进程发送一个`SIGIO`信号，用户进程编写信号处理函数在其中调用`read`进行读取
