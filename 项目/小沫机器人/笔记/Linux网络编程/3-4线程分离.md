线程的分离使用的是`pthread_detach`函数：

```c
#include <pthread.h>

int pthread_detach(pthread_t thread);
/*
    - 功能：将指定的线程设置成分离状态（Detached State）。
    - 返回值：
        - 成功：0
        - 失败：错误号（errno），通常是因为线程 ID 无效。
*/
```

一个线程默认是 **joinable** 的，这意味着：

- 线程结束后
- 它占用的一些资源不会立刻完全释放
- 需要别的线程调用 `pthread_join` 来回收

如果调用：

```
pthread_detach(thread);
```

就表示把这个线程改成 **detached** 状态，这样线程结束后：

- 系统会自动回收它的资源
- 不需要再 `pthread_join`

---

`pthread_detach`和`pthread_join`的区别

1. `join`：
   1. 子线程退出后，返回的结果主线程需要拿来处理
   2. 主线程必须等待子线程干完拿结果
2. `detach`：
   1. 主线程不需要**等待**子线程的结果，自己跑自己的

## 案例

主线程把数组和数组长度传给子线程，子线程负责求和，再把结果返回给主线程。

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

typedef struct
{
    int *arr;
    int size;
} thread_arg;

void *callback(void *arg)
{
    static int sum = 0;
    thread_arg *data = (thread_arg *)arg;
    for (int i = 0; i < data->size; i++)
    {
        sum += data->arr[i];
    }
    return &sum;
}

int main(void)
{
    int arr[] = {1, 2, 3, 4, 5};
    thread_arg arg;
    arg.arr = arr;
    arg.size = sizeof(arr) / sizeof(int);

    pthread_t child_thread;
    if (pthread_create(&child_thread, NULL, callback, &arg) != 0)
    {
        printf("create child thread error.\n");
    }

    void *retVal = NULL;
    pthread_join(child_thread, &retVal);	// 获取子线程的值
    printf("child thread's result is %d.\n", *(int *)retVal);

    return 0;
}
```
