# 1.概念

当前线程等待其他线程通知，否则放下持有的锁，并且进入阻塞状态。

> 看起来有点绕，为啥要先放下锁：
>
> 不放锁，其他线程可能也要持有锁，那就干不了活，别的线程永远不会通知你。

# 2.函数

```c
pthread_cond_t
```

## 2.1 条件变量初始化

```c
int pthread_cond_init(
    pthread_cond_t *restrict cond,
    const pthread_condattr_t *restrict attr
);
/*
    - 功能：初始化一个条件变量
    - cond：要初始化的条件变量
    - attr：属性（一般传 NULL，表示默认）
    - 返回值：
        - 成功：0
        - 失败：错误码（不是 -1）
*/
```

## 2.2 条件变量销毁

```c
int pthread_cond_destroy(pthread_cond_t *cond);
/*
    - 功能：销毁一个条件变量
    - cond：要销毁的条件变量
    - 注意：销毁前，不能还有线程在等待这个条件变量
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

## 2.3 等待条件满足

```c
int pthread_cond_wait(
    pthread_cond_t *restrict cond,
    pthread_mutex_t *restrict mutex
);
/*
    - 功能：阻塞等待条件变量满足
    - cond：要等待的条件变量
    - mutex：配套使用的互斥锁
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 说明：
    - 调用前，线程必须先加锁 mutex
    - 调用后，线程会自动释放 mutex，并进入阻塞等待
    - 当被唤醒后，会自动重新加锁 mutex，然后再继续执行

## 2.4 超时等待条件满足

```c
int pthread_cond_timedwait(
    pthread_cond_t *restrict cond,
    pthread_mutex_t *restrict mutex,
    const struct timespec *restrict abstime
);
/*
    - 功能：在指定时间内等待条件变量满足
    - cond：要等待的条件变量
    - mutex：配套使用的互斥锁
    - abstime：超时时间点
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 说明：
    - 调用前，线程必须先加锁 mutex
    - 调用后，线程会自动释放 mutex，并进入阻塞等待
    - 如果在超时时间到达前被唤醒，则自动重新加锁 mutex 后继续执行
    - 如果超时仍未等到条件满足，则返回超时错误

## 2.5 唤醒一个条件变量

```c
int pthread_cond_signal(pthread_cond_t *cond);
/*
    - 功能：唤醒至少一个等待该条件变量的线程
    - cond：要操作的条件变量
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 说明：
    - 如果有多个线程在等待，通常只唤醒其中一个
    - 如果没有线程在等待，则此调用没有效果

## 2.6 唤醒所有等待该条件变量的线程

```c
int pthread_cond_broadcast(pthread_cond_t *cond);
/*
    - 功能：唤醒所有等待该条件变量的线程
    - cond：要操作的条件变量
    - 返回值：
        - 成功：0
        - 失败：错误码
*/
```

- 说明：
    - 所有等待在该条件变量上的线程都会被唤醒
    - 被唤醒后，这些线程还需要重新竞争互斥锁 mutex

# 3.案例

下面这个是worker线程的代码，在执行任务之前，需要等待别的线程，比如：

- 初始化线程会改 `initialized`
- 管理线程会改 `paused`
- 配置线程会改 `config_valid`
- 连接线程会改 `conn_ready`
- 生产线程会改 `has_task`
- 停机线程会改 `shutdown`

```c
pthread_mutex_lock(&mutex);

while ((!initialized || paused || !config_valid || !conn_ready || !has_task) 
       && !shutdown) {
    pthread_cond_wait(&cond, &mutex);
}

if (shutdown) {
    pthread_mutex_unlock(&mutex);
    return NULL;
}

// 到这里说明：系统可运行，而且有任务
task = get_task();
pthread_mutex_unlock(&mutex);

// 处理任务
process_task(task);
```

最重要的是这个：

```c
while ((!initialized || paused || !config_valid || !conn_ready || !has_task) 
       && !shutdown) {
    pthread_cond_wait(&cond, &mutex);	// 让当前的worker线程主动睡觉
}
```

`pthread_cond_signal`唤醒函数是被其他线程调用的，每有一个线程发送signal，当前的worker线程就醒一次，检查`while`里的条件是否都满足，但凡有一个条件不满足，worker线程就继续睡觉。

```c
initialized = 1;
pthread_cond_signal(&cond);	// 初始化线程发送信号
```
