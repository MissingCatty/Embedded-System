# 1.概述

| 容器                                        | 底层实现 | 特点                                                     | 插入时间复杂度    | 查看（查找/访问）     | 删除时间复杂度    |
| :------------------------------------------ | :------- | :------------------------------------------------------- | :---------------- | :-------------------- | :---------------- |
| `vector`                                    | 动态数组 | 元素连续存储，支持随机访问，插入/删除需要移动元素        | O(N)              | O(1)                  | O(N)              |
| `deque`                                     | 双向队列 | 支持两端高效插入和删除，随机访问性能次于 `vector`        | O(N)              | O(1)                  | O(N)              |
| `list`                                      | 双向链表 | **不支持随机访问**，插入/删除操作效率高                  | O(1)              | O(N)                  | O(1)              |
| `map` / `set`                               | 红黑树   | 元素有序存储，插入/删除自动排序，不允许重复键            | O(log N)          | **O(log N)**          | O(log N)          |
| `multimap` / `multiset`                     | 红黑树   | 支持重复元素，其他特性与 `map` / `set` 相同              | O(log N)          | O(log N)              | O(log N)          |
| `unordered_map` / `unordered_set`           | 哈希表   | 元素无序存储，快速查找和插入，可能因哈希冲突导致性能下降 | O(1)（最坏 O(N)） | **O(1)**（最坏 O(N)） | O(1)（最坏 O(N)） |
| `unordered_multimap` / `unordered_multiset` | 哈希表   | 不保证顺序，快速查找，支持重复键（一对多）               | O(1)（最坏 O(N)） | O(1)（最坏 O(N)）     | O(1)（最坏 O(N)） |

1. 如何区分哪些情况使用map哪些使用unordered_map？
   - **要顺序 → `map`**
   - **只要速度（不关心顺序）→ `unordered_map`**
   - **数量极小（<100）→ 随意，`map` 有时更简单**

> 我的理解：
>
> 1. 无序map才是真正的hashmap，通过hash函数找到对应的value。map是红黑树，本质是一个二叉树。

2. vector和list的使用场景分别是什么？

   - vector（顺序表）：大部分情况用这个，**随机访问**（根据下表取值）速度快
   - list（链表）：涉及频繁插入和删除操作

3. map，set，multimap，multiset

   本质都是红黑树（平衡的二叉搜索树），找key的过程就是在树里找节点的过程。

   > 二叉搜索树：key作为树的节点，一个节点左边的子树里的节点值一定比该节点小，右边的都比该节点大。

4. 基于红黑树的map和基于hashtable的unordered_map在使用场景中的区别？

   - 如果只是想存储key-value对时，操作时只是取key对应的value，使用unordered_map更优，因为速度快

   - unordered_map的劣势：**更占内存**
   - map的优势：
     - **按 key 顺序遍历**
     - **范围查询**：找出所有 key 在 [a, b] 之间的元素
     - **找最小/最大 key**
