## 2.1 uart

### 2.1.1 裸机方式的缺陷

#### 2.1.1.1 接收效率低

由于本项目采用的是基于freertos的方式实现的，所以uart的接收部分必须要进行修改。因为如果裸机方式，像如下这样：

```c
volatile uint8_t rx_data = 0;
...
    
void USART1_IRQHandler(void)
{
    if (USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)
    {
        rx_data = USART_ReceiveData(USART1);
        USART_ClearITPendingBit(USART1, USART_IT_RXNE);
    }
}
```

即使串口的接收中断被触发，收到一个字节保存到`rx_data`中时，主程序并不知道`rx_data`的值已经改变了，仍然需要轮询去检查`rx_data`，轮询也是需要占用cpu时间的。

#### 2.1.1.2 发送效率低

原本裸机发送的代码如下：

```c
void usart_send(uint8_t data)
{
    USART_SendData(USART1, data);
    while (USART_GetFlagStatus(USART1, USART_FLAG_TC) != SET);
}
```

发送一个字节完成后，cpu在该函数中死循环等待标志位，不符合使用freertos的初衷，应该是在等待的过程中cpu可以去执行其他任务。

### 2.1.2 基于freertos的修改

针对上述问题，解决办法如下：

#### 2.1.2.1 接收效率低

为了防止某个任务轮询检查`rx_data`变量，我们应该设计为：当中断接收到数据时，通知任务可以取了。

可以使用经典的“生产者-消费者”模型：某个需要去串口缓冲区取数据的任务作为消费者，串口接收中断作为生产者。

- 消费者在开始时先take一个信号量，保证数据为空时可持有的信号量为0。之后消费者任务在尝试读取缓冲区前take一个信号量，如果信号量不够，就阻塞在这；否则，就顺利读取数据。
- 生产者在读取到一定数量的数据后，需要通知对应消费者任务取数据，此时give信号量。此时消费者发现有信号量可以take，就执行取数据的逻辑。

整个过程避免了某个任务一遍又一遍的询问“缓冲区是否有数据可取”的耗时操作，改为使用rtos阻塞，由于任务被阻塞，在有数据可取之前cpu都不会在该任务上耗时。

这就是信号量的“**同步**”作用，即：**我准备好了，你开始吧。**

同时，为了防止每接收到一个数据就give信号量，可以采用uart的IDLE标志位，当uart空闲时，表示一帧的数据已经发送完毕，此时可以通知消费者任务来取了。

```c
#include "usart.h"
#include "ringbuffer8.h"
#include "FreeRTOS.h"
#include "semphr.h"
#include "task.h"
#include <stdio.h>

#define BUFFER_SIZE 256

uint8_t       buffer[BUFFER_SIZE];
ringbuffer8_t rb;

SemaphoreHandle_t xUsartSemaphore;

...

void USART1_IRQHandler(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    if (USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)
    {
        if (!rb8_full(rb))
        {
            uint16_t rx_data = USART_ReceiveData(USART1); // 9位数据
            rb8_put(rb, (uint8_t)rx_data);                // 强制截断为8位
        } else
        {
            printf("[ringbuffer error]: ringbuffer is full.");
        }
        USART_ClearITPendingBit(USART1, USART_IT_RXNE);
    }

    if (USART_GetITStatus(USART1, USART_IT_IDLE) != RESET)
    {
        // 读取DR数据寄存器以清除IDLE标志，volatile防止编译器优化掉这句
        volatile uint16_t dummy = USART_ReceiveData(USART1); 
        if (!rb8_empty(rb))
        {
            // 如果give后，有比当前更高优先级的任务就绪，xHigherPriorityTaskWoken则变为true
            xSemaphoreGiveFromISR(xUsartSemaphore, &xHigherPriorityTaskWoken); 
        }
        USART_ClearITPendingBit(USART1, USART_IT_IDLE);
    }

    // 手动进行任务切换
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

F4系列和F1系列不同，F1系列的DMA只有Channel的概念，也就是说DMA的某个通道DMAx_CHy只对应固定的硬件请求，比如：`USART1_TX` 的DMA请求 **永远** 链接到 `DMA1_Channel4`。

而F4系列多了一个Stream的概念，如上表所示，每个DMA通道可以有多个Stream，但每个Stream是独立的，并且只能被一个DMA通道使用。

```c
void _usart_dma_init(void)
{
    // 使能DMA2时钟
    RCC_AHB1PeriphClockCmd(RCC_AHB1Periph_DMA2, ENABLE);

    // DMA配置
    DMA_InitTypeDef DMA_InitStructure;
    DMA_StructInit(&DMA_InitStructure);
    DMA_InitStructure.DMA_Channel            = DMA_Channel_4;
    DMA_InitStructure.DMA_PeripheralBaseAddr = (uint32_t)&USART1->DR;
    DMA_InitStructure.DMA_DIR                = DMA_DIR_MemoryToPeripheral;  // 从内存到外设
    DMA_InitStructure.DMA_PeripheralInc      = DMA_PeripheralInc_Disable;   // 外设地址不变
    DMA_InitStructure.DMA_PeripheralDataSize = DMA_PeripheralDataSize_Byte; // 外设数据宽度为8位
    DMA_InitStructure.DMA_MemoryInc          = DMA_MemoryInc_Enable;        // 内存地址递增
    DMA_InitStructure.DMA_MemoryDataSize     = DMA_MemoryDataSize_Byte;     // 内存数据宽度为8位
    DMA_InitStructure.DMA_Mode               = DMA_Mode_Normal;             // 非循环模式
    DMA_InitStructure.DMA_Priority           = DMA_Priority_Low;            // DMA优先级为低
    DMA_InitStructure.DMA_FIFOMode           = DMA_FIFOMode_Enable;         // 启用FIFO提高传输效率
    DMA_InitStructure.DMA_FIFOThreshold      = DMA_FIFOThreshold_Full;      // FIFO全满时开始传输
    DMA_InitStructure.DMA_MemoryBurst        = DMA_MemoryBurst_INC8;        // 一次性传输8个数据单元，配合FIFO进一步提高效率
    DMA_InitStructure.DMA_PeripheralBurst    = DMA_PeripheralBurst_Single;  // 外设突发传输为单次传输
    DMA_Init(DMA2_Stream7, &DMA_InitStructure);

    // DMA中断配置
    DMA_ITConfig(DMA2_Stream7, DMA_IT_TC, ENABLE);
}
```

使用DMA进行串口发送：

```c
SemaphoreHandle_t xUsartTxSemaphore;

...
    
void usart_send(const char *str)
{
    int32_t len = strlen(str);

    int32_t chunk_size = 65535;

    // 如果数据过长，就分块发送
    do
    {
        len = len < chunk_size ? len : chunk_size;

        // 准备发送
        DMA2_Stream7->M0AR = (uint32_t)str;
        DMA2_Stream7->NDTR = len;
        DMA_Cmd(DMA2_Stream7, ENABLE);

        // 上锁，等待发送完毕
        xSemaphoreTake(xUsartTxSemaphore, portMAX_DELAY);

        // 后续数据处理
        len -= chunk_size;
        str += chunk_size;
    } while (len > 0);

    xSemaphoreGive(xUsartTxSemaphore);
    USART_ClearFlag(USART1, USART_FLAG_TC);
}

...

void DMA2_Stream7_IRQHandler(void)
{
    if (DMA_GetITStatus(DMA2_Stream7, DMA_IT_TCIF7) != RESET)
    {
        BaseType_t xHigherPriorityTaskWoken = pdFALSE;
        xSemaphoreGiveFromISR(xUsartTxSemaphore, &xHigherPriorityTaskWoken); // 开锁
        portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
        DMA_ClearITPendingBit(DMA2_Stream7, DMA_IT_TCIF7);
    }
}
```

### 时钟源

RTC 时钟源 —RTCCLK 可以从**LSE**、**LSI**和**HSE_RTC**这三者中得到。其中**使用最多的是LSE**，LSE由一个外部的32.768KHZ（6PF负载）的晶振提供，精度高，稳定，RTC首选。LSI是芯片内部的30KHZ晶体，精度较低，会有温漂，一般不建议使用。HSE_RTC由HSE分频得到，最高是4M，使用的也较少。

### 预分频器

RTC 的预分频器 PRER 由两部分组成：

- **APRE（7 位异步预分频器）**：为亚秒计数器 RTC_SSR 提供时钟。
- **SPRE（15 位同步预分频器）**：生成更新日历的 1 Hz 时钟。

异步和同步预分频后的频率为：

- $f_{\text{CK\_APRE}} = \frac{f_{\text{RTC\_CLK}}}{PREDIV\_A + 1}$
- $f_{\text{CK\_SPRE}} = \frac{f_{\text{CK\_APRE}}}{PREDIV\_S + 1}$

使用两个预分频器时，**APRE 取较大值以降低功耗**。常见做法是使用 **LSE 32.768 kHz** 作为 RTC 时钟源，并通过 PRER 分频得到 **1 Hz** 更新时钟。

最常用的配置：

- $PREDIV\_A = 127$
- $PREDIV\_S=255$

$$
f_{\text{CK\_SPRE}}=\frac{32.768k}{(127+1)(255+1)}=1 Hz
$$

### 设置日期和时间

```c
#include "stm32f4xx.h"
#include <string.h> // 用于 memcmp 和 memcpy

// 自定义的时间日期结构体
typedef struct {
    uint16_t year;    // 年 (例如: 2025)
    uint8_t  month;   // 月 (1-12)
    uint8_t  day;     // 日 (1-31)
    uint8_t  weekday; // 星期 (1=Mon, 7=Sun, 对应 SPL 的 RTC_Weekday_...)
    uint8_t  hour;    // 时 (0-23)
    uint8_t  minute;  // 分 (0-59)
    uint8_t  second;  // 秒 (0-59)
} rtc_date_time_t;

/*
 * ====================================================================
 * 内部辅助函数 (static)
 * ====================================================================
 */

/**
 * @brief  单次设置硬件RTC时间 (无保护)
 * @param  date_time: 指向自定义时间结构体的指针
 */
static void _rtc_set_time_once(const rtc_date_time_t *date_time)
{
    RTC_DateTypeDef date;
    RTC_TimeTypeDef time;
    
    // 确保结构体被正确初始化
    RTC_DateStructInit(&date);
    RTC_TimeStructInit(&time);
    
    // 1. 将自定义结构体 转换为 SPL 库结构体
    // 注意: SPL库的年份是 0-99，所以需要 - 2000
    date.RTC_Year = (uint8_t)(date_time->year - 2000); 
    date.RTC_Month = date_time->month;
    date.RTC_Date = date_time->day;
    date.RTC_WeekDay = date_time->weekday;
    
    time.RTC_Hours = date_time->hour;
    time.RTC_Minutes = date_time->minute;
    time.RTC_Seconds = date_time->second;
    time.RTC_H12 = RTC_H12_AM; // 假设使用24小时制, 此项无所谓
    
    // 2. 将 BINARY (二进制) 格式的数据写入硬件
    //    使用 BIN 格式比 BCD 格式更简单直观
    RTC_SetDate(RTC_Format_BIN, &date);
    RTC_SetTime(RTC_Format_BIN, &time);
}

/**
 * @brief  单次读取硬件RTC时间 (无保护)
 * @param  date_time: 用于存储读出时间的指针
 */
static void _rtc_get_time_once(rtc_date_time_t *date_time)
{
    RTC_DateTypeDef date;
    RTC_TimeTypeDef time;
    
    // 确保结构体被正确初始化
    RTC_DateStructInit(&date);
    RTC_TimeStructInit(&time);
    
    // 1. 从硬件读取 BINARY 格式的数据
    RTC_GetDate(RTC_Format_BIN, &date);
    RTC_GetTime(RTC_Format_BIN, &time);
    
    // 2. 将 SPL 库结构体 转换回 自定义结构体
    // 注意: 读出后要 + 2000
    date_time->year = (uint16_t)date.RTC_Year + 2000;
    date_time->month = date.RTC_Month;
    date_time->day = date.RTC_Date;
    date_time->weekday = date.RTC_WeekDay;
    date_time->hour = time.RTC_Hours;
    date_time->minute = time.RTC_Minutes;
    date_time->second = time.RTC_Seconds;
}


/*
 * ====================================================================
 * 公共API函数 (Public)
 * ====================================================================
 */

/**
 * @brief  【你请求的】设置RTC时间 (带异步保护)
 * @param  date_time: 指向要设置的时间结构体的指针
 */
void rtc_set_time(const rtc_date_time_t *date_time)
{
    rtc_date_time_t rtime; // 用于存储回读的时间

    // 关键: 为了防止在“设置”和“回读”之间被中断打断，
    // 进入临界区保护。
    __disable_irq(); 

    do {
        // 1. 尝试设置时间
        _rtc_set_time_once(date_time);
        
        // 2. 立即读回刚刚设置的时间
        _rtc_get_time_once(&rtime);
        
    // 3. 比较“想要设置的秒数”和“实际读回的秒数”
    //    如果不相等，说明设置时发生了RTC跳秒，或者写入尚未生效
    //    此时需要重新设置。
    } while (date_time->second != rtime.second);

    // 退出临界区
    __enable_irq();
}

/**
 * @brief  读取RTC时间 (带跨秒一致性保护)
 * @param  date_time: 用于存储读出时间的指针
 */
void rtc_get_time(rtc_date_time_t *date_time)
{
    rtc_date_time_t time1, time2;

    do {
        // 1. 第一次读取
        _rtc_get_time_once(&time1);
        
        // 2. 极快地进行第二次读取
        _rtc_get_time_once(&time2);
        
    // 3. 比较两次读取的全部内容
    //    如果两次读取不一致 (memcmp != 0)，说明在两次读取之间
    //    发生了时间跳变 (例如从 11:59:59 -> 12:00:00)
    //    此时 time1 或 time2 可能是错误数据 (如 12:00:59)
    //    必须重新读取，直到两次完全一致。
    } while (memcmp(&time1, &time2, sizeof(rtc_date_time_t)) != 0);
    
    // 两次读取一致，说明 time1 是一个有效的、一致的时间快照
    memcpy(date_time, &time1, sizeof(rtc_date_time_t));
}
```
