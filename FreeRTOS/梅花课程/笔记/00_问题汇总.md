# 1.配置成复用模式时，到底是输入还是输出

1. **复用模式（GPIO_Mode_AF）的作用**：
   - 表示该引脚被外设（如USART、SPI等）占用，**方向由外设功能自动管理**。
   - 例如，USART_TX是外设的发送端，方向为**输出**；USART_RX是接收端，方向为**输入**。
2. **为何需要配置输出类型（GPIO_OType）和上拉/下拉（GPIO_PuPd）**：
   - **对于输出（如TX引脚）**：
     需明确指定推挽输出（`GPIO_OType_PP`）或开漏输出（`GPIO_OType_OD`），并设置输出速度（如`GPIO_Speed_100MHz`）。
   - **对于输入（如RX引脚）**：
     无需配置输出类型，但需根据硬件设计选择是否启用上拉/下拉电阻（如`GPIO_PuPd_UP`或`GPIO_PuPd_NOPULL`）。

# 3.为什么被中断访问的变量一般要声明为`volatile`

未被关键字`volatile`修饰的变量，在编译器优化时，往往为了提高访问速度，会将该变量放到寄存器里（访问寄存器速度比直接访问内存要快）。但中断对变量的读写都是直接作用于内存上的。

例如，在主循环中每次输出变量`data`的值，假如某次执行中断前，内存中`data=10`，寄存器中的`data=10`，执行中断后，内存中的`data=11`，而寄存器中的`data`有可能还是10。这就对于主程序和中断访问同一个变量时会造成不同步的问题。

```c
int data = 0;

while(1)
{
	printf("%d", data);
}

void xxx_IRQHandler(void)
{
    data++;
}
```

为了解决该问题，应该使用`volatile`关键字去修饰该变量，意味着禁止编译器对该变量进行优化，即所有对该变量的读写都是直接读写内存。

# 4.stm32的时钟配置如何查看

![image-20250302204421902](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20250302204421902.png)

`system_stm32f4xx.c`文件中`SystemInit`函数中包含RCC时钟树的初始化。

其中，调用`SetSysClock`函数进行相关配置，该函数设置了`SYSCLK`时钟选择（f40系列默认选择主PLL作为时钟输入）。

需要注意的是系统时钟`Sysclk`的计算，公式为：
$$
\text{Sysclk} = \frac{\text{HSE}\cdot\text{PLL} _N}{\text{PLL}_M\cdot\text{PLL}_P}
$$
要满足168Mhz的要求，需要注意$\text{HSE}$和$\text{PLL}_M$（一般两个相除正好为**1MHz**）

同时默认设置了AHB，APB1，APB2的分频系数分别为：1，4，2，即默认频率为AHB（168），APB1（42），APB2（84）

# 5.使用定时器完成延时功能为什么不需要考虑计数溢出？

假设定时器的时钟频率是42MHz（42000000Hz），要让程序延迟1us，就是要CNT按照1000000Hz的频率计数，此时CNT每记一个数，就耗时1ns。所以要配置预分频器为`42-1`。

计数逻辑如下：

```c
void Delay_ns(uint32_t us)
{
    uint32_t start = TIMx->CNT;  // 记录开始时间
    while((TIMx->CNT-start)>us); // 每次循环查看计数次数是否达到us
}
```

按照上述逻辑，开始时记录下当前计数值，然后每次检查定时器的计数值与开始计数值的差是否达到规定值。这么做一定会在某次计数时产生溢出，所以是否需要对溢出次数进行计数？

答案是不用，只要将时基单元中的自动重装值记为`uint32_t`最大的`0xFFFFFFFF`即可。

**核心原理：无符号数的自动回绕**

- 假设定时器的计数器是**32位**（如STM32的TIM2/TIM5），最大值为`0xFFFFFFFF`。
- 当计数器溢出时，无符号数的减法运算会自动处理回绕。

示例：

- `start = 0xFFFFFF00`（开始值）
- `current = 0x000000FF`（当前值，溢出后）
- **计算差值**：`current - start = 0x000000FF - 0xFFFFFF00 = 0x000001FF`（无符号数自动回绕，结果为正确差值）。

# 6.assert函数的作用

 **概念：C/C++** 中的 **`assert`** 是一个**宏**，用于在运行时检查一个条件是否为真，如果条件不满足，则运行时将终止程序的执行并输出一条错误信息。

```c
int main() {
    int a = 10, b = 20;
    assert(a > b);    // 条件不成立，程序会终止
    return 0;
}
```

# 7.编译时出现`Undefined symbol __aeabi_assert (referred from xxx.o)`错误

"Error: L6218E: Undefined symbol __aeabi_assert..." 能够发生是因为“ **Project -> Options for Target -> on the Target tab - Use MicroLIB**” 被选中。

microlib是一个比ARM标准C库小的独立库。为了节省大小，arm microlib c库不支持或实现几乎所有与操作系统交互的函数，例如abort（）、exit（）或assert（）。

**解决办法**

在调用`<assert.h>`之前定义`NDEBUG`，标准C的 `assert()` 会被禁用，所有断言检查在编译时被移除。

```c
// <assert.h>

#ifdef NDEBUG
#   define assert(ignore) ((void)0)
#   define __promise(e) ((__ARM_PROMISE)((e)?1:0))
#else
#   if defined __DO_NOT_LINK_PROMISE_WITH_ASSERT
#      if defined __OPT_SMALL_ASSERT && !defined __ASSERT_MSG && !defined __STRICT_ANSI__ && !(_AEABI_PORTABILITY_LEVEL != 0 || (!defined _AEABI_PORTABILITY_LEVEL && __DEFAULT_AEABI_PORTABILITY_LEVEL != 0))
#          define assert(e) ((e) ? (void)0 : __CLIBNS abort())
#      elif defined __STDC__
#          define assert(e) ((e) ? (void)0 : __CLIBNS __aeabi_assert(#e, __FILE__, __LINE__))
#      else
#          define assert(e) ((e) ? (void)0 : __CLIBNS __aeabi_assert("e", __FILE__, __LINE__))
#      endif
#      define __promise(e) ((__ARM_PROMISE)((e)?1:0))
#   else
#      if defined __OPT_SMALL_ASSERT && !defined __ASSERT_MSG && !defined __STRICT_ANSI__ && !(_AEABI_PORTABILITY_LEVEL != 0 || (!defined _AEABI_PORTABILITY_LEVEL && __DEFAULT_AEABI_PORTABILITY_LEVEL != 0))
#undef __promise
#          define assert(e) ((e) ? (void)0 : __CLIBNS abort(), (__ARM_PROMISE)((e)?1:0))
#      else
#          define assert(e) ((e) ? (void)0 : __CLIBNS __aeabi_assert(#e, __FILE__, __LINE__), (__ARM_PROMISE)((e)?1:0))
#      endif
#      define __promise(e) assert(e)
#   endif
#endif
```

# 8.什么叫弱编译

在Keil的MicroLib中，许多底层函数和关键功能被定义为**弱符号（Weak Symbols）**，允许开发者根据需求自定义实现。以下是具体信息和查找方法：

MicroLib默认将以下类别的函数标记为弱符号，允许用户重写：

1. **内存管理函数**

   - `void* malloc(size_t size)`

   - `void free(void* ptr)`

   - `void* realloc(void* ptr, size_t size)`

   - `void* calloc(size_t num, size_t size)`
     ​**用途**：自定义堆管理策略（如静态内存池、动态内存分配器）。

2. **底层I/O函数**

   - `int _write(int fd, char* ptr, int len)`
     ​**用途**：重定向标准输出（如将`printf`输出到UART）。

   - `int _read(int fd, char* ptr, int len)`
     ​**用途**：重定向标准输入（如从串口读取数据）。

3. **错误处理函数**
   - `void __aeabi_assert(const char* expr, const char* file, int line)`
     ​**用途**：自定义断言失败时的行为（如记录错误日志）。
4. **系统初始化函数**
   - `void _sys_init(void)`
     ​**用途**：自定义系统启动时的初始化流程。

5. **其他工具函数**

   - `int _close(int fd)`

   - `int _lseek(int fd, int offset, int whence)`
     ​**用途**：文件操作相关的底层实现（适用于有文件系统的场景）。

重写时只需要在项目中包含.c文件，在该源文件中重写函数。

注意：重写时需要保证函数名、参数表、返回类型与定义的完全相同。

```c
__weak void weakFunction(void){
    //do something
    return;
}
 
 
void someFunctionCall(void){
    //do something
    weakFunction();
    //do something
    return;
}
```

**注意：`__weak`修饰符只用在源文件里，不在头文件使用。**

# 9.stm32在哪一种输出模式下同时可以读取电平

开漏输出模式

需要注意的是，如果在配置为开漏输出模式时想要读取引脚上的有效电平，主设备需要往该io口写入1，释放总线控制权，让其他设备控制总线，此时读取到的电平才是有效的。否则，如果输出低电平，该总线会被一直拉低，这样其他设备无论输出高低电平，读取到的都是低电平，也就是无效电平。
