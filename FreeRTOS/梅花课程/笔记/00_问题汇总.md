# 1.配置成复用模式时，到底是输入还是输出

1. **复用模式（GPIO_Mode_AF）的作用**：
   - 表示该引脚被外设（如USART、SPI等）占用，**方向由外设功能自动管理**。
   - 例如，USART_TX是外设的发送端，方向为**输出**；USART_RX是接收端，方向为**输入**。
2. **为何需要配置输出类型（GPIO_OType）和上拉/下拉（GPIO_PuPd）**：
   - **对于输出（如TX引脚）**：
     需明确指定推挽输出（`GPIO_OType_PP`）或开漏输出（`GPIO_OType_OD`），并设置输出速度（如`GPIO_Speed_100MHz`）。
   - **对于输入（如RX引脚）**：
     无需配置输出类型，但需根据硬件设计选择是否启用上拉/下拉电阻（如`GPIO_PuPd_UP`或`GPIO_PuPd_NOPULL`）。

# 3.为什么被中断访问的变量一般要声明为`volatile`

未被关键字`volatile`修饰的变量，在编译器优化时，往往为了提高访问速度，会将该变量放到寄存器里（访问寄存器速度比直接访问内存要快）。但中断对变量的读写都是直接作用于内存上的。

例如，在主循环中每次输出变量`data`的值，假如某次执行中断前，内存中`data=10`，寄存器中的`data=10`，执行中断后，内存中的`data=11`，而寄存器中的`data`有可能还是10。这就对于主程序和中断访问同一个变量时会造成不同步的问题。

```c
int data = 0;

while(1)
{
	printf("%d", data);
}

void xxx_IRQHandler(void)
{
    data++;
}
```

为了解决该问题，应该使用`volatile`关键字去修饰该变量，意味着禁止编译器对该变量进行优化，即所有对该变量的读写都是直接读写内存。
