# 1.通信原理

dht11采用单总线的通信方式与主机通信，只有在接收到开始信号后才会触发温湿度采集。

**传输格式**

一次传输5个字节，传输时，高位在前，低位在后。

数据格式为：

> 湿度整数数据(8bit) + 湿度小数数据(8bit) + 温度整数数据(8bit) + 温度小数数据(8bit) + 校验和(8bit)

- 湿度小数部分（DHT11 恒为 0）
- 8bit 温度小数部分（DHT11 恒为 0）
- 校验和（前 4 个字节的和）

注：在传输数据时，需要严格按照上述时序，不允许打断。

**硬件连线**

dht11只需要一根DATA线与主机相连，且需要**外接上拉电阻**。

**传输时序**

1. **主机请求**

   主机需要通过单总线向 DHT11 发送一个起始信号，以触发数据传输：

   - **主机拉低总线**（起使信号）：主机将 **数据线拉低**（低电平 0） **至少 18ms**（确保 DHT11 能够检测到请求）
   - **主机释放总线**（释放控制权）：主机将数据线 **拉高** **20~40µs**，进入等待状态，准备接收 DHT11 的响应信号

2. **DHT11响应**

   DHT11 在检测到主机的起始信号后，会进行如下操作：

   - **DHT11 拉低数据线**（响应信号）：DHT11 **主动拉低** 数据线 **80µs**，表示已经收到主机的请求
   - **DHT11 拉高数据线**（准备发送数据）：DHT11 **主动拉高** 数据线 **80µs**，准备开始传输温湿度数据

3. **数据传输**

   DHT11 发送 **40bit（5字节）** 的数据给主机，**单个数据位**传输方式：

   - DHT11 拉低总线 50µs（**起始信号**）
   - **发送 0**：数据线拉高 **26~28µs** 后拉低
   - **发送 1**：数据线拉高 **70µs** 后拉低
