# 1.芯片简介

SP32-C3 采用的是 32 位 **RISC-V** 单核处理器。

## **核心特性：RISC-V 处理器**

- **CPU：** 搭载了 32 位的 RISC-V (RV32IMC) 单核处理器。这是一个开源的指令集架构 (ISA)，是其与 ESP8266 (Xtensa) 和 传统 ESP32 (Xtensa-LX6) 最大的区别之一。
- **时钟频率：** 主频最高可达 **160** MHz。
- **性能：** 提供了足够的计算能力来处理中低等复杂度的 IoT 任务，如传感器数据处理、本地控制和云端通信。

## **核心连接性：Wi-Fi 和 蓝牙**

ESP32-C3 是一款专注于连接的芯片，它集成了两种关键的无线技术：

1. **Wi-Fi (2.4 GHz)**
   - 支持 IEEE 802.11 b/g/n 标准。
   - 提供了完整的 Wi-Fi 协议栈，使其可以轻松连接到路由器和互联网。
2. **蓝牙 5 (LE)**
   - 支持低功耗蓝牙 (Bluetooth LE)，包括 Bluetooth 5.0 和 Bluetooth Mesh。
   - 这使其非常适合用于设备配网 (通过手机APP)、信标 (Beacon)、或组建大规模的低功耗设备网络。

ESP32-C3 的外设配置是**“连接优先，安全增强，模拟精简”**。它砍掉了 DAC 和部分高级模拟外设，但保留了 IoT 项目最核心的通信接口和控制功能。

**1. 通信接口 (Communication)**

这是它的强项，除了无线之外，有线的接口也很实用：

- **Wi-Fi 4：** 支持 2.4 GHz, 802.11 b/g/n
- **蓝牙 5 (LE)：** 支持低功耗蓝牙 (BLE) 和 蓝牙 Mesh (Bluetooth Mesh)
- **UART (串口)：** 2 组。最常用的接口，用于日志打印、GPS 模块、与其他 MCU 通信等。
- **I2C (Inter-Integrated Circuit)：** 1 组。用于连接各种传感器，如温湿度计、陀螺仪、OLED 屏幕等。
- **SPI (Serial Peripheral Interface)：** 2 组。一组是通用 SPI (GP-SPI)，另一组 (MSPI) 专用于连接片外的 Flash 和 PSRAM (C3 通常只外挂 Flash)。常用于驱动 TFT 彩色屏幕、SD 卡、以太网模块等。
- **I2S (Inter-IC Sound)：** 1 组。用于连接数字音频设备，如 I2S 麦克风 (INMP441) 或音频功放 (MAX98357)。
- **TWAI (Two-Wire Automotive Interface)：** 1 组。它是一个兼容 CAN 2.0 B 协议的控制器，这使 ESP32-C3 **可以用于工业自动化或汽车电子相关的总线通信**。

**2. 模拟功能 (Analog)**

这部分相对精简：

- **ADC (模数转换器)：** 2 个 12 位的 SAR ADC。总共可以在多达 6 个 GPIO 引脚上进行模拟信号读取（例如读取光敏电阻、电位器）。
- **温度传感器：** 1 个内置的温度传感器，用于感知芯片内部温度。

> **注意：** ESP32-C3 **没有** 内置 DAC (数模转换器)，无法像“经典”ESP32 那样直接输出模拟电压波形。

**3. 定时器与控制 (Timers & Control)**

- **LEDC (LED PWM 控制器)：** 6 个通道。这是一个专门用于 PWM（脉冲宽度调制）的模块，常用于控制 LED 的亮度（呼吸灯）或电机的转速。
- **RMT (红外遥控/Remote Control)：** 4 个通道。这是一个非常灵活的脉冲发生器和接收器，虽然叫“红外遥控”，但它最常见的用途是驱动 **WS2812 (Neopixel)** 这类可编程 RGB LED 灯带。
- **通用定时器：** 2 组 64 位的通用定时器。
- **看门狗定时器 (WDT)：** 3 个（1个系统看门狗，2个模块看门狗），用于在程序死锁时自动重启芯片。

# 3.芯片使用

## 3.1 准备工作

硬件：

- 一款 **ESP32-C3** 开发板
- **USB 数据线** （A 转 Micro-B）
- 电脑（Windows、Linux 或 macOS）

软件：

如需在 **ESP32-C3** 上使用 ESP-IDF，请安装以下软件：

- 设置 **工具链**，用于编译 ESP32-C3 代码；
- **编译构建工具** —— CMake 和 Ninja 编译构建工具，用于编译 ESP32-C3 **应用程序**；
- 获取 **ESP-IDF** 软件开发框架。该框架已经基本包含 ESP32-C3 使用的 API（软件库和源代码）和运行 **工具链** 的脚本；

## 3.2 环境设置

我们使用Linux系统进行工具链的搭建

**下载软件包**

编译 ESP-IDF 需要以下软件包，Ubuntu 和 Debian:

```shell
sudo apt-get install git wget flex bison gperf python3 python3-pip python3-venv cmake ninja-build ccache libffi-dev libssl-dev dfu-util libusb-1.0-0
```

检查是否有python3：

```
python3 --version
```

**获取ESP-IDF**

在围绕 ESP32-C3 构建应用程序之前，请先获取乐鑫提供的软件库文件 [ESP-IDF 仓库](https://github.com/espressif/esp-idf)。

```bash
mkdir -p ~/esp
cd ~/esp
git clone -b v5.5.1 --recursive https://github.com/espressif/esp-idf.git
```

**设置工具**

除了 ESP-IDF 本身，还需要为支持 ESP32-C3 的项目安装 ESP-IDF 使用的各种工具，比如编译器、调试器、Python 包等。

```bash
cd ~/esp/esp-idf
./install.sh esp32c3
```

上述命令仅仅为 ESP32-C3 安装所需工具。如果需要为多个目标芯片开发项目，则可以一次性指定多个目标，如下所示：

```bash
cd ~/esp/esp-idf
./install.sh esp32,esp32s2
```

如果需要一次性为所有支持的目标芯片安装工具，可以运行如下命令：

```bash
cd ~/esp/esp-idf
./install.sh all
```

> **自定义工具安装路径**
>
> 上述步骤的默认安装路径为`$HOME/.espressif`，如果想要将工具安装在其他地方，**需要在运行安装脚本前，导出环境变量 IDF_TOOLS_PATH**。注意，请确保用户账号已经具备了读写该路径的权限。
>
> ```bash
> export IDF_TOOLS_PATH="$HOME/required_idf_tools_path"
> ./install.sh
> 
> . ./export.sh
> ```
>
> 如果修改了 `IDF_TOOLS_PATH` 变量，请在运行任意 ESP-IDF 工具或脚本前，将该变量导出到环境变量中。

**设置环境变量**

此时，刚刚安装的工具尚未添加至 PATH 环境变量，无法通过**命令行**使用这些工具。因此，必须设置环境变量，将可执行文件的路径添加到环境变量里。

**※※※※※※请在需要运行 ESP-IDF 的终端窗口运行以下命令**：

```bash
. $HOME/esp/esp-idf/export.sh
```

只有这样才能使用`idf.py`作为命令。

如果需要经常运行 ESP-IDF，可以为执行 `export.sh` 创建一个别名，具体步骤如下：

1. 复制并粘贴以下命令到 shell 配置文件中（`.profile`、`.bashrc`、`.zprofile` 等）

   ```bash
   alias get_idf='. $HOME/esp/esp-idf/export.sh'
   ```

2. 通过重启终端窗口或运行 `source [path to profile]`，如 `source ~/.bashrc` 来刷新配置文件。

现在可以在任何终端窗口中运行 `get_idf` 来设置或刷新 ESP-IDF 环境。

不建议直接将 `export.sh` 添加到 shell 的配置文件。这样做会导致在每个终端会话中都激活 IDF 虚拟环境（包括无需使用 ESP-IDF 的会话）。这违背了使用虚拟环境的目的，还可能影响其他软件的使用。

现在你已经具备了使用 ESP-IDF 的所有条件，接下来将介绍如何开始第一个工程。

## 3.3 ESP-IDF使用

【注意】：该部分中要使用`idf.py`作为命令，必须在开启窗口后执行

```bash
. $HOME/esp/esp-idf/export.sh
```

### 3.3.1 创建工程

可以从 ESP-IDF 中 [examples](https://github.com/espressif/esp-idf/tree/v5.5.1/examples) 目录下的 [get-started/hello_world](https://github.com/espressif/esp-idf/tree/v5.5.1/examples/get-started/hello_world) 工程开始。

将 [get-started/hello_world](https://github.com/espressif/esp-idf/tree/v5.5.1/examples/get-started/hello_world) 工程复制至本地的 `~/esp` 目录下：

```bash
cd ~/esp
cp -r $IDF_PATH/examples/get-started/hello_world .
```

### 3.3.2 连接设备

现在，请将 ESP32-C3 开发板连接到 PC，并查看开发板使用的串口。

通常，串口在不同操作系统下显示的名称有所不同：

- **Linux 操作系统：** 以 `/dev/tty` 开头

### 3.3.3 配置工程

请进入 `hello_world` 目录，设置 ESP32-C3 为目标芯片，然后运行工程配置工具 `menuconfig`。

```bash
cd ~/esp/hello_world
idf.py set-target esp32c3
idf.py menuconfig
```

打开一个**新工程**后，应首先使用 `idf.py set-target esp32c3` 设置“目标”芯片。注意，此操作将清除并初始化项目之前的编译和配置（如有）。也可以直接将“目标”配置为环境变量（此时可跳过该步骤）。

可以通过此菜单设置项目的具体变量，包括 Wi-Fi 网络名称、密码和处理器速度等。`hello_world` 示例项目会以默认配置运行，因此在这一项目中，可以跳过使用 `menuconfig` 进行项目配置这一步骤。

idf.py build

### 3.3.5 烧录到设备

请运行以下命令，将刚刚生成的二进制文件烧录至 ESP32-C3 开发板：

```bash
idf.py flash
```

> 如果使用的是WSL，需要将windowsPC上的USB口映射到WSL上。
>
> **1. (在 Windows 上) 安装 `usbipd-win`**
>
> 最简单的方法是使用 `winget`。打开 **PowerShell (管理员)** 并运行：
>
> ```powershell
> winget install --interactive --id dorssel.usbipd-win
> ```
>
> 安装过程中，它会安装一个服务和一个命令行工具。
>
> **2. (在 Windows 上) 共享 (Attach) 设备**
>
> 这一步将一个物理 USB 设备“绑定”到 `usbipd` 服务，以便它可以在 WSL (Windows Subsystem for Linux) 环境中被访问。
>
> 这是关键步骤。打开一个新的 **PowerShell (管理员)**：
>
> 列出所有连接到 Windows 的 USB 设备：
>
> ```powershell
> usbipd list
> ```
>
> 你会看到一个列表，包含 `BUSID`、`VID:PID` 和设备描述。找到你想要连接到 WSL 的设备，并记下它的 **`BUSID`**。
>
> **将设备绑定到usbipd：** 假设你S-VO-U-B-S的设备 `BUSID` 是 `2-1`：
>
> ```powershell
> usbipd bind --busid 2-1
> ```
>
> ![image-20251102161831511](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/25%2F11%2F98e2f5a4b78696fe0b4a8b271e0b5712.png)
>
> **3.绑定设备到 WSL**
>
> ```powershell
> usbipd attach --wsl --busid 2-1
> ```
>
> ![image-20251102162123594](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/25%2F11%2F1e3711813c24e796c077af4322635fe3.png)

>
> 回到你的 **WSL 终端**，运行：

> ```
>
> 如果一切顺利，你现在应该能看到刚刚附加的 USB 设备，就像它被物理连接到 Linux 系统一样。你现在可以在 WSL 中使用这个设备了。

### 3.3.6 监视输出

可以使用 `idf.py monitor` 命令，监视 “hello_world” 工程的运行情况。

如果输出发现

![image-20251102165805790](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/25%2F11%2F8cc37e604cf98d1fdd7113b5af969d71.png)

**关键警告 (Warning)**: `W (225) spi_flash: Detected size(4096k) larger than the size in the binary image header(2048k). Using the size in the binary image header.`

- **含义**：你的开发板上**物理安装的是 4MB 的 Flash 芯片**。
- **但是**：你的项目配置 (`idf.py menuconfig` -> Serial flasher config -> Flash size) 被设置为了 **2MB**。
- **结果**：ESP-IDF 将**只会使用这 4MB 空间中的前 2MB**。这**不会导致错误**，但你“浪费”了 2MB 的空间。

因此，需要将项目配置为4MB，要用满flash。
