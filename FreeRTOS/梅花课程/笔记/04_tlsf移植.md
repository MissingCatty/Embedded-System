# 1.TLSF简介

**Arena（内存池）是什么？**

- **核心目的**：减少频繁系统调用（如`malloc`和`free`）的开销。
  - 系统调用分配/释放内存代价高，因此预先通过系统调用申请一大块内存（称为“大块内存”或“Arena”），后续需要内存时直接从这块内存中“切分”小块使用。
  - Arena负责管理这些小块内存的分配和回收，避免频繁与操作系统交互。
- **关键问题**：
  所有Arena算法需解决两个问题：
  1. **分配时间复杂度**：如何快速找到合适大小的内存块。
  2. **内存碎片**：频繁分配释放后，剩余内存可能零散，无法被有效利用。

**常见的Arena实现方式**

1. **Sequential Fit**
   - 按内存块大小从小到大排列空闲块，分配时遍历链表找到第一个满足要求的块。
   - **缺点**：时间复杂度为O(N)，N是空闲块数量，效率低。
2. **Segregated Free Lists**
   - 将内存块按大小分类，每个类别维护一个链表（例如：16字节、32字节等）。
   - **优点**：若类别少，直接定位到对应链表，时间复杂度O(1)。
   - **缺点**：若类别多（如按1字节间隔分类），会占用大量内存维护链表。
3. **Buddy Systems（伙伴系统）**
   - 将内存按2的幂次分割，分配时递归分割大块，释放时合并相邻块。
   - **优点**：碎片较少，时间复杂度O(logN)。
   - **缺点**：合并条件严格，可能产生内部碎片。

---

**TLSF（Two-Level Segregated Fit）**

**TLSF的目标**

- **低碎片** + **O(1)时间复杂度分配/释放**，适合实时系统（耗时可控）。

**核心设计**

1. **两级索引结构**
   - **第一级（First Level）**：按内存大小的**指数范围**分类（如2⁴~2⁵、2⁵~2⁶等）。
   - **第二级（Second Level）**：在每个指数范围内，进一步**线性分割**（如将2⁶~2⁷分为4等份）。
   - **效果**：通过两级索引快速定位到目标内存块，时间复杂度O(1)。
2. **示例：分配85字节**
   - **步骤1**：确定指数范围。85属于2⁶（64）~2⁷（128）。
   - **步骤2**：在第二级中，将64~128分为4份：64~80、80~96、96~112、112~128。
   - **步骤3**：85落在80~96区间，直接取该区间的空闲块（如89字节的块），多余4字节作为内部碎片。
3. **数据结构**
   - **Segregated List**：仅维护空闲块，用于快速分配。
   - **Physical List**：维护所有块（包括已分配和空闲块），按物理地址连接，用于合并相邻空闲块减少碎片。
   - **位图（Bitmap）**：标记每个槽位是否有空闲块，加速查询。
4. **合并策略**
   - 释放内存时，检查相邻块是否空闲，若空闲则合并成大块，避免碎片。

---

**TLSF的优势**

1. 时间复杂度O(1)
   - 通过位运算直接计算两级索引，无需遍历链表或树。
2. 低碎片
   - 线性分割减少内部碎片，合并策略减少外部碎片。
3. 适用性广
   - 支持任意大小内存分配，适合实时系统和高性能场景。

---

**总结**

- **Arena本质**：通过预分配大块内存，自主管理小块内存的分配/释放，避免频繁系统调用。
- **TLSF亮点**：结合指数分割（减少槽位数）和线性分割（快速定位），实现高效低碎片的内存管理。

# 2.TLSF移植方法

下图中程序使用的是RAM1，总大小为`0x20000B = 131072 / 1024 = 128KB`

![image-20250308153501582](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20250308153501582.png)

在移植时，首先需要申请一块大的内存池，假定使用64KB作为动态内存池（Arena）。

```c
tlsf_t  tlsf_ptr;
tlsf_ptr = tlsf_create_with_pool(arena, ARENA_SIZE);
```

之后再在这个动态内存池中申请空间

```c
void *ptr  = tlsf_malloc(tlsf_ptr, size);
```

可以使ptr指向任意的内容，例如：

```c
uint8_t* a = (int *)ptr;
```

此时，ptr申请了`size`个字节的空闲空间，然后用指针`a`指向这块内存首地址。

# 3.为什么要移植TLSF

由上面的介绍可知，TLSF是一类内存管理的实现方法，microlib库中声明了`malloc`等内存管理的函数，并且实现了这些函数，但函数的具体实现在源文件里，源文件一般看不到，如果使用固定的一套源文件在不同的设备上运行出现问题或者需要实现自己的功能时，这就不好对源文件进行修改。

所以，microlib库对部分函数的实现（源文件中）使用了`__weak`修饰，使项目在编译时，如果发现在一个源文件中还有一个`malloc`函数的实现，且未被`__weak`修饰，则使用该文件中的函数实现，而忽略库文件中的被`__weak`修饰的函数实现
