# 1.HC05模块介绍

HC-05和HC-06是广泛使用的经典蓝牙模块，常用于与单片机进行无线通信。

基于**蓝牙2.0标准**，**支持串口UART通信协议**，所以与单片机的连接和数据交换较为简单。

HC-05可以作为**主设备**或**从设备**，支持蓝牙SPP（串口通信）和AT命令模式。

HC-06通常作为从设备，仅支持蓝牙SPP模式。

**通信范围在10米**左右。

# 2.HC-05工作模式

具有两种工作模式：

- 命令响应工作模式
- 自动连接工作模式，该模式下又分为三种工作角色
  - master
  - slave
  - loopback（回环）

当工作在**自动连接工作模式**时，将**会自动根据事先设定好的方式连接和数据传输**。

当工作在**命令响应工作模式**时，能够执行下述所有AT指令。

**工作模式的切换由外部引脚（PIO11）输入的电平**，可以实现工作模式的动态转换。

# 3.HC05角色

- Master：**查询周围SPP蓝牙从设备，并主动发起连接**，从而建立主、从蓝牙设备间的透明数据传输通道。
- Slave：被动连接
- Slave-Loop：回环角色，被动连接，接收主设备数据并将数据原封不动的返回给主设备。

# 4.HC05引脚

| 名称     | 类型     | 功能描述                                                     | 引脚       |
| -------- | -------- | ------------------------------------------------------------ | ---------- |
| UART_TXD | CMOS输出 | 串口输出                                                     | TX         |
| UART_RXD | CMOS输入 | 串口输入                                                     | RX         |
| VCC3.3   | 电源输入 | 3.3v电源                                                     | VCC        |
| GND      | 地       | 地                                                           | GND        |
| PIO(9)   | 输出     | 配对状态输出：配对成功输出高电平，未配对输出低电平           | STATE(LED) |
| PIO(11)  | 输入     | 状态切换引脚：高（AT命令响应工作状态）；低/悬空（蓝牙常规工作状态） | EN(KEY)    |

# 5.命令响应模式

进入命令响应模式的方法：

1. 按下模块上的按键
2. 对HC-05重新上电，模块进入AT响应命令模式
3. 使用串口助手，配置成波特率38400（复位后默认），数据位8位，停止位1位，无校验位，无流控制
4. 串口助手发送"AT\r\n"，正常情况下会返回"\r\n"为回车换行，也叫做新行。

（简单来说，长按HC05按键时，才会进入AT命令响应模式）
