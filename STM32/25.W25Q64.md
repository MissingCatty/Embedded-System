# 1.简介

- W25Qxx系列是一种低成本、小型化、使用简单的非易失性存储器，常应用于数据存储、字库存储、固件程序存储等场景
- 存储介质：Nor Flash（闪存）
- 时钟频率：80MHz / 160MHz (Dual SPI) / 320MHz (Quad SPI)
- 存储容量（24位地址）：
  - W25Q40：  4Mbit / 512KByte
  - W25Q80：  8Mbit / 1MByte
  - W25Q16：  16Mbit / 2MByte
  - W25Q32：  32Mbit / 4MByte
  - W25Q64：  64Mbit / 8MByte
  - W25Q128： 128Mbit / 16MByte
  - W25Q256： 256Mbit / 32MByte

# 3.框图

- 图示Flash芯片为8MB空间（128*64KB = 8MB）
- 地址长度3B（24位）
- 存储器的两种分法
  - 总共分为128个block(64KB)，每个block(4KB)又分为16个sector
    - block：xx0000~xxFFFF
    - sector：xxx000~xxxFFF
  - 总体上以256B为一页进行页的划分，即一页有2^8^个地址
    - page：xxxx00~xxxxFF
- 一个block分有FFFF+1（2^16^）个地址，则每个地址为64KB/2^16^=1B，即按字节编址
- 读写SPI指定地址需要送3个字节。总空间大小为8MB=2^23^B，一页大小为256B=2^8^B，所以一共有2^15^页，一页256=2^8^个地址。指定地址时，送三个字节，最低一个字节指明页内地址，高两个字节指明页号（起始页号15位就能表示，多出来一位，所以最大地址为7FFFFF而不是FFFFFF）

# 4.注意事项

**写入**

1. 写入操作前，需要写使能
2. 每个数据只能由1改写成0，不能由0改写成1（RAM可以覆盖写入）
   - 如：原数据1010，现在写入0101，最后结果为0000
3. 写入数据前，必须先擦除，擦除后，所有数据位变为1
4. 擦除必须按照最小单元进行擦除（这个芯片可以选择整块擦除/按block擦除/按sector擦除
5. 连续写入多字节时，**最多写入一页的数据**，超过页尾位置的数据，会回到页首覆盖写入
6. Buffer大小为一页，同时Buffer和每一个页是对应的，即写入某一页第N个地址，那数据在Buffer中也是第N个地址，所以如果数据在页的中间开始写，并连续写入256B，那么必然会造成一部分数据从页首开始写入，导致打算写入的地址和实际写入的地址不一致。**所以，一次写入必须从每一页的第一个地址开始。**
7. 写入操作结束后（此时待写数据全部放在Buffer里了），flash进入忙状态，不响应新的请求（从Buffer到Flash的过程较慢，所以写入操作结束后，需要等一段时间将Buffer中的数据搬到Flash里）

**读取**

直接调用读取时序，无需使能，无需额外操作，没有页的限制，读取操作结束后不会进入忙状态，但**不能在忙状态时读取**
