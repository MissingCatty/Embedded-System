# 1.简介

- 可配置8位/16位数据帧、高位先行/低位先行
- 时钟频率： fPCLK / (2, 4, 8, 16, 32, 64, 128, 256)
  - 如：SPI1是APB2的外设，f~PCLK~=72MHz

- 支持多主机模型、主或从操作
- 可精简为半双工/单工通信
- 支持DMA
- 兼容I2S协议（数字音频协议，用于音频阶码）
- STM32F103C8T6 硬件SPI资源：SPI1、SPI2

## 二者区别

- 连续传输：TDR的数据被转移到移位寄存器的时候，此时**TXE=1**，就可以把下一个要发的数据放到TDR里了
- 非连续传输：TDR数据被转移到移位寄存器进行发送的时候，此时TXE=1，不着急往TDR里写下一个数据，等待当前字节发送完毕后，由于发送完毕也是接收完毕，先把RDR里数据读出来，此时RXNE=1，把RDR接收的数据读出来，再把下一个要发的字节送进TDR
  - 等待TXE=1，准备要发送的数据如TDR
  - TXE=0，(TDR）→移位寄存器，转移完成后TXE=1
  - 等待RXNE=1，此时RDR满，从数据将RDR读出
  - 往RDR里写下一个数据
  
  非连续传输在程序上如下：
  
  ```c
  //等待TXE=1
  while(SPI_I2S_GetFlagStatus(SPI2,SPI_I2S_FLAG_TXE) == 0);
  //把数据移入TDR
  SPI_I2S_SendData(SPI2,ByteSend);
  //等待RXNE=1
  while(SPI_I2S_GetFlagStatus(SPI2,SPI_I2S_FLAG_RXNE) ==0);
  //把数据从RDR移入
  return SPI_I2S_ReceiveData(SPI2);
  ```

发送多个字节时，上述过程不断循环，总是发送→接收→发送→...→接收，每次要发的数据在接收之后才写入TDR，所以是非连续的

## 注意事项

非连续传输发送的每个字节之间都有间隙，这个间隙在频率低的时候不明显可以忽略，但在时钟频率较快的时候，这个间隙就很严重了无法忽视，会极大的影响传输速率
