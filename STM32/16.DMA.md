# 1.简介

DMA可以提供外设和存储器或者存储器和存储器之间的高速数据传输，无须CPU干预，节省了CPU的资源。

- 12个独立可配置的通道： DMA1（7个通道）， DMA2（5个通道）
- 每个通道都支持软件触发和特定的硬件触发
- STM32F103C8T6 DMA资源：DMA1（7个通道）

# 4.DMA通道

![image-20240118182812759](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240118182812759.png)

- 数据宽度：单位数据传输长度
- 传输计数器：一共要传多少数据
- M2M：控制是硬件触发还是软件触发
- 软件触发：当设置了DMA_CCRx寄存器中的MEM2MEM位之后，在软件设置了DMA_CCRx寄存器中的EN位启动DMA通道时，DMA传输将马上开始。当DMA_CNDTRx寄存器变为0时，DMA传输结束。
  - 如果MEM2MEM=0，说明是硬件触发，当启动DMA通道时，不会立刻开始传输，需要一个硬件的触发信号才开始传输。
  - 【注意】：所以，当MEM2MEM=1时，一旦开启DMA通道，传输数据计数器计到零为止，如果此时DMA处于循环模式，即计数器到0后立刻重装，那么DMA就永远停不下来了。**故MEM2MEM=1不可与循环模式一起使用**。

# 8.DMA循环模式

在配置DMA时，会指定`DMA_BufferSize`的值，这个值实际上就是一共要转移多少字节的数据。每转移一个字节，`CNDTR`（DMA转移计数寄存器，初值为`DMA_BufferSize`）自减1，自减到0时，DMA配置全部自动变为初始值（就算使DMA通道失能，`CNDTR`的值也不会变）：

```c
DMA_InitTypeDef DMA_InitStructure;
DMA_InitStructure.DMA_PeripheralBaseAddr = (uint32_t)&USART1->DR;
DMA_InitStructure.DMA_PeripheralDataSize = DMA_PeripheralDataSize_Byte;
DMA_InitStructure.DMA_PeripheralInc = DMA_PeripheralInc_Disable;
DMA_InitStructure.DMA_MemoryBaseAddr = (uint32_t)DMA_BUF;
DMA_InitStructure.DMA_MemoryDataSize = DMA_PeripheralDataSize_Byte;
DMA_InitStructure.DMA_MemoryInc = DMA_MemoryInc_Enable;
DMA_InitStructure.DMA_DIR = DMA_DIR_PeripheralSRC;
DMA_InitStructure.DMA_BufferSize = 5;
DMA_InitStructure.DMA_Mode = DMA_Mode_Circular;
DMA_InitStructure.DMA_M2M = DMA_M2M_Disable;
DMA_InitStructure.DMA_Priority = DMA_Priority_Medium;
DMA_Init(DMA1_Channel5, &DMA_InitStructure);
```

循环模式最有用的一点就是：设置为循环模式，当`DMA_BufferSize`个字节传输完成时，`DMA_PeripheralBaseAddr`和`DMA_MemoryBaseAddr`会自动变为初始值。比如，DMA转移目的地是一个长度为1024的字节数组`Buf`，设置`DMA_Mode = DMA_Mode_Circular`（循环模式）和`DMA_MemoryInc = DMA_MemoryInc_Enable`（Memory地址自增），如果某一时刻转移了1023个字节，此时`CNDTR=1`、目的地址`Buf+1023`，不会更新；再转移一个字节，此时`CNDTR=0`、目的地址复原为`Buf`，之后重装为`DMA_BufferSize`。说白了，就是只有传了`DMA_BufferSize`个字节后，DMA的`CNDTR`才复原。

当设置为单次模式时，启动下一次转换需手动设置`CNDTR`的值，**目的地址的值会自动复原**。
