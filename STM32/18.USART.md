# 1.通信接口

| **名称** | **引脚**             | **双工** | **时钟** | **电平** | **设备** |
| -------- | -------------------- | -------- | -------- | -------- | -------- |
| USART    | TX、RX               | 全双工   | 异步     | 单端     | 点对点   |
| I2C      | SCL、SDA             | 半双工   | 同步     | 单端     | 多设备   |
| SPI      | SCLK、MOSI、MISO、CS | 全双工   | 同步     | 单端     | 多设备   |
| CAN      | CAN_H、CAN_L         | 半双工   | 异步     | 差分     | 多设备   |
| USB      | DP、DM               | 半双工   | 异步     | 差分     | 点对点   |

- 时钟用于何时对输入信号进行采样，以知道多个连续的高/低电平来临时，是有几个1/0
- 同步通信和异步通信
  - 异步通信：双方之间没有CLK的约束，各自有各自的CLK，所以异步通信的发送数据必须标明什么时候开始什么时候结束，否则接收方不知道什么时候读到的是有效的数据
  - 同步通信：双方CLK连在一起，按同一个时钟进行收发


# 2.硬件电路

- 简单双向串口通信有两根通信线（发送端TX和接收端RX）

- TX与RX要交叉连接

- 当只需单向的数据传输时，可以只接一根通信线

- 当电平标准不一致时，需要加<u>电平转换芯片</u>
- 如果两个芯片都有独立的供电时，VCC可以不接
- TX和RX的高低电平是相对于GND的

# 3.电平标准

电平标准是数据1和数据0的表达方式，是传输线缆中人为规定的电压与数据的对应关系，串口常用的电平标准有如下三种：

- TTL电平：+3.3V或+5V表示1，0V表示0
- RS232电平：-3\~-15V表示1，+3\~+15V表示
- RS485电平：两线压差+2\~+6V表示1，-2\~-6V表示0（差分信号）

# 4.串口时序

- 波特率：一秒发送多少个二进制位
- 比特率：波特率 × 单个数据帧有效数据的二进制位数的占比

- 起始位：标志一个数据帧的开始，固定为**低电平**
- 数据位：数据帧的有效载荷，1为高电平，0为低电平，低位先行
- 校验位：用于数据验证，根据数据位计算得来

- 停止位：用于数据帧间隔，固定为**高电平**（用于将电平拉高，保证传下一帧之前是高电平）

# 5.STM32中USART简介

- USART是STM32内部集成的硬件外设，可根据数据寄存器的**一个字节数据**自动生成数据帧时序，从TX引脚发送出去，也可自动接收RX引脚的数据帧时序，拼接为一个字节数据，存放在数据寄存器里
- 自带波特率发生器，最高达4.5Mbits/s
- 可配置数据位长度（8/9）、停止位长度（0.5/1/1.5/2）
- 可选校验位（无校验/奇校验/偶校验）
- 支持同步模式、硬件流控制、DMA、智能卡、IrDA、LIN
  - 同步模式：多了时钟sclk的输出
  - 硬件流控制：硬件电路多一根线，当接收设备B准备好时，这根线置低电平，此时发送设备A才发送数据，防止B处理过慢导致数据积压被覆盖
- STM32F103C8T6 USART资源： USART1、 USART2、 USART3

# 6.USART电路图

![image-20240121161111865](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240121161111865.png)

- 发送数据
  - **待发数据**从总线写入**发送寄存器TDR**
  - 检查发送移位寄存器是否正在移位发送数据，如果没有，则**将TDR里的数据全部放进移位寄存器**，并置标志位TXE为1**（发送寄存器空），此时立刻在TDR写入新数据**，然后移位寄存器在发送控制器的控制下，向右移位从TX引脚发送出去
  - 当移位完成后，**新数据自动从TDR转移至移位寄存器**
- 接受数据
  - 移位寄存器右移读入RX引脚的值
  - 当移满8位时，**移位寄存器里的值转移进接受数据寄存器RDR**，并置标志位RXNE接收数据寄存器非空
  - 当移位完成后，移位寄存器可接收下一帧数据了

- 波特率发生器：用于产生发送器和接收器的时钟，按规定波特率速度进行通信
  - $f_{\text{PCLK}_x}$：x代表时APB1还是APB2，APB1频率为36MHz，APB2频率为72MHz。
  - USARTDIV：USART的分频，该分频值由波特率发生器控制，DIV_Mantissa代表分频的整数部分，DIV_Fraction代表分频的小数部分，TE=1代表时发送器的时钟，RE=1代表时接收器的时钟
  - 所以最后的发送器和接收器时钟频率为：$f_{\text{PCLK}_x}/\text{USART}_{\text{DIV}}/16$（为什么要除以16？因为对于一个数据位，需要进行16次采样以确认其值）

# 8.起始位侦测

**什么叫数据采样？**

接收方需要时刻知道信号线上的高低电平是什么样的，才能检测各个位是高电平还是低电平。但是信号线上数据的连续的（类比离散数据求和积分的关系），不可能知道所有时刻电平的高低，只能通过采样的方式去看（就好比绘制电平图），理论上来说，采样的频率越高越好越接近真实的时序。

![image-20240121180008181](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240121180008181.png)

- 采样频率为$\text{接收频率}*16=f_{\text{PCLK}_x}/\text{USART}_{\text{DIV}}$（接收频率$f_{\text{PCLK}_x}/\text{USART}_{\text{DIV}}/16$）

- 过程：如果一开始空闲状态采样一直是高电平，在某一时刻接收信号线侦测到了由1至0的跳变，说明有可能是起始帧，如果理想状态下，电路中不存在噪声，那么这就是起始帧；但实际电路存在噪声，所以还需要多次采样以确定之后是低电平，所以分别在3，5，7次和8，9，10次进行采样，如果两个三次采样中低电平的至少有2个，说明确实是低电平，是起始帧

# 9.数据采样

![image-20240121183437311](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240121183437311.png)

每一位的第8，9，10次采样判断是高电平还是低电平，如果这三次采样至少两次是高电平，那么就是高电平；如果至少两次是低电平，那么就是低电平

# 10.如何区分起始位和数据位的下降沿

起始位的侦测是接收方检测到一个下降沿时触发的。

如果本身接收方处于空闲状态（连续8位检测为高电平就认定是空闲状态），此时接收方为空闲状态$s_{idle}$，检测到下降沿后，在第0.5个波特周期采样确认为低电平，此时确认检测到起始位，接收方进入数据接收状态$s_{receive}$，连续8个波特周期后（接收8位后）的下一个0.5波特周期采样（**在一个周期中间进行采样**），如果为高电平，就说明检测到停止位，并进入暂停状态$s_{suspend}$；如果为低电平，说明最后一个周期不是停止位，帧传输错误，就丢弃该数据帧。

- 如果在停止位后（高电平）下一个周期紧跟一个低电平，此时就表明检测到起始位，接收方进入接收状态$s_{receive}$，继续接收数据；
- 如果在停止位后，保持高电平8个周期，则进入空闲状态$s_{idle}$。

所以说问题的关键在于起始位的侦测，只要侦测到起始位，后面再检测到下降沿和低电平一律视为数据位，**二者不会冲突**。同时，设立一个数据位的计数器，计数器为0时，表明规定数量的数据位已经接收完毕，就开始检测停止位。
