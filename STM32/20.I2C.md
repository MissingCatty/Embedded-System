# 1.I2C简介

- 两根通信线：SCL（Serial Clock）、SDA（Serial Data）
- 同步，半双工
- 带数据应答
- 支持总线挂载多设备（一主多从、多主多从）

# 2.硬件电路

- 所有I2C设备的SCL连在一起，SDA连在一起
- 设备的SCL和SDA均要配置成**开漏输出模式**
- SCL和SDA各添加一个上拉电阻，阻值一般为4.7KΩ左右

---

**SCL和SDA**

**1. SCL（时钟线）**

- **功能**：提供同步时钟信号，控制数据传输的时序。
- **特点**：
  - 由**主设备（Master）**生成，所有从设备（Slave）必须同步于此时钟。
  - 时钟频率由主设备配置（如标准模式100 kHz、快速模式400 kHz等）。
  - 电平为**开漏输出**，需外接上拉电阻（通常4.7 kΩ）确保信号拉高。
- **作用**：
  - 定义数据传输的速率（每个时钟脉冲对应一次数据位传输）。
  - 确保主从设备在**相同时序**下读写数据。

**2. SDA（数据线）**

- **功能**：双向数据线，传输实际数据（地址、命令、数据内容等）。
- **特点**：
  - 主设备和从设备均可驱动（需通过协议协调）。
  - 电平为**开漏输出**，需外接上拉电阻。
  - **数据传输仅在SCL低电平时允许变化，高电平时需保持稳定（确保采样正确性）**。
- **作用**：
  - 传输**起始条件（Start）**和**停止条件（Stop）**（由主设备控制）。
  - 传输**设备地址**（用于选择从设备）。
  - 传输**读写命令**（读/写操作指示）。
  - 传输**实际数据**（8位数据 + 1位应答ACK/NACK）。

# 3.起始条件和终止条件

- 起始条件：SCL高电平期间，SDA从高电平切换到低电平
- 终止条件：SCL高电平期间，SDA从低电平切换到高电平

![img](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/v2-2e19f3dfc07854c2e222409b9b23434a_720w.webp)

**注意：数据传输时，SCL高电平期间，任何设备都不能动SDA线，一旦形成一个上升沿/下降沿，就构成了终止条件/起始条件，扰乱通信时序！！！**

# 4.I2C的主机和从机

- 主机：负责控制时钟、发送起始信号、发送停止信号，并在通信过程中决定数据的流向。
- 从机：被动响应主设备的请求，接收来自主设备的数据或向主设备发送数据。

例如，某个传感器支持与主机的I2C协议数据传输。主机需要获取该传感器的数据，就需要向从机发送请求，从机从而响应，并返回数值。所以，从机总是被动的，不能主动的且随心随遇的发送任何数据。

同时，主机与从机的通讯频率完全是主机掌握的，主机将SCL拉低并释放SDA时，从机才能往SDA上放数据。

# 5.读写时序

无论主机还是从机，都要遵守：发送方低电平准备数据，接收方高电平读取数据。

谁发数据是由主机或从机的SDA决定的，SCL低电平期间，如果主机拉高释放SDA，就是让从机发数据，主机在下一个高电平期间读取。

- 主机发送字节（从机接收字节）

  - SCL**低电平**期间，主机将数据位依次放到SDA线上（**准备数据**）
  - 释放SCL，从机将在SCL**高电平**期间**读取数据**位，所以**SCL高电平期间SDA不允许有数据变化**，只有在**SCL低电平期间SDA才能变**
    - **如果非要在SCL高电平期间更改SDA，就形成了起始条件和终止条件**

  - **高位先发**

  ![image-20240123142908370](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240123142908370.png)

- 主机接收字节（从机发送字节）

  - SCL**低电平**期间，从机将数据位依次放到SDA上（**准备数据**）
  - SCL高电平期间，主机读取SDA线
  - **主机在接收数据之前，需要释放SDA（主机对SDA输出高电平，推挽输出时高电平无效），让从设备选择是否要拉低SDA**
    - SDA是具有线与的特性，所有主从设备共用SDA，所以只要有一个设备拉低SDA，SDA就是低电平，所以主机在接收数据之前，不能对SDA输出高电平

# 6.应答机制

- 主机发送应答信号：主机在接收到一个**字节**后，在**下一个时钟**发送一位应答数据，往SDA上输出0表示应答，输出1表示非应答
  - 如果从机没有接收到主机的应答信号，就会释放SDA，防止干扰主机后续操作
- 主机接收应答信号：主机在发送完一个**字节**之后，在下一个时钟接收一位数据，判断从机是否应答，数据0表示应答，数据1表示非应答（主机在接收之前，需要释放SDA：释放SDA，即主机对SDA输出高电平，在SCL下一个高电平时，主机读取SDA，若此时SDA还是高电平，说明从机没有应答）

# 7.完整时序分析

- 主机指定从机地址写一个字节

  ![image-20240124101437084](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240124101437084.png)

- 主机指定从机地址读一个字节

  ![image-20240124103159855](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240124103159855.png)

  从机中有一张内部**寄存器地址表**和一个**地址指针**，第二步中，主机送寄存器地址，地址指针会指向该地址在表中的所在位置，每读取/写入一次当前寄存器，地址指针+1。所以第三步中，主机改变模式为接收模式时，从机从地址指针指向的寄存器中送数据到SDA。

  - **SA=1**时，说明主机发送了非应答，从机接收该信号，知道主机不需要再接收数据了，于是**停止发送**
  - **SA=0**时，说明主机还想要数据，此时从机就会**继续发送**下一个数据（**多字节接收**）

【PS】：从机的接收和发送动作，完全由主机控制，主机需要切换发送/接收数据模式时，都要发送一个字节的Slave Address+R/W通知从机到底是发送还是接收

- 主机指定从机地址读/写多个字节

  时序大体上没有变化，只是发送/接收数据阶段重复多次，同时在读/写时，读写一个寄存器，都会使地址指针自增，所以连续写入/读出多个字节时，并不会出现覆盖的问题

# 8.问题思考

**在实际使用时，什么时候生成结束条件，理论上不是可以一直不结束吗，只要一直开机？**

**为什么要生成结束条件？**

I2C 总线是由主设备控制的。每次通信完成后，主设备需要生成停止条件（Stop Condition）来 **释放总线**，允许其他设备或后续通信能够顺利进行。如果不产生停止条件，总线将一直处于忙碌状态，可能会影响其他设备的通信或导致总线冲突。

如果是总线上总共只有两个设备进行通信，不大可能会出现总线占用的情况，但当有多个主机和从机时，一个主机就不能一直占着总线不放。
