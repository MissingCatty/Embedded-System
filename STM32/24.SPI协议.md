# 1.简介

- 四根通信线：SCK（Serial Clock）、MOSI（Master Output Slave Input）、MISO（Master Input Slave Output）、SS（Slave Select）
- 同步，全双工

# 2.硬件电路

- 输出引脚配置为**推挽输出**，输入引脚配置为**浮空或上拉输入**

# 3.传输流程

![image-20240127151740230](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240127151740230.png)

收发一个字节是通过数据交换完成的，主机或从机在发送一个字节的同时，也接收到一个字节，因为数据收发是靠移位寄存器移位实现的。如果主机只想接收一个字节，就需要往自己的移位寄存器里写一个无关紧要的值，用于与从机的数据进行交换。发送同理。

以主机发送为例，时钟上升沿，主机和从机的移位寄存器各向左移出一位，分别放到MOSI和MISO上；在紧接着的下降沿，主机和从机再把数据移入寄存器最低位

## 交换一个字节

### 模式0

- CPOL=0：空闲状态时，SCK为低电平
- CPHA=0：SCK第一个边沿移入数据，第二个边沿移出数据

![image-20240130084440689](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240130084440689.png)

当SS为高电平时，说明从机未被选中，此时，MISO为高阻态，用一条中间的线表示高阻态（不允许多个从机同时给主机输入数据）

由于数据必须先移出再移入，而模式0第一个上升沿是移入，所以再SS下降沿是就开始移出数据，

### 模式1

- CPOL=0：空闲状态时，SCK为低电平
- CPHA=1：SCK第一个边沿移出数据，第二个边沿移入数据

![image-20240127160935285](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240127160935285.png)

### 模式2

- CPOL=1：空闲状态时，SCK为高电平

- CPHA=0：SCK第一个边沿移入数据，第二个边沿移出数据

### 模式3

- CPOL=1：空闲状态时，SCK为高电平
- CPHA=1：SCK第一个边沿移出数据，第二个边沿移入数据
