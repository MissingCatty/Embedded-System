# 3.传输流程

![image-20240127151740230](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240127151740230.png)

收发一个字节是通过数据交换完成的，主机或从机在发送一个字节的同时，也接收到一个字节，因为数据收发是靠移位寄存器移位实现的。如果主机只想接收一个字节，就需要往自己的移位寄存器里写一个无关紧要的值，用于与从机的数据进行交换。发送同理。

以主机发送为例，时钟上升沿，主机和从机的移位寄存器各向左移出一位，分别放到MOSI和MISO上；在紧接着的下降沿，主机和从机再把数据移入寄存器最低位（移入操作包括：数据位的采样和锁存）

具体来说，移出实际上指的是将移位寄存器的最高位放到数据线（MISO或MOSI）上，移位寄存器内部向左移动一位，空出最低位等待接收新的数据位；移入指的是采样数据线（MISO或MOSI）上的电平，并将其锁存到移位寄存器的最低位。

## CPOL和CPHA

CPOL(时钟极性)位**控制在没有数据传输时时钟的空闲状态电平**，此位对主模式和从模式下的设备都有效。

- CPOL被清’0’，SCK引脚在空闲状态保持低电平（高电平时进行数据传输）
- CPOL被置’1’，SCK引脚在空闲状态保持高电平

CPHA(时钟相位)位**控制数据采样和所存的时机**。

- CPHA位被清’0’，SCK时钟的第一边沿进行数据位采样，数据在第一个时钟边沿被锁存（CPOL位为’0’时就是上升沿，CPOL位为’1’时就是下降沿）
- CPHA被置’1’，SCK时钟的第二边沿进行数据位的采样，数据在第二个时钟边沿被锁存（CPOL位为0时就是下降沿，CPOL位为’1’时就是上升沿）

## 交换一个字节

传输时序本质上就是靠SCK的上升沿和下降沿决定的，在每一个时钟周期，根据CPOL和CPHA，选择一个边沿进行数据位的发送（改变数据线并移位），对应的另一个边沿接收数据位（读取/采样数据线并锁存）。

**SCK空闲状态的用处**

数据在SCK空闲时才能放到MISO和MOSI上，供接收方读取。当SCK为高电平时，正在进行数据的采样和锁存，此时不能进行数据的改变。（**这就是为什么要设计成在某一边沿开始采样和改变数据，边沿就是一个操作的开始信号**）

**总结**

高电平期间读取数据并保存，低电平期间准备数据

| 模式   | CPOL | CPHA | SCK 空闲状态 | 采样沿<br />（<u>开始</u>读取MISO和MOSI并锁存） | 数据变化沿<br />（<u>开始</u>往MISO和MOSI上放数据） |
| ------ | ---- | ---- | ------------ | ----------------------------------------------- | --------------------------------------------------- |
| Mode 0 | 0    | 0    | 低电平       | 上升沿                                          | 下降沿                                              |
| Mode 1 | 0    | 1    | 低电平       | 下降沿                                          | 上升沿                                              |
| Mode 2 | 1    | 0    | 高电平       | 下降沿                                          | 上升沿                                              |
| Mode 3 | 1    | 1    | 高电平       | 上升沿                                          | 下降沿                                              |

# 5.常见问题

## 5.1 NSS的软件和硬件模式

通信时，通信双方的NSS必须拉低为低电平，

- 当配置为硬件模式时，该SPI设备的NSS由外部电路拉低，无法自己通过程序主动拉低。
- 当配置为软件模式时，NSS可以由软件模拟拉低，具体的方法是：
  - `SPI_CR1 -> SSM = 1`：开启SPI的软件管理使能
  - `SPI_CR1` 的 `SSI` 位用于手动控制 NSS 信号：
    - `SSI = 1`：模拟 NSS 被拉低（选中）。
    - `SSI = 0`：模拟 NSS 被拉高（取消选中）。

在 **SPI主机模式** 下，主机的 **NSS引脚对自身的SPI外设是没有作用的**，它主要用于控制从机的选通信号。主机不会读取自己NSS的状态，它只是一个输出信号，通常用于通知从机它何时应该监听数据传输。

所以，在使用SPI通信时，最常用的做法是：主机配置为软件NSS（没什么实际作用，就是不配置为硬件模式，防止硬件电路干扰），从机配置为硬件NSS。
