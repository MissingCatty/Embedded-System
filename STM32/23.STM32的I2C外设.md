# 1.简介

- STM32内部集成了硬件I2C收发电路，可以由硬件自动执行时钟生成、起始终止条件生成、应答位收发、数据收发等功能，减轻CPU的负担
- 支持多主机模型
  - 固定多主机：主机和从机固定，主机不会变成从机，从机也不会变成主机
  - 可变多主机：所有设备都是从机，任何一个设备都可以在总线空闲时，跳出来做主机，指定其他任何设备进行通信
- 支持7位/10位地址模式
  - 10位地址模式：在7位地址模式中，通信时第一个字节为7位地址+1位读写。而10位地址模式规定，发送第一个字节前5位固定为11110，告诉从机是10位地址模式，发送的第二个字节还是从机地址而不是寄存器地址，第一个字节6、7位为10位地址的前两位，第8位为读写位；地址中剩下的8位全部在第二个字节。
- 支持不同的通讯速度，标准速度(高达100 kHz)，快速(高达400 kHz)
- 支持DMA
- 兼容SMBus协议
- STM32F103C8T6 硬件I2C资源：I2C1、I2C2

# 3.I2C硬件发送和接收流程

## 主机发送

![image-20240125200826586](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240125200826586.png)

STM32的I2C设计是按照可变多主机模型设计的，所有挂在I2C上的设备（包括STM32），一开始都是从机。当STM32需要发送数据时，首先创造S（起始条件），STM32由从模式变为主模式。

EVx表示事件，事件就是一系列操作的组合：

- EV5：SB=1时该事件发生。SB=1表示起始条件已发送，当发送出起始条件时该位被置’1’。软件读取SR1寄存器后，写数据寄存器的操作将清除该位。
- EV6：ADDR=1时该事件发生。ADDR，7位地址模式时，当收到地址的ACK后该位被置’1’。EV6期间，数据寄存器非空，因为刚刚接收了从机的应答位。EV6结束时，TXE=1，即发送数据寄存器空。
- EV8-1：TXE=1、移位寄存器空、数据寄存器空时该事件发生。移位寄存器和数据寄存器都是空的，此时就要写入DR寄存器（准备数据）
- EV8：EV8-1数据准备完成后，立刻转移至移位寄存器进行发送，此时数据寄存器空，TXE=1（数据从DR转移完成），数据在移位寄存器里正在发送。
  - 之后每次准备发送一个字节数据时（写数据寄存器），EV8都会被清除（也就是数据2下面EV8长度没有与数据2对齐，空出了一截就是被清除了）
  - **EV8开始，代表DR移入移位寄存器开始发送；EV8结束，代表下一个待发送数据已经写入DR里了**
- EV8-2：TXE=1，BTF=1时该事件发生。BTF=1表示字节发送结束。
  - BTF何时置1？当移位寄存器移完了，找数据寄存器要数据时，TXE=1（数据寄存器没有数据），BTF置1。

## 主机接收

![image-20240125203510710](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240125203510710.png)

- EV7：移位寄存器开始往数据寄存器里移数据时，发生该事件。因为开始移了，则RXNE=1。该事件结束，说明数据转移完成并且已经从数据寄存器里取走了。
- EV7-1：在EV7的基础上，外加ACK=0和STOP=1。ACK=0，产生非应答；STOP=1，产生终止条件

# 4.程序用法

```c
typedef struct
{
    uint32_t I2C_ClockSpeed;
    uint16_t I2C_Mode;
    uint16_t I2C_DutyCycle;
    uint16_t I2C_OwnAddress1;
    uint16_t I2C_Ack;
    uint16_t I2C_AcknowledgedAddress;
}I2C_InitTypeDef;
```

- `I2C_ClockSpeed`：0\~100KHz为标准速度，100KHz\~400KHz为快速，不可超过400KHz
- `I2C_DutyCycle`：占空比（低电平/高电平），标准速度下固定为1:1，快速模式下该项才有意义
  - 为什么要有占空比？因为速度较慢时（标准速度下），低电平和高电平时间之比为1:1；快速模式下，由于传输速度很快，SCL的低电平时间要略微多于高电平，因为在低电平期间，要往SDA上准备数据（涉及若干位电平翻转），耗费的时间往往要比高电平期间读取数据的时间长，所以要保证低电平有充足的时间将数据准备好，不能准备了一半就开始读了。
