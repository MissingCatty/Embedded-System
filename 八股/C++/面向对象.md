# 1.面向对象的三大特性

## 1.1 继承

子类可以使用父类的属性和方法。

## 1.2 多态

父类指针可以指向子类对象。

# 2.虚函数和纯虚函数

**虚函数**

```cpp
virtual void work() { ...具体代码... }
```

父类自己写好了一段完整的代码。如果子类（子类）比较懒，不重写这个函数，编译器就会默认使用父类提供的这段代码。如果子类重写了，就用子类的新代码。

**纯虚函数**

```cpp
virtual void work() = 0;
```

父类根本就不提供任何实现代码。它只是在头文件里占个坑。任何继承它的子类，**必须、绝对、毫无条件地**亲手实现这个函数的代码，否则编译器会当场报错，拒绝编译子类。

# 3.区分重写（override）和隐藏（hide）

- 重写：子类函数与**父类的虚函数（加virtual）**同名同参数列表叫“重写/覆盖”
- 隐藏：子类函数与**父类的普通函数（不加virtual）**同名同参数列表叫“隐藏”

# 4.虚函数表

编译器会为**每个包含虚函数的类**创建一张虚函数表，表里包含该类的**<u>所有虚函数</u>及其<u>真实的物理地址</u>**。

```cpp
class Base {
public:
    virtual void func1() { /* 代码 A */ }
    virtual void func2() { /* 代码 B */ }
};

class Derived : public Base {
public:
    // 重写了 func1
    void func1() override { /* 代码 C */ } 
    // 没有重写 func2
};
```

**在 `.rodata` 段里，它们各自拥有一张表：**

- **`Base` 的虚函数表：**
  - 第 0 项：指向 `Base::func1` 的物理地址 (代码 A)
  - 第 1 项：指向 `Base::func2` 的物理地址 (代码 B)
- **`Derived` 的虚函数表（子类会复制父类的表，并替换掉自己重写的项）：**
  - 第 0 项：**被强行替换！** 指向 `Derived::func1` 的物理地址 (代码 C)
  - 第 1 项：原封不动，依然指向 `Base::func2` 的物理地址 (代码 B)

**怎么拿到这个表呢？**

- 当你 `new` 一个 `Base` 对象时，编译器把它的 `vptr` 初始化为指向 `Base` 的虚函数表。
- 当你 `new` 一个 `Derived` 对象时，编译器把它的 `vptr` 初始化为指向 `Derived` 的虚函数表。

# 5.静态绑定和动态绑定

- 静态绑定：编译器在扫描代码的时候只看指针的字面声明类型，不会看其指向的堆内存的实际对象

  **适用情况**：

  - 所有的普通函数调用
  - 函数重载 (Overload)：注意不是重写
  - C++ 模板

- 动态绑定：（晚绑定）在运行时才知道指向的堆内存对象是什么

  **适用情况**：同时满足三个规则

  - 必须是**虚函数**（
  - 必须通过【指针】或【引用】来调用
  - 指针/引用指向的必须是一个发生了【重写 (Override)】的子类对象
