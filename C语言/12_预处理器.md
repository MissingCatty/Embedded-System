预处理器功能：

- 删除注释
- 插入被`#include`指令包含的文件的内容
- 替换由`#define`定义的符号
- 确定部分代码是否应该根据一些条件编译指令进行编译

# 1 预定义符号

| 符号       | 样例值        | 含义                                      |
| ---------- | ------------- | ----------------------------------------- |
| `__FILE__` | "name.c"      | 进行编译的源文件名                        |
| `__LINE__` | 25            | 文件当前的行号                            |
| `__DATE__` | "Jan 31 1997" | 文件被编译的日期                          |
| `__TIME__` | "18:04:30"    | 文件被编译的时间                          |
| `__STDC__` | 1             | 如果编译器遵循ANSI C，其值为1；否则未定义 |

# 2 `#define`

```c
#define A (B)
```

`#define`可以将A替换成任何文本B（B是可缺省的）。如果B过长，可以分成多行，每一行最后加一个反斜杠`\`。如果存在字符串，当字符串被反斜杠隔开时，前后两个部分都要用双引号包含起来

```c
#define	DEBUG_PRINT	printf("File %s line %d:"	\
							"x = %d, y = %d",	\
							__FILE__, __LINE__,	\
							x, y
							)
```

# 3 宏

`#define`允许将参数替换到文本中

```c
#define SQUARE(x)	x * x
```

## 3.1 宏定义中的坑

```c
int a = 5;
SQUARE(a+1);
```

上述代码中，理想中的结果应该是`(5+1)*(5+1)=36`，但实际上执行的却是`5+1*5+1=11`

所以，在宏定义中，是严格执行宏定义的替换模板的，不会自动加上括号

## 3.2 字符串宏定义

如果想要实现这么一个功能：用宏定义一个固定格式的字符串，其中部分内容可变

- 方法一：宏定义嵌套（编译后不可变，需手动修改源程序）

```c
#define ID          "10011"
#define PASSWORD    "22162216"
#define MSG         "ID:" ID ", Password:" PASSWORD

printf(MSG);
```

- 方法二：用宏定义一个拼接字符串语句

```c
#include <string.h>

#define MSG(BUFFER, ID, PASSWORD)   sprintf(BUFFER, "ID:%s, Password:%s", ID, PASSWORD)

char    ID[] = "10011";
char    PASSWORD[] = "22162216";
char    result[50];

MSG(result, ID, PASSWORD);

printf(result);
```

## 3.3 宏和函数

现在有一个要求：动态分配一个类型未定的数组空间

使用宏可以如下实现：

```c
#define MALLOC(n, type)	(type *)malloc(n * sizeof(type))
```

由于宏执行的是直接的文本替换，所以此处当作一个固定格式的模板，在使用时再具体指明类型

而如果使用函数，是无法完成这样的任务的，必须在编写代码的时候就指定好数组的类型

**宏和函数的比较**

| 属性     |                  宏                  |                        函数                        |
| -------- | :----------------------------------: | :------------------------------------------------: |
| 代码长度 | 每次使用时，宏都生成一份代码进行替换 | 函数只需要定义一次，每次调用都会转到定义的地方执行 |
| 执行速度 |                  快                  |             函数调用和返回存在额外开销             |
| 参数类型 |              与类型无关              |                     与类型有关                     |

## 3.4 `undef`

如果一个现存的宏定义需要被重新定义，首先必须将旧的定义删除

```c
#define	name	stuff1
#undef	name
#define	name	stuff2
```

# 4 条件编译

**功能1**

有时候，一个程序中的调试代码在调试的时候，想让其编译执行；而在真正运行时，却想让其被忽略不被编译。此时，就需要用到条件编译。

条件编译的基本形式为：

```c
#if	constant-expression
	// ...
#endif
```

`#if`后面跟的是字面值常量，只有当**字面量非零**时，才执行夹在中间的就是调试代码

```c
#if	DEBUG
	printf("This is a Debug code");
#endif
```

如果想要开放调试，就`#define DEBUG 1`

**功能2**

在编译时选择代码的不同部分。

```c
#if	constant-expression1
	// ...
#elif	constant-expression2
	// ...
#elif	constant-expression3
	// ...
#endif
```

**功能3**

检查某个名字是否被宏定义

```c
#ifdef	symbol
#ifndef	symbol
```

**功能4**

防止同一个头文件被重复编译

假如`#include "b.h"`，`#include "c.h"`两个文件中都`#include "a.h"`，那么头文件`a.h`就被编译了两次。为了防止这种情况发生，需要在`a.h`中使用一个条件编译：

```c
#ifndef	_A_H
#define	_A_H
	// all statements in a.h
#endif
```

上面的代码的意思是，如果第一次编译该文件，宏`_A_H`还没有被定义，则定义`_A_H`并编译中间的代码。之后再编译该文件时，`_A_H`就已经被定义了，跳过中间所有代码的编译。

# 5 其他指令

`#error`：在编译时，遇到该指令，则停止编译，并输出后面的错误信息

```c
#if __STDC_VERSION__ < 199901L
    #error "C99 or higher is required to compile this program."
#endif
```
