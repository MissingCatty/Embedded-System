# 1 未初始化和非法指针

这段代码会出现什么问题？

```c
int *a;
*a = 12;
```

因为在声明指针时，未进行初始化。

如果指针被声明为`static`类型，则会默认初始化为0；如果像上述代码这样，被声明为自动指针变量，且未进行初始化，则该指针会是认意值。这意味着它的值可能是内存中的任何地址。

**未初始化指针的潜在风险**

- **程序崩溃**：未初始化指针可能指向一个非法的内存区域，访问该区域会导致程序崩溃。

- **数据损坏**：如果未初始化指针恰好指向了某个合法但不应该修改的内存区域，写入数据可能导致程序行为异常，甚至破坏系统中的其他数据。
- **安全漏洞**：未初始化指针可能被利用来进行攻击（如缓冲区溢出等），导致程序被恶意利用。

# 2 指针运算

## 2.1 与常量的运算

如果一个指向`float`类型（占4个字节）的指针`p`，执行操作`p=p+3`，此时`p`会指向什么位置？

是指向`float`类型中的第二个字节，还是指向`float`类型后的第一个字节？

答案是后者。在执行`p=p+3`时，实际上会执行`3*4`的操作，最后将`12`加到指针上去

所以，**`p+3`指的不是后移3个字节，而是后移3个`float`的大小**

## 2.2 指针之间的运算

**条件**：两个指针指向同一个数组中的元素

**结果**：两个指针相减结果的类型是`ptrdiff_t`（有符号整型），结果为**两个指针在内存中的距离**（**以数组元素大小为单位，而非字节为单位**）

如果两个指针不是指向同一个数组中的元素，则结果是未定义的。不在同一数组内的两指针相减，完全没有意义。
