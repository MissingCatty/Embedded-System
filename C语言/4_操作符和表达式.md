# 1 移位操作符

左移位`>>`和右移位`<<`，两个操作数必须是**整数**

标准规定：**无符号数**的移位是**逻辑移位**；**有符号数**的移位方式**取决于编译器**（可以写一个简单的测试程序）。所以，如果使用有符号数，在使用移位运算符时，不能保证在每个编译器上的结果都是一致的

所以，**一个程序使用了有符号数，那么它就是不可移植的**

---

**逻辑移位（Logical Shift）**

- **规则**：无论是左移还是右移，都将空出的位用**0**填充。

- **记忆技巧**：逻辑移位只关心“位”，不管符号。因此，你可以记住“**逻辑=零填充**”，表示移位后空出的部分总是补上0。这个方法适用于无符号数的操作。

**算术移位（Arithmetic Shift）**

- **规则**：

  - **左移**：与逻辑左移相同，空出的位补0。

  - **右移**：保持**符号位**（最高位）不变，其他空出的位填充为符号位的值（即符号扩展）。

- **记忆技巧**：算术移位需要考虑“符号”。你可以记住“**算术=符号保留**”，意味着在右移时会保留最高位来保持符号的一致性，适用于有符号数的操作。

# 2 位操作符

位操作符包括：按位与`&`、按位或`|`、按位异或`^`，操作数必须是**整数**

# 3 赋值

这段代码有什么问题？

```c
char ch;

while((ch = getchar()) != EOF)
{
    ...
}
```

`EOF`是一个整型值，同时`getchar`返回的也是一个整型值，而`ch`是一个`char`类型的变量，所以在执行赋值语句后`ch`会截断原来的数值

当`int`赋值给`char`时，如果数值超出了`char`的范围，只会**保留最低的8位**，高位会被截断

如果编译器将 `char` 解释为**无符号类型**（`unsigned char`），`EOF`（`-1`）赋值给 `char` 变量时会被截断和转换，变为 255，无法与 `EOF` 进行正确比较。

# 4 sizeof运算符

```c
sizeof(a = b+1)
```

这句代码并没有执行赋值操作，因为`sizeof`运算符是一个编译期运算符，在编译时计算操作数的类型大小，它只要考虑操作数的类型即可，无需计算值
