# 1 错误报告

当操作系统执行一些任务的时候，存在失败的可能。标准库函数在一个外部整型变量`errno`（包含在`errno.h`中）中保存错误代码后，再把错误信息传递给用户程序。

`perror`（包含在`stdio.h`中）函数简化了向用户程序报告错误的过程。

```c
void perror(char const *message);
```

如果`message`不为空，且指向一个非空字符串，则该函数就打印该字符串，然后跟一个冒号和空格，再打印出用于解释`errno`的错误信息。

下面这个例子，如果打开文件失败，就调用`perror`打印错误信息：

```c
#include <stdio.h>

int main() {
    int fd = open("non_existent_file.txt", O_RDONLY);
    if (fd == -1) {
        perror("Error opening file");
    }
    return 0;
}

/*
**	如果打开文件失败，则输出：
**		Error opening file: No such file or directory
*/
```

也可以使用`strerror`和`errno`的组合（`strerror`的功能是根据`errno`错误代码的值找到对应的详细错误信息，定义于`string.h`），实现上述功能：

```c
if(fd == -1) {
	printf("Error opening file: %s\n", strerror(errno));
}
```

# 2 输入输出概念

## 2.1 流

对于C程序而言，所有的IO操作只是简单的从程序移进或移出字节，这种字节流被称为流。程序只需要关心创建正确的输出字节数据，和正确解释输入的字节数据。

**流可以看作是数据的来源和去向**，即：对一个程序来说，从哪里接收数据和向哪里写入数据。

绝大多数的流是完全缓冲的，这表明读取和写入操作实际上是对一块缓冲区来回复制数据。

数据写到缓冲区时，并不会立刻输出到外部设备里，用于输出流的缓冲区只有当它被写满时，才会被刷新（输出）到设备或文件里。输入缓冲区类似。

---

**如果数据的最后一部分没有填满缓冲区，那岂不是一直留在缓冲区里了吗，导致不会写入外部设备里**

实际上，程序结束时或在特定条件下，缓冲区中的数据会被自动写入外部设备或文件，不会永久留在缓冲区里。即使缓冲区未被完全填满，也有机制保证剩余的数据被输出。

1. **程序结束时的缓冲区刷新**：当程序正常结束时，标准库会确保缓冲区的数据被刷新到外部设备或文件。即使缓冲区没有填满，系统也会在关闭文件或终止程序时清空缓冲区。
2. **手动刷新**：你可以通过显式调用 `fflush()`（C 语言中的函数）或其他类似函数来手动刷新缓冲区，确保即使缓冲区没有填满，数据也会被写入。
3. **自动刷新机制**：某些流（如标准输出）可能使用**行缓冲模式**，即遇到换行符时会自动刷新缓冲区。还有些流可能使用**无缓冲模式**，直接将每次写入的数据立即传递给外部设备。

因此，缓冲区中的数据不会被永远留在缓冲区里，而是通过这些机制最终写入外部设备或文件。

---

一个常见的调试策略是把一些 printf 函数的调用散布于程序中，确定错误出现的具体位置。

但是，这些函数调用的输出结果被写入到缓冲区中，**并不立即显示于屏幕上**。

事实上，如果程序失败，缓冲输出可能不会被实际写入，这就可能使程序员得到关于错误出现位置的不正确结论。这个问题的解决方法就是在每个用于调试的`printf`函数之后立即调用`ffush`，如下所示:

```c
printf("something or other");
fflush(stdout);
```

---

## 2.2 文本流和二进制流

流分为两类：

- 文本流
- 二进制流

这两类流本质上都是一连串的字节数据，但对其的解释方式和处理方式不同：

- 解释方式：文本流在使用时，会将其转为各种编码格式；而二进制流直接就是字节数据本身，一般不做处理，直接给程序使用

- 处理方式：文本流的“一行”的定义在不同的操作系统中，在MS-DOS系统中，一行以`\r\n`结尾；而在Unix系统中，以`\n`结尾

文本文件对于这些字节数据，会按照规定的编码格式对其进行编码，规定哪些字节应该解释为什么字符。

---

**例如**：系统读取到一串字节流 `0x41 E4 BD A0 E5 A5 BD`

根据 UTF-8 编码，系统将 `0x41` 解码为字符 `A`，将 `E4 BD A0` 解码为 `你`，将 `E5 A5 BD` 解码为 `好`

最终，系统返回字符序列 `A你好`

---

## 2.3 文件

`FILE`是一个声明于`stdio.h`的结构体类型，代表一个流（文件或设备）。如果同时激活几个流，每个流都有一个`FILE`与其关联。

`FILE` 是一个抽象的数据结构，实际包含了文件的**状态信息**，如**文件描述符**、**读写模式**、**当前文件指针位置**等。

在使用 `fopen()` 打开文件时，会返回一个指向 `FILE` 结构的指针。

`FILE` 实际上是对底层 I/O 操作的封装，提供了一个方便的接口来处理各种外部设备的输入/输出。

## 2.4 标准输入、标准输出、标准错误

对于每个C程序，运行时系统必须提供至少三个流（`stdin`，`stdout`，`stderr`），他们都是一个指向`FILE`结构的指针。

**标准输入**

- **定义**：标准输入是程序从外部接收输入数据的流，通常与**键盘**关联。（也就是说标准输入从键盘接收数据）
- **重定向**：可以将标准输入从键盘重定向到**文件**或其他数据流。

**标准输出**

- **定义**：通常用于将结果或信息输出到**终端**。程序可以通过标准输出将结果显示给用户。

- **重定向**：可以将标准输出从终端重定向到**文件**或其他数据流。

**标准错误**

- **定义**: 标准错误是专门用于输出**错误信息**的流，通常也与**终端**关联。标准错误流独立于标准输出，用于输出错误、警告和调试信息。

- **重定向**: 标准错误流也可以重定向到文件或其他流，与标准输出流分开。

## 2.5 标准IO常量

- `EOF`：文件末尾标志，通常比一个字符多几位，用于与字符进行区分
- `FOPEN_MAX`：规定了一个程序最多打开的文件数，至少为8，且包含了三个标准流

- `FILENAME_MAX`：一个整型值，规定了用于保存合法文件名的字符数组的最大长度

# 3 文件流IO

标准库函数提供了一系列接口，方便我们对文件进行IO操作。

1. 当访问文件时，程序会为每个被访问文件创建一个指向`FILE`结构体的指针；
2. 流（数据来源）通过调用`fopen`进行指定（需要设置访问的文件或设备、访问方式），指定无误后，初始化`FILE`结构
3. 根据需要对流进行读写
4. 调用`fclose`关闭流

IO函数以三种方式对流进行处理：字符、文本行、二进制数据。对于每组形式，都有特定的方式进行处理。

| 数据类型   | 输入            | 输出             | 描述                                                   |
| ---------- | --------------- | ---------------- | ------------------------------------------------------ |
| 字符       | getchar         | putchar          | 读取（写入）单个字符                                   |
| 文本行     | gets<br />scanf | puts<br />printf | 文本行未格式化的输入（输出）<br />格式化的输入（输出） |
| 二进制数据 | fread           | fwrite           | 读取（写入）二进制数据                                 |

---

**从文件流读取字符的具象理解**

在每个 `FILE` 结构体中，维护着一个**文件指针**（位置指针），它指向文件中的某个位置。最初打开文件时，文件指针通常位于文件的开头。

每次读取操作（如 `fgetc` 或 `fread` 等），都会从文件指针所指向的位置开始，并根据读取的数据量将文件指针**向后移动**，以指向下一个字节（或字符）。

当使用 `fgetc(FILE *stream)` 函数从文件流中读取一个字符时，文件指针从当前位置读取一个字节的数据，然后将文件指针向后移动一个字节。读取的数据将作为返回值返回。

（对于多字节字符集，需要多次调用`fgetc`获取多个字节然后编码成字符）

---

## 3.1 打开流

`fopen`函数打开一个特定的文件，并把一个流与这个文件相关联

```c
FILE *fopen(char const *name, char const *mode);
```

第一个参数是文件或设备名，第二个参数是访问模式

|        | 读取 | 写入                | 读写                                       | 添加 |
| ------ | ---- | ------------------- | ------------------------------------------ | ---- |
| 文本   | r    | w（源文件内容清空） | r+（文件必须存在）<br />w+（文件可以存在） | a    |
| 二进制 | rb   | wb                  | rb+<br />wb+                               | ab   |

- 如果一个文件打开用于**读取**，那么它原先必须就是**已经存在的**
- 如果一个文件打开用于**写入**
  - 如果原先已经存在，就往里面写
  - 如果不存在，就先创建这个文件，再往里写

如果`fopen`函数执行成功，就返回一个指向`FILE`的指针，该结构体代表新建的流。如果执行失败，就返回`NULL`

---

**文本模式和二进制模式的区别**

**文件内容的处理**

- 在 **文本模式** 下，可能会对某些特殊字符进行转换处理
  - 在 **文本模式** 下打开文件时，读写时可能会自动进行这些换行符的转换
    - 从文件中读取时：操作系统会把 `\r\n` 转换成 `\n`（Windows 环境下）
    - 写入文件时：在 Windows 中，如果写入的是 `\n`，它会自动转换为 `\r\n`
  - 在文本模式下，有时会将文件末尾标记 `EOF` 特殊处理，尤其是 Windows 系统会在文本文件的末尾插入一个 `Ctrl+Z`（ASCII 26，`EOF`）来标记文件的结束
- 在 **二进制模式** 下，文件被认为是纯字节序列，任何数据都不做处理。换言之，文件中的每一个字节都会原封不动地读入或写出。

**使用场景的差异**

- **文本模式** 主要用于处理纯文本文件，如 `.txt` 文件、源代码文件（`.c`、`.java` 等），它们包含可以直接被人阅读和编辑的文本数据。
- **二进制模式** 适用于处理二进制文件，如图片文件（`.jpg`、`.png`）、音频文件（`.mp3`、`.wav`）、视频文件（`.mp4`）、压缩文件（`.zip`）、数据库文件等。这些文件的内容不仅仅是人类可读的字符数据，还包含特殊的字节序列。

---

## 3.2 关闭流

```c
int fclose(FILE *f);
```

对于输出流，在关闭流之前会刷新缓冲区，将缓冲区中剩余的数据输出到文件或设备中。

执行成功，返回`0`；否则，返回`EOF`

## 3.3 字符IO

字符输入是由`getchar`函数家族执行的

```c
int fgetc(FILE *stream);
int getc(FILE *stream);
int getchar(void);
```

前两个需要指定输入字符的流，最后一个始终由标准输入读入数据。每个函数从流中读取下一个字符，并作为函数的返回值。如果流中不存在更多的字符，就返回`EOF`

注意：返回值是`int`而不是`char`，因为`EOF`不只有8位，它区别于其他的只有8位的ASCII字符，所以在读取字符时，就很好区分ASCII字符和`EOF`

字符输出是由`putchar`家族执行的，作用是把单个字符写入流

```c
int fputc(int character, FILE *stream);
int putc(int character, FILE *stream);
int putchar(int character);
```

如果写入失败（比如流已经关闭），就返回`EOF`

---

**为什么同样的功能要设置两个参数相同的函数`fgetc`和`getc`、`fputc`和`putc`？**

`getc`和`putc`都是真正的函数，而`fgetc`、`fputc`、`getchar`、`putchar`都是通过`#define`定义的宏。

前几节介绍过函数和宏的区别，即：宏在代码长度上比函数要长（直接复制替换），但在执行效率上一般要优于函数（函数需要经过入栈、跳转、返回、函数栈的创建和销毁等操作）

设置两种实现方式，就提供给使用者两种选择，择情况而定

（但其实使用哪种方法，对程序的影响很小，不必刻意选择使用的方法）

---

**撤销字符IO**

假设，需要连续读入一连串数字，知道非数字为止，但此时需要保留这个字符，作为下一次读取的开始，就可以使用撤销函数

```c
int ungetc(int character, FILE *stream);
```

函数把先前读入的一个字符返回到流中（文件指针向前退回一个字节）

## 3.4 未格式化的行IO

直接读取字符串，简单来说就是：把文件中的内容不做处理，直接读到一个字符串里。而不是将文件中出现的数字，读到一个`int`变量里。

`puts`和`gets`家族的函数用于操作字符串，而不是字符。

```c
char *fgets(char *buffer, int buffer_size, FILE *stream);
char *gets(char *buffer);
```

`fgets`从指定的流中读取字符串到`buffer`里，有两种停止方式：

- 读的字符数不超过`buffer_size-1`时，遇到换行符停止（`NULL`会自动添加到末尾，使形成一个字符串）
- 读满`buffer_size-1`时停止

每次调用`fgets`都从上一次停止的地方继续读取。

如果某一次调用`fgets`，什么都没读到，就到达了文件尾，就返回`NULL`；否则，返回`buffer`指针

```c
int fputs(char const *buffer, FILE *stream);
int puts(char const *buffer);
```

`buffer`为要写入文件的字符串，逐字符写入

`fputs`函数在遇到换行符时，不会像`fgets`那样停止，换行符就是一个普通的字符，有几个就往文件里写几个。所以，该函数可以一次往文件里写好几行（`buffer`包含多个换行符），也可以不写满一行（`buffer`不包含换行符）

---

**`gets`和`puts`**

`gets`：不指定一次性读取的最大长度`buffer_size`，一直读到换行符为止。这就意味着，如果`buffer`长度小，实际读入的一行字符串很长，多余的部分会接在`buffer`后面的内存继续读入，导致内存泄漏。所以，该函数一般用的比较少。

`puts`：当写入一个字符串时，总是会在末尾添加一个换行符。

---

## 3.5 格式化的行IO

### 3.5.1 `scanf`家族

```c
int fscanf(FILE *stream, char const *format, ...);
int scanf(char const *format, ...);
int sscanf(char const *string, char const *format, ...);
```

- `fscanf`的输入源是指定的流`stream`
- `scanf`的输入源是标准输入流`stdin`
- `sscanf`的输入源是字符串`string`，遇到`NULL`停止

省略号是可变长的指针类型的参数列表

`fscanf`整个执行的流程是：

1. 从文件指针当前位置开始读取
2. 将文件流中的字符，于格式化字符串逐一对比，每读取一个字符，都会尝试解析并存储它。如果成功解析，则继续匹配下一个格式符号；若不匹配，则停止读取。
3. 如果所有的字符都能够成功解析，或者解析过程中出现不匹配的情况，此时返回成功读取的数据项数；否则，文件结束，则会返回`EOF`，表示读取失败或到达文件末尾

注意：

- 在`format`中，如果遇到空白字符（空格、制表符、换行符），会**自动跳过所有空白字符**
- 在`format`中，当其他的任何字符（除空白字符、格式字符外）出现时，下一个输入字符必须与其匹配，否则，就不再读取直接返回

---

例子：假设文件中有如下两行

```c
Alice   21  female
Bob 23  male
```

```c
#include <stdio.h>

int main() {
    char filename[] = "../file.txt";
    FILE *file = fopen(filename, "r");

    if (!file) {
        perror("Open file error");
        return 1;
    }

    char name[10];
    int age;
    char gender[10];
    
    /*
    **	这里务必写成==3，因为失配或文件结束时，返回值可能是-1（文件结束）,0（无匹配）,1,2
    */
    while (fscanf(file, "%s %d %s", name, &age, gender)==3) {
        printf("姓名：%s\t年龄：%d\t性别：%s\n", name, age, gender);
    }

    fclose(file);
}
```

第一次执行`fscanf`时，根据`format`逐步读到一个字符串、一个`int`、一个`float`，完全匹配，第一次调用结束，文件指针处于第二行开头

之后，读取第二行的内容

---

**在Clion中文件路径的坑**

Clion中生成的exe文件不在项目根目录里，在`cmake-build-debug`目录里

所以，如果要对某一文件进行读写，必须记住读写文件**对于exe文件的相对位置**，**而不是对于项目路径的相对位置**，或者直接使用绝对路径

---

**`scanf`格式码**

| 格式码                 | 数据类型            | 说明                                                       |
| ---------------------- | ------------------- | ---------------------------------------------------------- |
| `%d`                   | `int`               | 读取一个**十进制整数**，正负均可                           |
| `%u`                   | `unsigned int`      | 读取一个**无符号十进制整数**                               |
| `%f`                   | `float`             | 读取一个**浮点数**，存储为 `float`                         |
| `%lf`                  | `double`            | 读取一个**双精度浮点数**（`double`）                       |
| `%c`                   | `char`              | 读取一个**字符**                                           |
| `%s`                   | 字符数组            | 读取一个**字符串**，直到遇到空白符（空格、制表符、换行符） |
| `%[...]`               | 字符集              | 读取在集合中的字符                                         |
| `%o`                   | `unsigned int`      | 读取一个**八进制整数**                                     |
| `%x`                   | `unsigned int`      | 读取一个**十六进制整数**                                   |
| `%p`                   | 指针                | 读取一个**指针地址**                                       |
| `%n`                   | `int*`              | **记录读取的字符数**（不消耗输入）                         |
| `%e`, `%E`, `%g`, `%G` | `float` 或 `double` | 读取科学计数法表示的浮点数                                 |
| `%l`                   | 修饰符              | 用于修饰其他格式符，表示长整型、长双精度等                 |

### 3.5.2 `printf`家族

```c
int fprintf(FILE *stream, char const *format, ...);
int printf(char const *format, ...);
int sprintf(char *buffer, char const *format, ...);
```

这三个函数的返回值，是实际打印的字符数

---

**`sprintf`中的注意点**

`sprintf`函数的功能是往内存中的一个内存中的数组写数据，但又没有给出数组的长度，所以如果读入过多，就容易造成内存泄漏。

为了避免这种问题的出现，必须在使用前对`buffer`的长度和格式化后字符串的长度进行比较。

---

**常见的 `printf` 格式化代码**

| 格式码     | 数据类型            | 说明                                                         |
| ---------- | ------------------- | ------------------------------------------------------------ |
| `%d`       | `int`               | 以**十进制整数**的形式输出（带符号）                         |
| `%i`       | `int`               | 与 `%d` 相同，输出**十进制整数**                             |
| `%u`       | `unsigned int`      | 以**无符号十进制整数**的形式输出                             |
| `%f`       | `float`             | 以**浮点数**形式输出（默认精度为小数点后 6 位）              |
| `%lf`      | `double`            | 以**双精度浮点数**形式输出                                   |
| `%e`, `%E` | `float` 或 `double` | 以**科学计数法**形式输出（如 `1.23e+05`）                    |
| `%g`, `%G` | `float` 或 `double` | 自动选择 `%f` 或 `%e`（或 `%E`），以较简洁的方式输出         |
| `%c`       | `char`              | 输出一个**字符**                                             |
| `%s`       | 字符串（`char*`）   | 输出一个**字符串**，直到遇到 `\0`                            |
| `%p`       | 指针                | 以**指针地址**形式输出                                       |
| `%o`       | `unsigned int`      | 以**八进制整数**的形式输出                                   |
| `%x`, `%X` | `unsigned int`      | 以**十六进制整数**形式输出（`%x` 使用小写字母，`%X` 使用大写字母） |
| `%%`       | 无                  | 输出一个**百分号字符 `%`**                                   |

---

**宽度和精度修饰符**

| 说明 | 例子    | 结果                                                         |
| ---- | ------- | ------------------------------------------------------------ |
| `%-` | `%-10d` | **左对齐**，宽度为 10，右边填充空格                          |
| `+`  | `%+d`   | 输出**正负号**，正数带 `+`，负数带 `-`                       |
| `0`  | `%010d` | **补零**，在不足的情况下左侧补 `0`                           |
| `.`  | `%.2f`  | **精度**，控制浮点数的小数点后位数（如 `%.2f` 输出小数点后 2 位） |
| 数字 | `%5d`   | **最小宽度**，表示字段的最小宽度，右边自动补空格             |

# 4 二进制IO

```c
size_t fread(void *buffer, size_t size, size_t count, FILE *stream);
size_t fwrite(void *buffer, size_t size, size_t count, FILE *stream);
```

读写的最小单位是**元素**，而非**字节**。`size`指定了一个元素由多少个字节组成，`count`指定了读写的**元素数**。

返回实际写入（读取）的元素个数，如果比`count`小，即过程中发生了错误。

```c
#include <stdio.h>

/*
**	程序：先向文件中写入，再读取写入的内容
*/
int main() {
    char filename[] = "../file.bin";
    FILE *file = NULL;

    file = fopen(filename, "wb");
    if (!file) {
        perror("Open file error");
        return 1;
    }

    // 将内容写到二进制文件
    int buffer_write[10] = {1, 2, 3, 4, 5, 6, 7, 8, 9, 0};
    int elementsWrite = fwrite(buffer_write, sizeof(int), 10, file);
    printf("向文件中写入%d个元素\n", elementsWrite);

    // 关闭文件
    fclose(file);

    // 从文件中读取数据
    file = fopen(filename, "rb");
    if(!file){
        perror("Open file error");
        return 1;
    }

    int buffer_read[20];
    int elementsRead = fread(buffer_read, sizeof(int), 10, file);
    printf("从文件中读取%d个元素：", elementsRead);
    for (int i = 0; i < elementsRead; i++) {
        printf("%d ", buffer_read[i]);
    }

    fclose(file);
}
```

# 5 刷新和定位函数

**刷新函数**

强制将输出流缓冲区中所有的数据进行物理写入，不管是否已经写满（缓冲区只有写满时才往外设输出数据，批写入）

```c
int fflush(FILE *stream);
```

**定位函数**

在程序向文件写入时，每写入一个字节，文件指针自动往后移一位，所以数据是依次写入的，且不需要人为控制文件指针。

但如果想在文件任意位置进行对数据进行访问，就需要先将文件指针移动到这个位置。

```c
long ftell(FILE *stream);
int fseek(FILE *stream, long offset, int from);
```

`ftell`返回流`stream`当前文件指针的位置，即：**距离文件起始位置的偏移字节数**。

`fseek`函数中，参数`from`表示从哪里开始数`offset`个字节：

- `from=SEEK_SET`时，从**流的起始位置**起，`offset`必须为一个**非负数**
- `from=SEEK_CUR`时，从**流的当前位置**起，`offset`可正可负，但应该在文件起始位置之后
- `from=SEEK_END`时，从**流的末尾位置**起，`offset`可正可负，如果为正，就被定位到文件尾的后面

---

**不同定位的几种情况**

- 定位在末尾位置的后n个字节，之后写入k个字节：文件自动扩展以包含新的数据。如果没有恰好从文件尾开始写入，而是又往后数几个字节再写入，中间没写的部分会产生**数据空洞**，如：

  一个文件中原来包含5个字节`0x01,0x02,0x03,0x04,0x05`，此时将指针定位到距离起始位置10个字节的地方，也就是距离文件末尾5个字节的位置，开始写入`0x06,0x07`两个数，写入后文件变为`0x01,0x02,0x03,0x04,0x05,0,0,0,0,0,0x06,0x07`，中间产生的**数据空洞，在大部分系统上为0**

- 定位在文件起始位置之前是非法的，标准库不支持

---

**注：从定位位置写入执行的是覆盖操作**
