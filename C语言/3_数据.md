# 1 数据类型

## 1.1 整型

### 1.1.1 长整型和短整型

长整型（long）短整型（short）

标准规定：**长整型至少和整型一样长，整型至少和短整型一样长**

标准规定的变量的最小位数如下：

|        类型        | 最小位数（至少） |     范围     |
| :----------------: | :--------------: | :----------: |
|        char        |       7位        |    0~127     |
|    signed char     |       8位        |   -127~127   |
|   unsigned char    |       8位        |    0~255     |
|     short int      |       16位       | -32767~32767 |
| unsigned short int |       16位       |   0~65535    |
|        int         |    16位/32位     |      -       |
|    unsigned int    |    16位/32位     |      -       |
|      long int      |       32位       |     ...      |
| unsigned long int  |       32位       |     ...      |

`short int` **至少**是16位，`long int` **至少**是32位，**int是至少是16还是32位由编译器的决定**

头文件`limits.h`规定了各种整数类型的特点：

![image-20240924153409152](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20240924153409152.png)

`char`为缺省的字符类型，至于缺省类型为`signed char`还是`unsigned char`，取决于编译器。

**只有当程序所使用的 char型变量的值位于signed char 和 unsigned char的交集中，这个程序才是可移植的。**

这句话的理解如下：

---

```c
#include <stdio.h>

int main() {
    char c = 200;
    printf("%d\n", c);
    return 0;
}
```

在这个例子中，`char c = 200;` 将一个值为 200 的整数赋给了 `char` 类型的变量 `c`。现在来看这个程序在不同平台上的行为：

1. **如果 `char` 被当作 `signed char` 处理**（范围是 -128 到 127），则 200 超出了 `signed char` 的取值范围。根据整数溢出的规则，200 会被当作负数处理。最终的结果可能是 `-56`（因为 200 - 256 = -56）。

   输出：`-56`

2. **如果 `char` 被当作 `unsigned char` 处理**（范围是 0 到 255），那么 200 在取值范围内，没有问题。

   输出：`200`

因此，为了保证可移植性，`char` 变量的值应该在 0 到 127 之间，这样在任何平台上（无论 `char` 是 `signed` 还是 `unsigned`），程序的行为都是一致的。

---

根据上述例子，**如果在声明变量时，就把该变量声明为`signed char`或者`unsigned char`，那么就可以提高这类程序的可移植性。**

然而，在某些机器或硬件架构上，处理带符号类型（`signed char`）的运算可能更高效，而如果强制将其改为无符号类型（`unsigned char`），可能会影响性能。同时，库函数中许多字符变量被声明为`char`类型，如果把参数显式声明为`unsigned char`或者`signed char`，会存在兼容性问题。

综上，显式声明为`signed`或`unsigned`既有好处也有坏处，所以最佳的方法就是把`char`型变量的值限制在`signed`和`unsigned`**交集内（一般是0~127)**，这样既可以获得最大程度的可移植性，同时又不牺牲效率。

### 1.1.2 整型字面值

- 十进制整型值（最正常的）：123、102、...
- 八进制整型值（以`0`开头）：077、0x12、...
- 十六进制整型值（以`0x`开头）：0x7b、0xfff、...

八进制或十六进制字面值可能的类型是`int`、`unsigned int`、`long`或`unsigned long`。

如果想指定某个字面值被解释为特定的数据类型，就可以在字面值的末尾加`l`或`u`。如现有字面值`0xff`

- 指定为`long`：`0xffl`
- 指定为`unsinged`：`0xffu`
- 指定为`unsigned long`：`0xfful`或`0xfflu`

**在缺省的情况下，字面值的类型默认为：上述四种类型中<u>取值范围最小</u>且<u>可以容纳该字面值</u>的类型**

### 1.1.3 字符常量

字符常量的类型总是`int`

## 1.2 枚举类型

```c
enum Day {Monday=1, Tuesday=2, Wednesday}

Day day1, day2;
```

枚举类型中的`Monday`符号名被视为**整型常量**，声明为枚举类型的变量`day1`实际上是**整数类型**。

## 1.3 浮点类型

浮点数类型包括：`float`、`double`、`long double`类型

`ANSI`标准规定：`long double`至少和`double`一样长，而`double`至少和`float`一样长。标准规定了一个最小范围：所有浮点类型至少能够容纳从$10^{-37}$到$10^{37}$之间的任何值。

浮点数的字面值总是写成十进制的形式，且必须带小数点或一个指数：`3.14`、`1E9`、`25.`、`.5`。缺省时默认为`double`类型，如果末尾跟一个`L`或`l`，则为`long double`；如果末尾跟一个`F`或`f`，则为`float`。

## 1.4 指针

**字符串常量**本质就是一个**指针常量**，因为在表达式中**字符串常量的值就是字符串首字符地址的值**，**而不是字符本身**。所以，只允许字符串常量给指针赋值，而**不允许字符串常量赋值给一个字符数组**，因为字符串常量本质是一个直接指针值，例如：

```c
char s[10];
s = "123";			// 错误，字符串常量无法给字符数组赋值

char *s1 = NULL;
s1 = "123";			// 正确
printf("%s", s1);
```

数组名本质是一个指针常量`char *const p`，即`p`所指向的地址不能变，所以使用字符串常量给字符数组赋值时，本质是企图修改指针**常量**的值，这一定是不被允许的。

# 2 typedef

```c
typedef char* ptr_to_char
```

需要注意的是，`#define`通常也能实现同样的功能：

```c
#define CHAR char
typedef char mychar;

int main() {
    CHAR a = 's';
    mychar b = 's';
    printf("%c%c", a, b);
}
```

但应该尽量使用`typedef`而不是`#define`，因为**`#define`无法处理指针类型**，如：

```c
#define ptr_to_char char*
ptr_to_char a,b;
```

此时只有第一个`a`被声明为`char *`，`b`只能被声明为`char`

# 3 常量

被声明为`const`变量的参数，**必须在声明时就给定一个值**，函数的形参除外，其值是在被调用时绑定。

**指针常量**

按照名字来理解：作为常量的指针，即所代表的地址无法被修改，不能指向其他的地址，但可以修改指向地址中的内容。

```c
int a = 10, b = 11;
int* const p = &a;	// int*表示整型指针， const p表示指针指向的地址无法修改

// p = &b;			// 错误，无法修改所指向的地址
*p = 12;			// 正确，可以修改指向地址中的内容
```

**常量指针**

按照名字来理解：指向常量的指针，即所指向的内容无法通过该指针修改

虽说是常量指针，但意思就是不能通过该指针修改其内容，而不是说指针指向的一定要是一个常量

```c
int a = 10;

int const *p = &a;		// 常量指针
int *q = &a;			// 普通指针
*q = 11;				// 修改指向的值
```

由此可见，常量指针仅仅限制被修饰的指针不能修改指向的内容，但对于其他普通指针，还是可以修改的

**注意**

被`const`修饰的值仍然是一个变量，只不过该变量无法被修改而已。当需要使用常量时，如定义一个数组`char [size]`时，`size`最好使用`#define`进行确认，而不是用`const`变量，因为变量无法定义一个数组。

# 4 链接属性

标识符的链接属性 规定了**多个文件中定义的多个同名变量在编译链接时的处理规则**。

链接属性一共有三种：`external`、`internal`、`none`

属于`none`的标识符总是被当作单独的个体，即：该标识符的多个声明被当作独立的不同实体

属于`internal`的标识符在同一个源文件内的所有声明中都指向同一个实体

属于`external`的标识符在多个源文件都表示同一个实体

**External Linkage**:

- **默认情况下**，**函数**和**全局变量**具有**外部链接属性**。它们可以在不同的源文件中被引用。
- 通过在文件的任何地方声明它们，就可以在其他文件中使用。

**Internal Linkage**:

- 使用 **`static` 关键字**定义的全局变量或函数具有**内部链接属性**。它们只能在定义它们的文件内可见。

```c
static void myFunction() {
    // 只能在此文件中调用
}
```

使用 `static` 关键字定义的全局变量或函数的作用域仅限于定义它们的文件。这意味着它们不能在其他源文件中被引用或访问。这样做可以**避免命名冲突，确保这些变量和函数仅在特定文件中使用**。

**No Linkage**:

- **局部变量**（在函数内部定义的变量）**没有链接属性**，它们只能在定义它们的块内可见。

```c
void myFunction() {
    int localVar; // 只有在myFunction内可见
}
```

# 5 存储类型

变量的**存储类型决定变量何时创建何时销毁以及它的值将保持多久**。有三个地方可以用于存储变量：**普通内存**、**运行时堆栈**、**硬件寄存器**。在这三个地方存储的变量具有不同的特性。

**静态内存**：全局变量的存储位置总是在静态内存中，全局变量的<u>生命周期</u>是整个程序，普通的全局变量的作用域为整个程序，而被`static`修饰的全局变量作用域只为当前的文件。**静态变量（无论全局还是局部）永远是在程序开始执行之前（即main函数之前）被创建和初始化**。**具有`external`链接属性的实体总是静态存储类型**。

**堆栈**：代码块中声明的变量缺省的存储类型为`auto`，存储于堆栈中。只有当程序运行到声明该局部变量的位置时，才在堆栈中创建。

**寄存器**：用关键字`register`声明自动变量，存放在寄存器里，**访问效率更高**。被声明的部分变量存放在寄存器里，之所以说是部分，因为如果有太多的变量被声明为`register`类型，编译器只选取前几个存储在寄存器中，其余的按照普通变量处理

注：函数的形参不能声明为`static`，因为实参传值给函数形参的过程为：

1. 实参入调用函数的那段程序所拥有的堆栈
2. 为函数单独分配另一个栈帧
3. 复制实参的值到函数的栈帧上

由此可见，函数的形参是在堆栈上的，如果声明为`static`，就放到静态内存里了，显然是不被允许的
