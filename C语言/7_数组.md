# 1 数组名

实质：指针常量，即所指向的地址无法改变

然而，有一种情况，数组名不会被当作指针常量，即：`sizeof`运算

在`sizeof`运算中，会返回该数组名所代表的数组长度（占用的字节数）

# 2 下标

```c
int a[10];
int *p = a+3;

p[-1]
```

上面的代码中，`p`指针指向数组`a`中的下标为3的元素，`p[-1]`等价于`*(p-1)`，所以指向的是`a[2]`，即当前指针的前一个元素

# 3 数组和指针的区别

当声明一个数组时，编译器根据数组声明的大小为其分配空间，而声明指针时，只为当前指针变量本身保留内存空间，即：分配一个空间存放指针所指向的地址的值

```c
int a[5];
int *b;
```

如上两种声明方式，可见`a`和`b`都未进行初始化，然而`a`在声明之后，就已经指向了一个固定的地址，而`b`是一个自动变量，未初始化时其所指向的地址是未知的，所以非法。

另外，不可对`a`执行`++`操作，因为数组名是一个指针常量，无法被改变

# 4 数组做形参

**一维数组**

```c
int func(int *a);
int func(int a[]);
```

**二维数组**

```c
int func(int (*matrix)[10]);
int func(int matrix[][10]);
```

为什么第二维必须要给出？

因为内存是线性存储的，二维数组的元素在内存中线性存储的，所以比如访问数组的某一个元素`matrix[2][3]`，编译器要计算`base_address + 2*col + 3`，如果不给出`col`，即第二维的大小，那就无法进行坐标的转换

# 5 数组的初始化

- 静态数组：未进行初始化时，元素都为0
- 自动数组：数组元素都是自动变量，未初始化时，值是任意数

如果数组的初始化是在代码块内部，那么每当程序执行到该代码块时，就都需要进行一次初始化。

所以，如果该代码块需要被频繁执行，就可以将该数组声明为`static`类型，其初始化只在程序运行之前一次完成

```c
int a[3] = {1,2}
```

上述初始化长度为3的数组，但给的初值只有两个，那么剩下的就自动赋值为0

**字符数组的初始化**

```c
char str[] = {'H','e','l','l','o',0};
char str[] = "Hello";
char *str = "Hello";
```

前两个是创建一个字符数组；

第三个是创建一个字符串，但这样创建存在一些问题：

**字符串字面量是只读的**：在这种情况下，`"Hello"` 是存储在**静态存储区**（通常是只读存储区）中的字符串字面量。虽然 `str` 是一个指向 `char` 的指针，但它指向的是只读的**内存区域**。因此，**尝试修改这个字符串的内容会导致未定义行为**，甚至可能导致程序崩溃

```c
str[0] = 'h';  // 这是未定义行为，可能会导致崩溃
```

正确的处理方法是：

```c
const char *str = "Hello";
```
