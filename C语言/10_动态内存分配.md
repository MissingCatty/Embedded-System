# 1 malloc和free

用于执行**动态内存分配和释放**，包含在`<stdlib.h>`中

函数维护一个**内存池**，调用`malloc`函数时，从内存池冲取出一块适合的内存，并向程序返回一个之指向这片内存的指针。此时，这块内存并没有进行初始化。

当这块内存之后不被使用时，须调用`free`函数释放这块区域，归还给内存池。

**函数原型**

```c
void *malloc(size_t size);
void free(void *p);
```

- `malloc`的参数`size`为需要的**字节数**，返回值为指向被**分配内存的起始地址**

- `malloc`分配的是一块连续的内存
- `malloc`实际上分配的内存可能会比请求的稍微多一点，具体由编译器决定

**内存池为空**

内存池为空时，`malloc`向操作系统申请另一块内存，并在这块内存上分配空间。

如果操作系统无法向`malloc`提供更多的空间时，`malloc`就会返回`NULL`

**free的参数**

必须是之前使用`malloc`、`calloc`或`realloc`返回的非空指针，否则没有任何效果

# 2 calloc和realloc

**函数原型**

```c
void *calloc(size_t num_elements, size_t element_size);
void realloc(void *ptr, size_t new_size);
```

- `calloc`在返回指向内存的指针之前**把被分配的内存全部初始化为0**
- `calloc`的参数，`num_elements`：元素的数量，`element_size`每个元素的字节数
- `realloc`修改原先已经分配的内存块大小
  - 扩大：新增加的内存直接添加到原先内存块的后面
    - 如果原先内存块的后面被占用，则分配另一块对应大小的内存，同时将原先的内容复制到新的内存上。所以，**使用`realloc`之后，原先的指针`ptr`就不能用了，必须使用`realloc`返回的指针**。
  - 缩小：超出的尾部直接被拿掉

# 3 使用动态内存分配

**【实例1】：读取一列整数，并按照升序排序**

```c
#include <stdio.h>
#include <stdlib.h>  // 包含malloc和free函数的头文件

void sort(int arr[], int n) {
    int temp;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                // 交换两个元素
                temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

int main() {
    int n;

    // 读取整数个数
    printf("请输入整数的个数：");
    scanf("%d", &n);

    // 动态分配内存
    int *arr = (int *)malloc(n * sizeof(int));
    if (arr == NULL) {
        printf("内存分配失败！\n");
        return 1; // 返回错误代码
    }

    // 读取整数
    printf("请输入 %d 个整数：\n", n);
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }

    // 调用排序函数
    sort(arr, n);

    // 输出排序后的结果
    printf("升序排列后的整数：\n");
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }

    // 释放动态分配的内存
    free(arr);

    return 0;
}
```

# 4 常见的动态内存错误

- 对`NULL`指针进行解引用
- 对分配的内存操作时越界
- 释放并非由动态内存分配的内存
- 试图释放动态分配的内存的一部分
- 动态内存被释放后继续使用，此时编译器并不会对此进行检查
