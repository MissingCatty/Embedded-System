# 1 翻译和执行

C语言源文件到程序运行需要两个步骤，**翻译和执行**

## 1.1 翻译

翻译阶段包含**编译和链接**两个步骤

- 每个源文件通过**编译**，转为目标文件
  - 预处理器处理：预处理在源文件上执行文本操作（用实际值代替`#define`指定的符号、读入由`#include`指定的文件）
  - 解析：判断语句的意思，生成错误和警告信息
  - 产生目标代码
  - 优化：如果在编译程序时指定需要进行目标代码优化，优化器会开始优化代码
- 各个目标文件与通过链接器，与库文件**链接**成完整程序

编译和链接在UNIX系统中，需要调用执行C编译器，C编译器被称为`cc`

1. 编译并链接一个**完全包含于一个源文件**的C程序

   ```shell
   cc main.c
   ```

   最终会产生一个`main.out`可执行程序，中间会产生一个`main.o`的目标文件，但会在**链接过程后自动删除**

2. 编译**多个源文件**

   ```shell
   cc main.c sort.c lookup.c
   ```

   当编译的**源文件超过一个**时，**目标文件不会被删除**。所以下次编译时，**可以只对修改后的源文件进行重新编译**

3. 编译**一个源文件**，并**与现存目标文件链接**

   ```shell
   cc main.c sort.o lookup.o
   ```

4. 编译一个源文件，**暂时不链接**

   ```shell
   cc -c main.c
   ```

5. 编译几个源文件，并为每个源文件产生一个目标文件

   ```shell
   cc -c main.c sort.c lookup.c
   ```

6. **链接**几个目标文件

   ```shell
   cc main.o sort.o lookup.o
   ```

## 1.2 执行

可执行文件的执行也分为几个阶段：

- 程序载入到内存中
  - 在宿主环境（带有操作系统的环境）中，一般由操作系统自动加载到内存里
  - 独立环境中，需要手动或者直接加载到只读内存ROM里完成
- 开始执行
  - 程序使用**运行时堆栈（stack）**，用于存储**函数的局部变量和返回地址**。
  - 使用**静态（static）**内存，用于存储静态内存中的变量，在整个程序运行过程中，一直保留它们的值
- 程序终止
