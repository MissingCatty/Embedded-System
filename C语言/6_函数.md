# 1 私有变量

函数的接口和具体实现需要分开写，接口放在头文件中，具体实现放在源文件中

如果源文件中有部分变量不想被用户直接访问，可以将其声明为`static`

```c
#define MAX_USERS		100
#define NAME_MAX_LEN	10

static char user_names[MAX_USERS][NAME_MAX_LEN]		// 姓名表不可被访问

char* check_name(int num)
{
	// ...
}
```

之所以声明为`static`是因为：**静态变量只能从该源文件中被访问，不能从其他文件引用**

# 2 可变参数列表

通过宏实现，需要用到一个类型`va_list`和三个宏`va_start`、`va_arg`和`va_end`，这四个都定义于头文件`stdarg.h`中。

```c
/*
**	使用可变参数，计算不定量输入的浮点数的平均值
*/
void my_print(int count, ...) {
    va_list args;
    double sum = 0;

    va_start(args, count);

    for (int i = 0; i < count; ++i) {
        sum += va_arg(args, double);
    }

    va_end(args);

    printf("average = %f\n", sum);
}
```

`va_start`：第一个参数是列表名，第二个参数是可变参数前最后一个有名字的变量（上例中为`count`）。初始化`va_list`列表，把接收到的可变参数全部放到list里。

`va_arg`：返回`args`指向的参数值，第一个参数是列表名，第二个参数是指定应该以什么类型返回。执行完成后，`args`自动指向列表中的下一个参数。

`va_end`：销毁`args`列表，释放空间

**注意事项**

1. 可变参数的访问必须从列表中第一个可变参数开始访问，不可以从中间开始，但可以不访问完就结束
2. 函数的参数列表中至少要有一个有名字的参数，否则`va_start`就用不了
3. 无法通过宏判断实际存在的参数数量和每个参数的具体类型
4. 务必注意在使用`va_arg`宏时，需要指定正确的类型

例如`printf`函数，它可变参数前一个参数是字符串，其中的`%d`、`%c`这样的格式化字符串就隐式的展现了后面使用的可变参数列表中可变参数的个数和具体的使用类型。
