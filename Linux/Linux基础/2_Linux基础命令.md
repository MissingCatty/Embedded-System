# 2 Linux文件操作命令

## 2.1 命令基础格式

```shell
command [-options] [parameters]
```

例如：`ls -l /home/alice`（以列表的形式显示文件夹中有什么内容）

## 2.2 ls命令

作用：列出目录下的内容

```shell
ls [-a -l -h] [路径]
```

Linux系统的命令行在启动时，**默认加载到`home`目录**（该目录为每个操作用户的个人目录，路径为`/home/username`）

|      options      |                             注释                             |
| :---------------: | :----------------------------------------------------------: |
|         a         | all，列出所有文件和文件夹，包含隐藏文件和文件夹（以`.`开头） |
|         l         |                  list，以列表的形式展示信息                  |
|         h         |                    列出文件大小（带单位）                    |
| la或al<br />-l -a |                           组合显示                           |

## 2.3 pwd命令

`print work Directory`，打印当前所在路径

## 2.4 路径

- 相对路径：以当前目录为起点，无须以`\`开头
- 绝对路径：以根目录为起点，以`\`开头

- 特殊路径符

  | 特殊路径符 | 解释       | 示例                             |
  | ---------- | ---------- | -------------------------------- |
  | .          | 当前目录   | ./dic 进入当前目录中的dic目录    |
  | ..         | 上一级目录 | ../dic 进入上一级目录中的dic目录 |
  | ~          | home目录   | ~/dic 进入home目录中的dic目录    |

## 2.5 mkdir命令

```shell
$ mkdir [-p] Linux路径
```

`-p`：用于创建不存在的父目录，适用于创建连续多层级目录

如：在`/home`下想要直接创建`/home/mid/last`（此时没有`/home/mid`父目录），可以使用`-p`；否则，由于父路径`mid`未创建，则`last`目录创建失败

## 2.6 touch命令

创建文件

```shell
$ touch Linux路径
```

例如：

```shell
$ touch /home/zyc/mid/last/test.txt
```

## 2.7 cat命令

查看文件内容，仅仅在命令行显示全部内容，不能进行其他操作

```shell
$ cat Linux路径
```

## 2.8 more命令

查看文件内容，支持翻页查看（按空格翻页，按q退出）

```shell
$ cat Linux路径
```

## 2.9 cp命令

复制文件和文件夹

```shell
$ cp [-r] 参数1 参数2
```

- `r`：递归复制，复制指定目录及其下的所有文件和子目录
- 参数1：Linux路径，被复制的文件/文件夹
- 参数2：Linux路径，表示要复制去的地方

## 2.10 mv命令

移动文件或文件夹

```shell
$ mv 参数1 参数2
```

- 参数1：Linux路径，表示被移动的文件
- 参数2：Linux路径，表示要移动去的地方

## 2.11 rm命令

删除文件或文件夹

```shell
$ rm [-r -f] 参数1 参数2 ... 参数n
```

- `-r`：递归删除，删除指定目录及其下的所有文件和子目录
- `-f`：force，强制删除。普通用户删除不会弹出提示，如果不使用`-f`超级管理员`root`删除某个文件时会提示是否删除（因为权限太大了，怕误删），使用`-f`就不会提示了
- 参数1 ... 参数n：一次性删除多个

| 通配符   | 含义                       |
| -------- | -------------------------- |
| `test*`  | 匹配任何以`test`开头的内容 |
| `*test`  | 匹配任何以`test`结尾的内容 |
| `*test*` | 匹配中间包含`test`的内容   |

**注意：千万不要在`root`管理员下删除进行`rm -rf /`和`rm -f /*`，直接把根目录全部删除了**

## 2.12 which命令

查看命令所对应的程序文件存放位置（每一个命令本质上就是一个可执行文件）

例如，要查看常用的`pwd`命令的存放位置

```shell
$ which pwd
/usr/bin/pwd
```

## 2.13 find命令

搜索文件

```shell
$ find 起始路径 -name 文件名
```

find命令支持通配符，`*`代表匹配任意内容（包含空格）

**按文件大小查找文件**

```shell
$ find 起始路径 -size +|-n[kMG]
```

- +、-表示大于和小于
- n代表大小
- `kMG`表示单位，`kb`，`MB`，`GB`

例如：

- 查找小于10kb的文件：`find / -size -10k`

**注：文件名和文件大小支持同时使用**

## 2.14 grep命令

从文件中通过关键字过滤文件行

```shell
$ grep [-n -i -w -v -c] 关键字 文件路径
```

- `-n`：在结果中显示行号
- `-i`：忽略大小写
- `-w`：匹配整个单词，而非单词中的一部分（如：关键字为`error`，不会匹配`errors`）
- `-v`：显示匹配之外的行
- `-c`：显示匹配的行数
- 关键字：过滤的关键字（包含空格），建议使用`""`包起来
- 文件路径，**可作为输入**

假设某文件中包含如下内容：

```
Hello my name is Alice.
I am very glad to be friend with you.
```

要查找`e`出现在那些地方：

```shell
grep -n "e" test.txt
```

输出如下：

## 2.15 wc命令

统计文件的行数、单词数量

```shell
$ wc [-c -m -l -w] 文件路径
```

- `-c`：统计字节数
- `-m`：统计字符数
- `-l`：统计行数
- `-w`：统计单词数量
- 文件路径，**可作为输入**

**注意，当该命令与管道符组合使用时，统计的是命令行输出的内容**

例如：如果当前路径下有一个`tmp.txt`，且`./folder`中也有一个`tmp.txt`

```
find . -name tmp.txt | wc -l
```

实际统计的内容是左边的输出：

```shell
./folder/tmp.txt
./tmp.txt
```

所以，结果是2

## 2.16 管道符

管道符：`|`，将管道符左边命令的结果作为右边的输入

例如，使用管道符的方式统计某个文件中的内容

```shell
$ cat tmp.txt | wc -l
```

左边`cat tmp.txt`会在命令行输出`tmp.txt`中的内容，所以这种方法实际上**统计的是命令行的输出，而不真的是文件的内容**

---

**在`/dev`路径下，只列出名为`log`文件的具体信息，过滤其他的信息**

```shell
$ ls /dev -lh | grep -w "log"
```

即：先输出所有文件的信息，在使用`grep`过滤行

## 2.17 echo命令

在命令行输出指定内容

```shell
$ echo "输出内容"
```

## 2.18 反引号

被反引号包围的内容将作为命令被执行，而不是字符串

```shell
$ echo pwd		# 输出：pwd
$ echo `pwd`	# 输出：/dev
```

## 2.19 重定向符

- `>`：将左侧的命令的结果，**覆盖**写入到符号右侧的文件中
- `>>`：将左侧命令的结果，**追加**写入到符号右侧的文件中

## 2.20 tail命令

查看文件尾部的内容，追踪文件的最新更改

```shell
$ tail [-f -num] Linux路径
```

- `-f`：表示持续追踪尾部更改
  - 什么叫持续追踪？执行`tail -f -n 3 test.txt`会先输出最后的三行，此时不停止输出（光标在原地跳动等待），如果文件尾部被追加了一串新的内容，此时追加的内容就被自动输出到命令行
- `-num`：表示查看尾部多少行（即：从后往前数多少行），不填默认10行

例如，有文件`test.txt`内容如下：

```
hello
my
name
is
Alice
```

现在执行指令：

```shell
$ tail -3 test.txt
```

输出的结果为：

```
name
is
Alice
```

**注意：有的版本不支持`-3`，只支持`-n 3`**

#  3 Linux用户和权限

Linux系统有且仅有一个超级管理员，名为`root`

普通用户的很多操作是受限的，如：在根目录下创建文件夹（但一般在其`home`目录下是不受限的），出了`home`目录，普通用户对大多数地方，一般只有读取和执行权，无修改权

## 3.1 su命令

切换账户

```shell
$ su [-] [用户名]
```

- `-`表示在切换用户后加载环境变量
- 用户名省略，表示默认切换到`root`

切换用户后，使用`exit`命令返回上一个用户，也可使用**快捷键`ctrl+d`**

- 使用普通用户时，切换到其他用户需要输入密码
- 使用`root`用户时，切换到其他用户则不需要输入密码

## 3.2 sudo命令

为普通用户的命令授权`root`权限，临时以`root`身份执行

但并不是所有用户都有使用sudo的权利（虽然输了密码就能用），需要为普通用户配置sudo认证（就是给普通用户一个sudo的免密认证）

---

**为普通用户配置sudo认证**

1. 切换到`root`用户
2. 执行`visudo`命令，自动通过vi编辑器打开`/etc/sudoers`
3. 添加`username ALL=(ALL) NOPASSWD: ALL`

## 3.3 用户、用户组管理

Linux系统中可以：

- 配置多个用户
- 配置多个用户组
- 用户可以加入多个用户组中

![image-20241123214608176](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20241123214608176.png)

Linux中关于权限的管控级别有2个级别，分别是：

- 针对用户的权限控制
- 针对用户组的权限控制

比如，针对某文件，可以控制用户的权限，也可以控制用户组的权限。

---

**用户组管理**

需要root权限

- 创建用户组

  ```shell
  $ groupadd 用户组名
  ```

- 删除用户组

  ```shell
  $ groupdel 用户组名
  ```

---

**用户管理**

需要root权限

- 创建用户

  ```shell
  $ useradd [-g -d] 用户名
  ```

  - `-g`：指定该用户的组，如果不使用`-g`，则会自动创建一个与用户名同名的组，并将该用户假如该组（加入的组必须已经存在）
  - `-d`：指定该用户的HOME目录，如果不使用`-d`，则默认为：`/home/用户名`

- 删除用户

  ```shell
  $ userdel [-r] 用户名
  ```

  - `-r`：删除用户的HOME目录

- 查看用户所属组

  ```shell
  $ id [用户名]
  ```

  - 如果不提供用户名，则查看自身

- 修改用户所属组

  ```shell
  $ usermod 用户名 -G 用户组名1[,用户组2]...
  ```

  - 将指定用户加入用户组，如果原本的用户组没有出现在`usermod`命令的参数中，则会删除。例如：如果原本属于`用户组1`，`用户组2`，在使用`usermod -G 用户组1`后，`用户组2`就没了，只属于`用户组1`

- 查看用户和用户组

  ```shell
  $ getent passwd # 本质是查看/etc/passwd文件内容
  ```

  ![image-20241213210659898](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20241213210659898.png)

  上面图片中每一行是一个用户，每个字段的含义如下：

  `用户名 : 密码 : 用户ID : 组ID : 描述信息 : HOME目录 : 执行终端`

  ```shell
  $ getent group # 本质是查看/etc/group文件的内容
  ```

  ![image-20241213211204312](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20241213211204312.png)

  每个字段的含义为：`组名称 : 组认证 : 组ID`

- 第一列：权限控制信息

---

**权限控制信息**（**针对当前登录用户`zyc`而言对该文件/文件夹的权限**）

权限细节总共分为10个槽位：

![image-20241213212251403](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/machine-learning-pic/image-20241213212251403.png)

例如：`drwxr-xr-x`

- `d`表示文件夹

  | 文件类型 | 描述                                                         |
  | -------- | ------------------------------------------------------------ |
  | `-f`     | 普通文件，一些应用程序创建的，比如文档、图片、音乐等等。     |
  | `d`      | 目录文件。                                                   |
  | `c`      | 字符设备文件，Linux 驱动里面的字符设备驱动，比如串口设备，音频设备等。 |
  | `b`      | 块设备文件，存储设备驱动，比如硬盘，U 盘等。                 |
  | `l`      | 符号连接文件，相当于 Windows 下的快捷方式。                  |
  | `s`      | 套接字文件。                                                 |
  | `p`      | 管道文件，主要指 FIFO 文件。                                 |
- 该文件所属用户的权限是：该用户对此文件（夹）有`rwx`权限
- 所属用户同组用户的权限是：该用户组中所有用户对此文件（夹）有`r-x`权限
- 剩余其他用户权限是：`r-x`

（前三个对于当前登录用户而言，中间三个对于②，后三个对于③）

**三个权限的具体解释**

- `r:read`
  - 对文件，可查看文件内容
  - 对文件夹，可查看文件夹内容

- `w:write`
  - 对文件，可修改文件内容
  - 对文件夹，可在文件夹内创建、删除、改名等操作

- `x:execute`
  - 对文件，可将文件作为程序执行
  - 对文件夹，可通过`cd`命令进入该文件夹

## 3.5 chmod命令

- 作用：修改文件（夹）权限信息

  ```shell
  $ chmod [-R] 权限 文件（夹）
  ```

  - `-R`：对文件夹内全部内容应用同样的操作

例如：`chmod u=rwx,g=rx,o=x hello.txt`，将文件权限修改为`rwxr-x--x`（该命令也可以用`chmod 751 hello.txt`替代，`751→111 101 001`）

**注意**：只有文件（夹）**所属用户**和**root用户**才有权限修改

## 3.6 chown命令

- 作用：修改文件（夹）的所属用户和用户组

  ```shell
  $ chown [-R] [用户][:][用户组] 文件（夹）
  ```

  - `-R`：对文件夹内全部文件应用该操作
  - `用户`：修改所属用户
  - `用户组`：修改所属用户组
  - `:`：用于分隔用户和用户组

**注意：此命令只适用于`root`用户**
