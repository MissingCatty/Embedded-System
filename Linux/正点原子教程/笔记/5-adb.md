**ADB** 的全称是 **Android Debug Bridge（安卓调试桥）**。

在嵌入式开发中，它是一个极其强大的**多用途命令行工具**。简单来说，它就像是一座“桥梁”，让你能够用电脑（PC）通过 USB 数据线或局域网，直接控制和调试你的嵌入式开发板或安卓设备。

虽然它的名字里带有 "Android"，但在很多纯 Linux 的嵌入式开发板上，开发者也会专门移植 ADB 守护进程（adbd），因为它比传统的串口线调试或配置网络 FTP/SSH 要方便得多。

结合你做嵌入式项目的场景，ADB 最常用的核心功能有：

- **`adb shell`（进入设备终端）** 敲下这个命令，你的电脑终端就会变成开发板的 Linux 命令行。你可以在里面执行 `ls`、`cd`、`top` 甚至运行你前面那个编译好的 `itest` 图像测试程序，体验和接了串口线一模一样。
- **`adb push` / `adb pull`（极速传文件）**
  - 当你在电脑上交叉编译好了 `itest` 可执行文件，不需要插拔 SD 卡，也不用配置复杂的 TFTP 或 NFS 网络，直接 `adb push itest /userdata/` 就能把文件秒传到板子里。
  - 如果板子里生成了测试图片或崩溃日志，直接 `adb pull /userdata/error.log ./` 就能拉回到电脑桌面上分析。
- **`adb logcat`（查看日志）** 如果你的开发板跑的是带 Android 框架的系统，这个命令可以实时打印系统底层到应用层的所有日志，是排查死机、报错的神器。
- **`adb devices`（查看连接状态）** 用来检查你的电脑有没有成功识别到开发板。
