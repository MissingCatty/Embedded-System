# 2.Ubuntu文件系统

## 2.1 文件系统简介

Ubuntu 中，以树状结构表示文件夹与文件，没有盘符的概念，不像 Windows 有 “C盘”、“D盘”，Linux 只有一个“根” `/`（比如：`/abc/def/hello.txt`）。从名字`/abc/def/hello.txt`中你无法知道 `hello.txt` 文件位于磁盘哪一个分区。  

要想查看**某个分区挂载在哪一个目录下**，可以执行命令：  

```bash
df -h
```

> #### Linux中的分区和目录
>
> 分区**代表**的是真实磁盘中的区域（“代表”：指的是分区文件`/dev/sdax`等是Linux操作硬件的接口，**Linux通过对文件的操作，来操控真实的物理硬件**）
>
> | 层次       | 名称                        | 举例                          | 说明                 |
> | ---------- | --------------------------- | ----------------------------- | -------------------- |
> | **物理层** | 分区（Partition）           | `/dev/sda1`, `/dev/mmcblk0p2` | 硬盘或闪存的物理区域 |
> | **逻辑层** | 目录结构（Filesystem Tree） | `/`, `/home`, `/boot`, `/etc` | 用户看到的“文件树”   |
>
> Linux系统安装的底层过程为：
>
> 1. 先在磁盘上划分物理区域
>
>    ```
>    /dev/sda1 → 512MB → 用来存 /boot
>    /dev/sda2 → 50GB  → 用来存 /
>    /dev/sda3 → 80GB  → 用来存 /home
>    ```
>
> 2. 每个分区需要“格式化”为某种文件系统
>
>    ```
>    mkfs.ext4 /dev/sda2
>    mkfs.ext4 /dev/sda3
>    mkfs.vfat /dev/sda1
>    ```
>
> 3. 系统中有一棵“逻辑目录树”，根是 `/`
>
>    ```
>    /
>    ├── boot/
>    ├── home/
>    ├── etc/
>    ├── var/
>    ```
>
> 4. 把物理分区**挂载**到逻辑目录上
>
>    ```
>    mount /dev/sda2 /
>    mount /dev/sda1 /boot
>    mount /dev/sda3 /home
>    ```
>
> #### 分区挂载的覆盖
>
> 根据上面的例子，根目录`/`被`/dev/sda2`挂载，而根目录下面的两个子目录`/boot`和`/home`分别被另外两个分区挂载。
>
> 那么，之后挂载的文件系统会覆盖原来的该目录对应的文件系统。具体来说，当根目录`/`被`/dev/sda2`挂载后，其所有子目录都属于`sda2`分区，但后来`/boot`又被`sda1`挂载，此时`/boot`目录下的内容又都属于`sda1`分区了。`/boot`目录下的所有原有内容仍然原封不动的存在`sda2`里（前提是还没有被其他数覆盖），之后`/boot`下文件操作都是在`sda2`里了。
>
> ```
> # 假设 /boot 目录中原来有一个文件 old.txt
> ls /boot
> => old.txt
> 
> # 挂载一个新的分区上去
> mount /dev/sda1 /boot
> 
> # 现在 /boot 里面看到的文件来自 /dev/sda1，而 old.txt 看不到了
> ls /boot
> => vmlinuz, initrd.img, grub/
> ```

## 2.3 文件属性

在终端中执行

```bash
ls -a -l
```

会列出当前目录下所有文件和文件夹的详细信息。

![image-20251113201828777](https://zyc-learning-1309954661.cos.ap-nanjing.myqcloud.com/25%2F11%2F4f28f7ef5f9a0fe217bdeac2721d5c06.png)

- 第一组字符（`drwxr-x---`字样）：1（文件类型） + 3 + 3 + 3（后面每三个为一组）

  | 符号 | 含义                                       |
  | ---- | ------------------------------------------ |
  | d    | 目录                                       |
  | -    | 文件                                       |
  | l    | 链接文件                                   |
  | b    | 设备文件里可供存储的接口设备               |
  | c    | 设备文件里的串行端口设备，如：鼠标、键盘。 |

  - 第一组表示“文件所有者的权限”  
  - 第二组表示“ 用户组的权限”  
  - 第三组表示“其他非本用户组的权限”  

  每组都是rwx 的组合， 其中 r 代表可读， w 代表可写， x 代表可执行。

- 连接数： 表示有多少文件名连接到此节点。

- 文件所有者：表示这个文件的“所有者的账号”。

- 文件所属用户组。

- 文件大小：表示这个文件的大小，默认单位是 B(字节)。

- 文件最后被修改的时间： 这个文件的创建文件日期或者是最近的修改日期

- 文件名：对应文件的文件名（如果文件名之前多了一个“.”,则说明这个文件为“隐藏文件”， 执行`ls -a`命令可以列出隐藏文件）

# 3.Linux常用命令

## 3.1 pwd

全称：print working directory

| **选项** | **说明**                     | **示例** |
| -------- | ---------------------------- | -------- |
| 无       | 显示当前工作目录的绝对路径   | `pwd`    |
| `-L`     | 显示逻辑路径（默认行为）     | `pwd -L` |
| `-P`     | 显示物理路径（解析符号链接） | `pwd -P` |

## 3.2 mkdir

| **选项**    | **说明**                                         | **示例**                      |
| ----------- | ------------------------------------------------ | ----------------------------- |
| 无          | 创建一个新的空目录                               | `mkdir new_directory`         |
| `-p`        | 创建父目录，如果父目录不存在，则一并创建（递归） | `mkdir -p /home/user/new/dir` |
| `-v`        | 显示每个创建的目录（verbose）                    | `mkdir -v new_directory`      |
| `-m <mode>` | 设置新目录的权限（如 `755`）                     | `mkdir -m 755 new_directory`  |
| `--help`    | 显示帮助信息                                     | `mkdir --help`                |

## 3.3 ls

| **选项** | **说明**                                                | **示例**    |
| -------- | ------------------------------------------------------- | ----------- |
| 无       | 列出当前目录中的文件和文件夹                            | `ls`        |
| `-l`     | 使用长格式列出文件详细信息，包括权限、所有者、大小等    | `ls -l`     |
| `-a`     | 显示所有文件，包括以`.`开头的隐藏文件                   | `ls -a`     |
| `-h`     | 与 `-l` 一起使用，显示人类可读的文件大小（例如 KB, MB） | `ls -lh`    |
| `-R`     | 递归列出所有子目录中的文件                              | `ls -R`     |
| `-S`     | 根据文件大小排序                                        | `ls -S`     |
| `-t`     | 根据修改时间排序                                        | `ls -t`     |
| `-r`     | 反向排序                                                | `ls -r`     |
| `--help` | 显示帮助信息                                            | `ls --help` |

## 3.4 cp

| **选项** | **说明**                            | **示例**                                |
| -------- | ----------------------------------- | --------------------------------------- |
| 无       | 复制文件或目录                      | `cp file1.txt file2.txt`                |
| `-r`     | 递归复制目录                        | `cp -r dir1/ dir2/`                     |
| `-i`     | 在覆盖文件前询问用户                | `cp -i file1.txt /path/to/destination/` |
| `-v`     | 显示复制过程中的详细信息（verbose） | `cp -v file1.txt /path/to/destination/` |

## 3.5 cat

| **选项** | **说明**                        | **示例**          |
| -------- | ------------------------------- | ----------------- |
| 无       | 显示文件内容                    | `cat file.txt`    |
| `-n`     | 显示行号                        | `cat -n file.txt` |
| `-b`     | 类似 `-n`，但不为空白行显示行号 | `cat -b file.txt` |
| `-E`     | 显示文件中的行尾符号（$）       | `cat -E file.txt` |

## 3.6 rm

| **选项** | **说明**                            | **示例**         |
| -------- | ----------------------------------- | ---------------- |
| 无       | 删除文件                            | `rm file.txt`    |
| `-r`     | 递归删除目录及其内容                | `rm -r dir1/`    |
| `-f`     | 强制删除，不提示确认                | `rm -f file.txt` |
| `-i`     | 删除前询问确认                      | `rm -i file.txt` |
| `-v`     | 显示删除过程中的详细信息（verbose） | `rm -v file.txt` |

## 3.7 touch

| **选项** | **说明**                                                     | **示例**                         |
| -------- | ------------------------------------------------------------ | -------------------------------- |
| 无       | 创建空文件（如果文件已存在，则更新其时间戳）                 | `touch newfile.txt`              |
| `-c`     | 如果文件不存在，不创建新文件                                 | `touch -c existingfile.txt`      |
| `-t`     | 使用指定的时间戳格式修改文件时间（例如：`touch -t 202201010000 file.txt`） | `touch -t 202201010000 file.txt` |
| `-a`     | 只修改访问时间（不修改修改时间）                             | `touch -a file.txt`              |
| `-m`     | 只修改修改时间（不修改访问时间）                             | `touch -m file.txt`              |
